Lab Note B5-003
Feedback · 04 — Setup

The rig and the signal path

Two ways to build a send-into-itself loop: an Ableton return that feeds its own input, and a compact-mixer aux loop — the original dub patch. Both get the same non-negotiable backstop: a master limiter at -3 dBFS and a kill route in reach. Wire the ceiling first, then the loop.

The signal path, end to end

Source → send → Loop: filter + saturator → Echo + Reverb Utility trim ↺ self-send Master limiter -3 dBFS
Safety first — wire this before anything else

Put the limiter on the master and set the kill route before you create the self-send. A loop with no ceiling is the one that hurts. Monitor level down. Then, and only then, raise loop gain.

Route A — the Ableton return self-send (recommended)

Route A

A return that feeds its own input

Master backstop first
  • On the master, add a Limiter. Ceiling -3 dBFS, fastest lookahead. This is the catch, never the sound.
  • Turn the monitor controller down to a quiet level. You will bring it up only once the loop is stable.
  • Open ⌘ , → Sends and enable Allow Sends to be assigned to other return tracks (the A↔B routing toggle) — this is what lets a return send back toward itself.
Build the loop return
  • Create two return tracks (⌘ ⌥ T): A — LOOP and B — TAP.
  • On A — LOOP, build the device chain in this exact order: EQ Eight (HP 200 Hz 24 dB/oct, LP 4 kHz 24 dB/oct) → Saturator (Soft Sine, output ceiling on) → Echo (sync, 3/16, its own feedback 20 % only) → Hybrid Reverb (decay 3–4 s, 100 % wet) → Utility (gain trim, start at -6 dB).
  • The filter is first so every pass is band-limited before the saturator and the time effects act on it.
  • The self-send: on A — LOOP show its sends, and set its send to B — TAP to -inf for now. Route B — TAP's output back to A — LOOP (Audio To → A — LOOP). B — TAP is the wire that closes the loop; A — LOOP's send-to-B is the loop-gain control.
Map the safety fader
  • ⌘ M → click A — LOOP's send-to-B knob → move a Launchpad fader/control. Set the mapping Min -inf, Max 0 dB. Down is fully counter-clockwise — confirm the direction now.
  • Map a second control to a master mute or to A — LOOP's Activator. That is your instant kill.
  • Exit map mode. The mouse never touches loop gain during a take again.
Find unity
  • Send a single seed into A — LOOP (one chord stab). Monitor quiet.
  • Raise the self-send fader by a hair at a time. Repeats start to thicken and regenerate. Stop the instant they stop decaying — that point is unity.
  • Back the fader off ~10 %. Mark the position. The Utility's -6 dB trim is your guaranteed decay even if the fader creeps.

Route B — the compact-mixer aux loop (the original dub patch)

Route B

Aux return trimmed back to aux send

The patch
  • Mixer Aux Send → outboard delay (or the Digitakt II / a software delay on a hardware return) → delay output → mixer return channel.
  • The feedback: raise the return channel's own Aux Send a hair, so a little of the delayed signal goes back out the aux bus and through the delay again. That return-to-send trim is the loop-gain knob — and it is a performance instrument.
  • Keep a hand on it the whole time. This is exactly how dub engineers played the desk.
The ceilings, in hardware
No undo means more layers of protection
Master backstopBring the mixer's main out into Ableton or a hardware limiter set to -3 dBFS before the monitors. Never feed raw mixer-feedback straight to amplified speakers.
Filter in the loopHP/LP on the delay's input or insert — band-limit before it re-enters the aux bus.
Loop-gain trimThe return channel's aux send. Mark "home" and "danger" with tape on the desk.
Kill routeThe return channel's mute, or pulling the aux-master fader. Know which one you'll grab before you start.
Monitor levelDown. Bring up only when stable. First test in headphones off or very low.

The gain ceilings — the numbers that keep it musical

Where to cap the loop, either route
Utility / trim in the loopNegative gain (start -6 dB) so each pass loses a fixed amount — guaranteed decay even if other knobs creep up
Saturator drive in the loopDrive + output ceiling means loud repeats compress and dirty rather than climb — soft-clipping is its own limiter
Feedback / self-send targetFind unity by ear (repeats stop decaying), then back off ~10 %. Live there; visit unity only as a brief, hands-on gesture
In-loop filterHP 200 Hz, LP 4 kHz — bands outside this can never compound
Master limiter ceiling-3 dBFS, fast — never the sound, always the catch if the loop slips past unity
Decision check
  • Is the master limiter set to -3 dBFS and the monitor level down before any self-send exists?
  • Is the in-loop filter the first device in the chain, so every pass is band-limited?
  • Is loop gain on a physical fader, and do you know without looking which way is down?
  • Can you reach the kill in under a second from playing position?
