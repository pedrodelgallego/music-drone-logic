Lab Note B4-003
Granular · 04 — Setup

The rig and the resampling loop

Granular work has one trap above all others: a feedback path you don't intend, or a CPU spike you can't repeat. This page wires the instruments — Ableton Granulator III, Grain Delay, the MicroFreak as a granular source, Digitakt slicing — into one predictable signal path with a deliberate resampling loop at its centre. Keep it open beside the build.

The instruments, by role

Granulator III

  • The main engine — plays one loaded file as grains
  • Size, Position, Spray, Density exposed as macros
  • Freeze plus a decoupled pitch keyboard — play chords from one frozen instant

Grain Delay

  • An effect, not an instrument — granulates whatever audio passes through
  • Frequency, Spray, Pitch, Feedback — clouds from a live signal
  • Used on a return for grain-haze you can send any track into

MicroFreak

  • Sample / Grain Stretch engine — a granular source in hardware
  • Wave = position; modulate it with LFO or cycling envelope
  • Records the original source note this whole track grows from

Digitakt II

  • Slices a recorded cloud and triggers slices per step
  • Sample start / length p-locks reassemble without new samples
  • Resamples a processed pattern back to a slot — generations on the box

The signal path

From source to cloud to print

MicroFreak note → record → SOURCE clip → load → Granulator III → FX → Resample track print new generation
Recommended path

Record one MicroFreak note as the single source. Load it into Granulator III on a MIDI track. Set a dedicated audio track's input to Resampling to print clouds. Keep a Grain Delay on a return for haze. The Digitakt enters later, fed a resampled cloud over the M4 to slice. One source file, one engine, one print track — everything else is generations.

Set up the project

Setup 01

Project, tracks, and the resample track

New set
  • ⌘ N, switch to Arrangement with Tab. Tempo 110 — this is textural music; the grid is loose. Time signature 4/4.
  • ⌘ , → Audio: device MOTU M4, sample rate 48000 Hz, buffer 256 (drop to 128 if scrubbing Position feels laggy; raise to 512 for the mix).
  • ⌘ , → Record, Warp & Launch: WAV, 24-bit. Turn OFF Auto-Warp Long Samples — you want untouched audio to granulate.
Create the tracks
  • ⌘ ⇧ T for a MIDI track → GRANULATOR. This hosts Granulator III.
  • ⌘ T for audio tracks: SOURCE REC (records the MicroFreak), RESAMPLE (prints generations), DT SLICE (records the Digitakt later), CLOUD A, CLOUD B, STUTTER for the printed layers.
  • ⌘ ⌥ T twice for returns: A — GRAIN HAZE (Grain Delay), B — REVERB (Hybrid Reverb, decay 6–8 s, 100 % wet).
  • On SOURCE REC: Audio From → Ext. In 1 (MicroFreak mono). On RESAMPLE: Audio From → Resampling. On DT SLICE: Audio From → Ext. In 3/4.
Project hygiene for generations
  • ⌘ S → save as granular_lab_b4-003.als in a dedicated folder.
  • Keep a text file generations.txt next to it. Every resample gets one line: gen2 = gen1 frozen @0.42s, size 90ms, spray 12%. This is non-negotiable — resamples are irreversible.
  • File → Collect All and Save at the end of each session.
Setup 02

Map Granulator III to four macros

The four dials, where they live
Granulator III — assign these to Macros 1–4 (right-click → Map to Macro)
Macro 1 — PositionThe grain start point in the file. This is the one you scrub and freeze. Range it across the useful part of the source, not the whole file.
Macro 2 — Grain SizeFrom ~20 ms to ~120 ms. Below 50 ms = texture; above = pad.
Macro 3 — Spray0 to ~80 ms of random position scatter. Tiny amounts hide the loop; large amounts dissolve pitch.
Macro 4 — Density / GrainsHow many voices overlap. Higher = smoother and denser; lower = sparse and rhythmic.
  • Engage Freeze so Position stops following any played note and sits where Macro 1 parks it.
  • Set the grain window (envelope shape) to a smooth bell to start — sharpen it only when you want grit.
  • Map Macro 1 to a Launchpad control later (⌘ M) so you can scrub Position by hand while everything plays.
Decision check
  • With Freeze on and a note held, does turning Macro 1 scrub through the source like scrubbing tape? If the pitch jumps instead, Freeze is off.
  • Does Macro 2 cross audibly from buzz to pad around the middle of its range? If it never smooths out, raise density.
Setup 03

Grain Delay on the haze return + Reverb

Return A — Grain Haze
Grain Delay starting values
Frequency~ 8 Hz — how often it samples the input into grains
Spray40–60 ms — scatters the grains into a haze rather than a clean echo
Pitch0 st to start; later, −7 or +12 st for shifted shimmer copies
Feedback20–35 % — keep it short of self-oscillation on a return
Dry/Wet100 % (it is a return — the send knob is the mix)
  • Put an EQ Eight after the Grain Delay, high-passed at 200 Hz — granular haze loves to build up mud.
Return B — Reverb
  • Hybrid Reverb, large algorithm, decay 6–8 s, pre-delay short, high-pass the reverb input at 300 Hz so the cloud's lows don't wash out.
  • This is the room every cloud sits in. Keep its low end disciplined or the whole piece turns to fog.
Setup 04

The resampling loop — and how to avoid feedback

How a resample is printed
  • Set RESAMPLE track Audio From → Resampling, Monitor → Off, and arm it.
  • Solo the tracks you want captured (e.g. GRANULATOR + its sends). Anything not soloed will not print.
  • F9 / Arrangement Record, capture the length you need, stop. Drag the printed clip to CLOUD A (or wherever it belongs) and rename it gen2.
  • Write the line in generations.txt immediately.
The feedback trap
  • Resampling captures the master output. If the RESAMPLE track is itself audible in the master while armed and monitoring, you get a feedback howl.
  • Keep RESAMPLE Monitor on Off at all times except the instant you check a take. Never leave it on In.
  • If you hear a rising tone the moment you arm, mute the RESAMPLE track's output or pull its fader — then print.
Digitakt as a slicing stage
  • Send a printed cloud out to the Digitakt: route CLOUD A output to the M4 so the Digitakt can sample it via its input, or transfer the WAV to the box. Sample it into a slot.
  • On the Digitakt, set the sample to Slicing, slice by transient or by an even grid, and trigger slices per step. P-lock sample start for finer cuts the cloud never had.
  • Use Retrig to machine-gun a single slice into a granular-style stutter at sequencer resolution.
  • Resample the Digitakt pattern back to a slot for a further generation, or record it into DT SLICE over the M4 (Ext. In 3/4).
Decision check
  • Can I print a 16-bar cloud to RESAMPLE and drag it out without a feedback howl? If not, fix monitoring before the build.
  • Does my generations.txt have a line for the source recording itself (gen1)? Start the log there.
  • Carry forward: the rig is one source, one engine, one print track, two returns. Page 05 gives you the grain settings — recipes for pad, cloud, stutter, and glitch — that you will dial into Granulator III and the haze return throughout the build.
