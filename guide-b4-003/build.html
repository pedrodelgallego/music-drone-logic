Lab Note B4-003
Granular · 06 — The build

Step-by-step: a textural piece from one source cloud

Now we make the track. One short recording becomes a 6–8 minute textural piece through freezing, scanning, slicing, and a chain of resamples. Every step is small, every step has a decision checkpoint, and every dial value is spelled out. Keep the setup page open for the rig and the grain palette open for the recipes — this page assumes both. Keep generations.txt open too; you will write in it eight times.

Step 01

Record the single source — generation 1

Choose what to record
  • You need one short sound with a clear, stable pitch. A single MicroFreak drone note, a struck string, a sung vowel. Two to four seconds of audio is more than enough — the whole track lives inside it.
  • For this build, use the MicroFreak: Shift + press Type to initialise, choose the Harmo engine for an organ-like, harmonically rich tone. Set the root to C3. Wave/Timbre/Shape around 40 / 55 / 50. Filter cutoff at 1 o'clock — you want harmonics present, because grains feed on harmonic content.
  • Long ENV attack and release so the note swells and never clicks. No LFO movement yet — the source should be static; all movement will be granular.
Capture it clean
  • Arm SOURCE REC (Ext. In 1), Monitor Auto. Press Hold on the MicroFreak, play C3, let it sustain.
  • ⌘ ⇧ R — record about 6 seconds. Stop. You want a clean swell with a sustained, stable middle — that middle is where you will freeze.
  • Peaks at -12 to -6 dBFS. A source recorded too hot leaves no headroom once dozens of grains overlap and sum.
  • Save the MicroFreak patch as SRC HARMO. Trim the recorded clip to the cleanest 3–4 seconds with ⌘ E, and consolidate (⌘ J) into one file named gen1_source.
  • Log it: open generations.txt and write gen1 = MicroFreak Harmo C3, 4s, clean swell.
Decision check
  • Does the source have a clearly nameable pitch? Hum it back. If you can't, the granular chord later will be mush — re-record.
  • Is there a stable sustained section, not just attack and decay? You need an instant worth freezing.
  • Is it harmonically rich but not noisy? Rich feeds the grains; noisy never becomes harmony.
Step 02

Load the source and find the freeze point

Into Granulator III
  • Drop gen1_source onto the GRANULATOR MIDI track to load it into Granulator III.
  • Engage Freeze. Set the dials to the Pad recipe to start: Grain Size 100 ms, Density high, Spray 5 ms, Window smooth bell.
  • Draw or play a single held C3 in the clip so the engine sounds while you scrub.
Scrub Position to the best instant
  • With the note holding, slowly turn Macro 1 — Position across the file. Listen for the spot where the timbre is richest and most stable — usually the sustained middle, not the attack.
  • Park Position there. This frozen instant is the harmonic seed of the entire piece. Note its rough value (e.g. 0.42 s) — you will reference it.
  • Now play a chord: hold C3 — G3 — E4. Because pitch is decoupled from position, one frozen instant becomes a sustained triad. This is the texture-into-harmony move, live.
Decision check
  • Scrub Position slowly end to end once more — is the parked instant genuinely the best one? Commit only after you've heard the alternatives.
  • Does the chord read as harmony, or still as noise? If noise, lengthen grains toward 120 ms and drop spray to 0.
  • Can you hear the loop point ticking? Add 3–5 ms of spray — just enough to hide it.
Step 03

Print the pad — generation 2

Play and record the chord progression
  • Plan a slow, three-chord movement in C natural minor: Cm — A♭ — E♭, each chord held about 16 bars at 110 BPM (roughly 35 seconds each). This is the harmonic skeleton of the whole piece.
  • Draw these as long held MIDI notes in the GRANULATOR clip across ~100 bars. The frozen position stays put; only the played pitches change.
  • Add a slow LFO to Spray inside Granulator (one cycle per ~18 s, depth ±4 ms) so the pad shimmers without any audible event.
Resample it
  • Solo GRANULATOR and its reverb send. Arm RESAMPLE (Audio From → Resampling, Monitor Off).
  • F9, capture the full progression, stop. Drag the printed clip to CLOUD A, rename gen2_pad.
  • Log it: gen2 = gen1 frozen @0.42s, size 100ms, spray 5±4ms LFO, chords Cm-Ab-Eb.
  • This printed pad is now committed audio — the previous settings are gone. That is intentional: the pad is the bed, and it no longer costs CPU or risks being knocked out of tune.
Decision check
  • Played quietly, does the progression feel like a piece beginning — or like a patch demo? It should have somewhere to go.
  • Is the printed pad clean, with no clicks at the chord changes? If a change clicks, the held notes overlap badly — re-draw with a short crossfade between chords.
Step 04

Scan a cloud over the pad — generation 3

Switch Granulator to the Cloud recipe
  • Reload gen1_source into Granulator (or duplicate the track). Set the Cloud recipe: Grain Size 60 ms, Density high-minus, Spray 30 ms.
  • Unfreeze Position, and put an LFO on Macro 1 — Position: one full pass per 45 s, full file range. The read point now scans slowly through the source the whole time.
  • Play the same chord progression an octave up (C4 — A♭4 — E♭4) so the cloud sits above the pad rather than fighting it.
Send to the Grain Haze, then print
  • Send the cloud generously to Return A — Grain Haze. Set Grain Delay Pitch to +12 st for a shimmering octave double, Spray 50 ms, Feedback 30 %.
  • Solo Granulator + both sends. Print to RESAMPLE, drag to CLOUD B, rename gen3_cloud.
  • Log it: gen3 = gen1 scanning 45s LFO, size 60ms, spray 30ms, +haze +12st.
Decision check
  • Play pad + cloud together: do they interlock, or fight in the same frequency band? If they clash, EQ the cloud with a high-pass at 300 Hz so it floats above the pad.
  • Is the scan glacial enough? If you can hear the timbre "move" as an event, halve the LFO rate to one pass per 90 s.
Step 05

Slice a cloud on the Digitakt — generation 4

Get the cloud onto the box
  • Take gen2_pad (or a fresh print of a denser cloud) and sample it into a Digitakt slot — transfer the WAV, or route CLOUD A out over the M4 and sample its input. Keep it to ~4 bars of audio.
  • On the Digitakt, set the sample to Slicing. Slice by an even grid first — 16 slices — so each slice is a clean, equal fragment of the cloud.
Reassemble into a pattern the cloud never played
  • Trigger slices per step on a 16-step pattern, but out of order: place slice 1 on step 1, slice 9 on step 3, slice 4 on step 6, and so on. The cloud is now re-sequenced into a rhythm it never contained.
  • P-lock sample start on two or three trigs for finer cuts — fragments the even slicing didn't give you.
  • Add gentle swing (52–54 %) and one Retrig on a single slice every few bars for a granular-style stutter (Recipe 3, on the grid).
  • Keep it sparse and tonal — this is texture made rhythmic, not a drum pattern. No kick.
Record the slice pass
  • Arm DT SLICE (Ext. In 3/4), Monitor Auto. Record 2 minutes while performing slice mutes and the occasional retrig.
  • Log it: gen4 = gen2 sliced x16 on Digitakt, reordered, 1 retrig/4bars.
  • Optional further generation: resample the Digitakt pattern back to a slot and granulate that — a cloud of slices of a cloud. Log it as gen5 if you do.
Decision check
  • Does the reassembled pattern relate to the source, or has it become fully new material? Either is valid — but decide which you wanted before keeping it.
  • Is it adding rhythm without becoming a beat? If it reads as drums, thin it until it reads as pulsing texture.
Step 06

Build a glitch transition — generation 6

Dial the Glitch recipe
  • Reload gen1_source. Set the Glitch recipe: Grain Size 15 ms, Density low-medium, Spray 80 ms, Position jumping (LFO on Macro 1, fast random or stepped).
  • Sharpen the grain window for grit. Add a hard pitch element — automate Granulator pitch to drop −12 st across the event.
  • This deliberately tears the source apart. It is a transition — a tear in the fabric between sections — not a layer that runs throughout.
Print a short burst
  • Print just 8 bars of the glitch to RESAMPLE, drag to STUTTER, rename gen6_glitch.
  • Log it: gen6 = gen1 glitch, size 15ms, spray 80ms, pos jump, -12st sweep.
  • You now hold six generations, all descended from one 4-second recording: source, pad, cloud, sliced cloud, (optional sliced-cloud cloud), glitch.
Decision check
  • Is the glitch short enough to be an event? More than a few seconds and it becomes the bed — trim it.
  • Can you still tell, faintly, that it came from the same source? If it is pure noise with no lineage, pull spray back a little so a thread of pitch survives.
Step 07

Arrange the generations on the timeline

Mark the map first
  • Right-click the scrub area → Add Locator at each boundary: EMERGE, CLOUD IN, PULSE, PEAK, TEAR, DISSOLVE.
Section → time (textural, not beat-driven)
EMERGE — 0:00Pad alone (gen2), surfacing from silence
CLOUD IN — 1:30Scanning cloud (gen3) enters above the pad
PULSE — 3:00Sliced-cloud rhythm (gen4) brings pulse
PEAK — 4:30All layers, haze and reverb blooming — nothing new added
TEAR — 5:30Glitch burst (gen6) tears the texture for ~15 s
DISSOLVE — 6:00Layers strip away; pad alone returns and fades
END — 7:30Silence
Place the clips
  • EMERGE — gen2_pad from bar 1, with a long fade-in via the clip corner handle so it surfaces over ~20 seconds. Reverb send up; the room is the first thing the listener hears.
  • CLOUD IN — drop gen3_cloud in, high-passed at 300 Hz, fading in over 8 bars. Two layers now breathe against each other at different scan rates.
  • PULSE — gen4_slice enters quietly (-18 dB) — the first sense of rhythm, made of cloud fragments.
  • PEAK — all three layers present. Nothing new enters. Energy comes from automation (Step 08), not addition.
  • TEAR — gen6_glitch punctures the peak; duck the pad and cloud underneath it for the 15 seconds it plays.
  • DISSOLVE — remove cloud and slice; gen2_pad alone carries the exit, master fade over the final 30–45 s.
  • Make every edit on a bar line. Crossfade clip seams by 1 bar — granular layers butt-joined will click.
Decision check
  • Does the piece evolve continuously, or does it lurch between blocks? Lengthen fades until transitions are felt, not seen.
  • At PEAK, is the fullness coming from layers opening rather than from a new sound arriving? That is the thesis of the whole exercise.
Step 08

Draw the automation arc

What moves, and how slowly
  • This is drawn movement on devices sitting over the printed clips — Auto Filter, sends, Utility width. The granular movement is already baked into the audio; do not fake more by extreme plugin automation. If a moment needs more grain motion, go back and re-print the generation.
  • A for automation lanes, B to draw. No ramp shorter than 16 bars — most will be 32–64 bars. Bend ramps to S-curves so they read as intention.
Automation map
Pad Auto Filter cutoff @ 0:00 → 4:30800 Hz → 4 kHz — the pad opens across the whole rise
Cloud volume @ 1:30 → 4:30-inf → -8 dB — the cloud blooms as it scans
Grain Haze send @ 4:00 → 4:30-10 dB → -2 dB — haze peaks exactly at PEAK
Reverb send @ 4:30up to -3 dB — space blooms at the top
Pad + Cloud volume @ 5:30 (TEAR)duck -6 dB under the glitch, recover after
Master volume @ 6:45 → 7:30slow fade to silence
Perform Position by hand
  • If you kept a live Granulator track rather than only printed clips, ⌘ M and map Macro 1 — Position to a Launchpad control. Set the mapping Min/Max to the useful file range so you can't scrub into the dead attack.
  • Record one pass performing slow Position scrubs at PEAK — felt as the timbre slowly transforming. Slower than feels natural. Keep the calmer of two takes.
  • After: right-click each lane → Simplify Envelope to thin breakpoints while keeping the shape.
Decision check
  • If you can hear the automation move as an event, it is too fast.
  • Can you remove one automation lane without weakening the arc? If yes, remove it.
  • Play 4:00–4:30 blind: does PEAK arrive because layers opened, not because something was added?
Step 09

Mix the cloud

Carve the layers apart
  • Granular layers all share the same harmonic DNA, so they mask each other badly. Separate them by register: pad owns the low-mids, cloud floats above 300 Hz, slice-rhythm sits in the upper-mids.
  • CLOUD A (pad): EQ HP 80 Hz to clear sub-rumble the grains generate. CLOUD B: HP 300 Hz. DT SLICE: HP 400 Hz, gentle LP shelf above 9 kHz.
  • Watch for grain build-up: dozens of overlapping grains sum into far more low-mid energy than the source had. A broad -2 dB dip around 250–400 Hz on the pad usually clears the fog.
Levels and checks
Starting balance
Pad (gen2)-10 dB
Cloud (gen3)-14 dB
Slice rhythm (gen4)-18 dB
Glitch (gen6)-12 dB, only during TEAR
Return A — Grain Haze-10 dB
Return B — Reverb-8 dB
Master peak≤ -3 dBFS
  • Mono check: flip a Utility to mono — granular haze widened too far collapses to nothing. Pull width back where it vanishes.
  • Quiet check: listen once at low volume; the true balance shows. If the pad disappears, the cloud is too loud.
  • No limiter — this is a learning piece, not a master. Protect the headroom you left.
Step 10

Export, archive the generations, step away

Render
  • Select 0:00 to 7:30. File → Export Audio / Video. Master, WAV, 48 kHz, 24-bit, Normalize OFF. Filename granular_b4-003_v1.wav.
Archive the lineage
  • File → Collect All and Save — every generation's WAV is now inside the project folder.
  • Confirm generations.txt is complete: one line per generation, gen1 through gen6. This file is the recipe — without it the track is unreproducible, because every resample erased its own settings.
  • Save the MicroFreak patch SRC HARMO and the Digitakt project.
Listen elsewhere
  • Copy to phone and laptop speakers — brutal on the low-mid grain build-up. Note timestamps, not feelings: 3:10 cloud masks pad.
  • Wait one day. Change one thing. Render v2.
  • What you built: a 7:30 textural piece in which every sound descends from a single 4-second recording, transformed through six logged generations of freezing, scanning, slicing, and glitching. The piece evolves over minutes from movement that is mostly baked into the audio, not drawn on top of it. Page 07 closes the loop.
