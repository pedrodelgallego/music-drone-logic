Lab Note B6-001
Rule-based Sequencing · 01 — Overview

Compose a piece whose pattern is generated by a rule-set you wrote down first

A single-topic deep reference for turning a musical intention into explicit, conditional, probabilistic rules — on the Digitakt II, in Live 12's MIDI tools and Max for Live, and in free code paths. You author the boundaries; the system fills them in. The track must be fully original: no borrowed patterns, no copied reference.

The goal

A rule-generated original

  • A 4–8 minute piece whose part-writing follows a rule-set you can recite
  • Made with Ableton Live 12 Suite, Arturia MicroFreak, Elektron Digitakt II — plus free code paths
  • Fully original: you author the conditions, not a borrowed pattern

Fundamentals you'll practice

  • Writing intentions as one-sentence conditions before touching a sound
  • Digitakt trig conditions; Live 12 Generators & Transformations; Max for Live logic
  • Bounded randomness — designing the box, then letting chance move inside it
Core objective

Author a system that makes the small decisions for you — and always be able to say what the rule decided versus what you decided in advance.

Why this exercise matters

The beginner instinct

  • Reach for a randomizer and call the drift "generative"
  • Turn up probability everywhere until nothing is predictable
  • Keep whatever the first roll produced

The opposite discipline

  • Write the rule as a sentence first; the machine only executes it
  • Spend the conditional budget on logic, chance last and sparingly
  • Run the rule long enough to hear its habits, then curate

The skill transfers far beyond techno. The same instinct — make the intention literal, fence the chance, listen back, choose — underpins generative ambient, algorithmic scores, live-coded sets, and any system you want to run longer than you can hold your hands on it.

Principle

A rule you cannot state in one sentence is not yet a rule — it is a hope.

How this Lab Note is laid out

From intention to system

  • 02 Listen — three pieces where the rule is the composition
  • 03 Decisions — rules theory, audio vs control path, bounded vs unbounded
  • 04 Setup — the rig and where each rule gets written

Then you build it

  • 05 Rule vocabulary — named rules with worked examples
  • 06 The build — an exhaustive step-by-step rule-generated track
  • 07 Reflection — fault, carry-forward, and the principle that survives
  • The rig: Ableton Live 12 Suite + Max for Live, Arturia MicroFreak, Elektron Digitakt II.
  • Free code paths, named throughout: VCV Rack for patching logic without buying modules, and Pure Data / SuperCollider / TidalCycles for writing the rules as code.
  • The throughline: keep the audio path and the control path separate, write the rule before the sound, and bound every chance so surprise reads as a choice.
