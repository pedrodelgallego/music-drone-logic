Lab Note B6-001
Rule-based Sequencing · 04 — Setup

The rig, and where each rule gets written

This page wires the control path and the audio path so that one clock drives everything and every rule has exactly one home. Keep it open beside the build — it is the routing and condition reference the next page assumes.

The signal path, decided once

Read this left to right: the control path generates decisions, the instruments turn them into sound, Ableton records and curates. Clock flows the same direction so nothing drifts.

Live clock master → Rules (DT cond / MIDI / Max / code) → MicroFreak + Digitakt II → MOTU M4 → Live (record + curate)
Step 01

One clock master, sync everything to it

Clock and transport
  • Ableton is the clock master. Set tempo 124, 4/4. A rule-based piece must run on one clock or the conditions count against drifting bar lines.
  • ⌘ , → Link, Tempo & MIDI. Enable Sync on the Digitakt II output row and the MicroFreak output row — Live sends clock and transport out.
  • Digitakt II: Settings → MIDI Config → Sync — enable clock and transport receive, disable clock send. Exactly one master.
  • MicroFreak: Shift + Tempo → source USB.
  • Verify: press play in Live; both boxes start instantly and read 124. A 1:4 condition is meaningless if the boxes count their own bars.
Audio return path
  • MicroFreak Audio Out → MOTU M4 Input 1 (mono TS). Digitakt II Main L/R → M4 Inputs 3/4 (TRS).
  • Create audio tracks MF VOICE (Ext. In 1) and DT DRUMS (Ext. In 3/4), monitor In while programming, Auto before recording.
  • Peaks around -12 to -6 dBFS. Headroom matters more here because a probabilistic accent can stack onto a downbeat unexpectedly.
Step 02

Digitakt II — conditions as a rule language

Where the rules live on the box
  • Hold a trig and turn the COND (trig condition) encoder. Each value is a sentence about when the trig is allowed to fire.
  • Per-track scale lengths and per-track scale (key) are pitch and rhythm fences set in the track length / scale settings.
  • Parameter locks (hold a trig, turn any knob) are per-step rules about timbre — a different decision frozen onto one step.
Digitakt II trig conditions — read them as rules
PREFires only if the previous conditional trig did play. Chains events into cause-and-effect.
PRE (negated) / NEIFires only if the previous one did not — the two branches of an if/else.
1:4, 2:4, 1:8"On the Nth pass of M." A guaranteed, countable heartbeat of variation across bars.
FILL / FILL (negated)Plays only while FILL is held, or only while it is not — a performable A/B without a second pattern.
FIRST / NOT FIRSTDistinguishes the loop's first cycle from every repeat after — an intro built into the loop.
% (e.g. 35 %)The one probabilistic condition. The system rolls a die each time. Use it last, and sparingly.
Step 03

Live 12 — Generators, Transformations, and MIDI effects

Where the rules live in Ableton

All of these act on MIDI before it reaches an instrument — control path upstream of audio path. That is the point: the rule is editable forever, the sound stays swappable.

Where to write rules in Live 12
MIDI clip — Velocity / probability per noteLive 12's per-note chance and velocity ranges: a note can be set to play, say, 70 % of the time, within a velocity range. Bounded randomness, built in.
MIDI clip — GeneratorsEuclidean, Rhythm, Stacks, Seed seed material from a rule. The generator is the rule; you curate the output it writes into the clip.
MIDI clip — TransformationsArpeggiate, Connect, Quantize, Strum, Recombine — rules that reshape existing notes. Non-destructive until you commit.
Random MIDI effectAdds bounded pitch deviation — set Chance and Choices to fence how far it can stray.
Note Length / Velocity / Scale MIDI effectsCheap, stackable rules. The Scale effect is itself a boundary: it forces stray notes back into a key — your pitch fence.
Max for Live — custom logicWhen the built-ins run out, a small Max patch is the control path written as a patch: probability gates, bouncing notes, state machines. Keep it generating MIDI, never audio, so the two paths stay separable.
Step 04

Free code paths — the same idea, written down

No hardware required

Every condition on the Digitakt has a one-line equivalent in free software. If you have no box, you can do this entire Lab Note in code — the discipline (write the rule, fence the chance, run it long, curate) is identical.

The rule, in four free environments
VCV RackPatch logic with no modules to buy. A clock → Bernoulli Gate is a probability condition; a counter + logic gate is a 1:4; sample-and-hold off a noise source is bounded random pitch into a quantizer (your pitch fence).
Pure DataA probability gate is two objects: [random 100] into a [< 35] test gating a bang. Counters and [moses] build counted conditions. The control path made of boxes and lines.
TidalCyclesA conditional pattern is a one-liner: every 4 (# crush 4) $ s "bd*4" applies a transform on every fourth cycle. sometimesBy 0.35 is the bounded coin.
SuperColliderA full stochastic part is a Pbind with Pwrand (weighted choice over an allowed set) and Pseq for the fixed spine — the box and the freedom in one expression.
Routing a code path into the rig (optional)
  • Any of these can drive your hardware: send their MIDI/clock out and let the MicroFreak or Digitakt be the audio path while the code is the control path.
  • Or render them straight to audio and curate in Ableton alongside the hardware takes. Either way, keep the generated material and the rule that made it in the same labelled place.
Setup check
  • Press play: do both boxes start, stop, and read 124 from Live's clock? If not, fix sync before writing any condition.
  • Can I point at exactly one place where each rule will live? No rule should exist in two homes.
  • Is every rule still upstream of audio — MIDI, conditions, or code — with nothing committed to a recorded take yet?
  • Do my record levels leave headroom for a probabilistic hit landing on a downbeat?
