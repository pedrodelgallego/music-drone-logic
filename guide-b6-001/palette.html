Lab Note B6-001
Rule-based Sequencing · 05 — Rule vocabulary

A vocabulary of rules, with worked examples

A rule-set is built from a small number of recurring shapes. Learn these as named building blocks — each stated in one sentence, with where to write it and a worked example you can run today. The build page assembles a piece out of exactly these.

The seven rule shapes

Each shape answers one question — when does a thing happen, which thing, or how much. Reach for them in the order on the Decisions page: deterministic, then counted/logical, then chance.

Shape 01

Fixed — the unconditional spine

The rule in one sentence
  • "This always plays." No condition, no chance. The element the listener locks to.
  • Home: Digitakt trig with COND off; an unconditional MIDI note; Pseq in SuperCollider.
Worked example
  • Kick on steps 1, 5, 9, 13, COND off. Closed hat on the offbeats, COND off.
  • Why: every other shape below is judged against this. Without a fixed frame, variation reads as mess, not as choice.
Shape 02

Counted — every Nth pass

The rule in one sentence
  • "This fires on the Nth pass of every M loops." Predictable, countable variation.
  • Home: Digitakt 1:4 / 2:4 / 1:8; TidalCycles every 4; a counter + comparison in Pd/VCV.
Worked example
  • Rim on step 11, condition 1:4: a punctuation once every four loops. You can hear the bar it lands on coming.
  • Why: this is the workhorse. It creates long-form motion without a coin flip and survives a defence — you authored exactly when it happens.
Shape 03

Conditional chain — if/else across steps

The rule in one sentence
  • "This fires only if the previous conditional trig did (or did not)." Cause and effect between events.
  • Home: Digitakt PRE and NEI; gate logic in Max/Pd/VCV.
Worked example
  • Ghost snare on step 7, condition PRE, placed after the rim in scan order: it answers only when the rim fired.
  • Add an alternative hat on the same step with NEI: when the rim is silent, the hat fills the gap instead. One if/else, two outcomes.
  • Why: chains make the loop feel like it remembers — the listener senses consequence without a new instrument.
Shape 04

Positional — first versus the rest

The rule in one sentence
  • "This behaves differently on the loop's first cycle than on every repeat after."
  • Home: Digitakt FIRST / NOT FIRST; a one-shot gate in code.
Worked example
  • A tom on step 1 with FIRST announces each new section once, then disappears for the repeats — an intro folded into the loop itself.
  • Why: it gives a phrase a beginning without writing a second pattern.
Shape 05

Bounded choice — pick from an allowed set

The rule in one sentence
  • "When this fires, choose one value from this fixed set." The fence and the freedom in one move.
  • Home: Digitakt sample-lock or p-lock across a few trigs; Live's Random MIDI effect with limited Choices into a Scale effect; Pwrand over an allowed list in SuperCollider.
Worked example
  • A bass note generator allowed only A1, C2, E2 (the root triad), chosen freely per trig but never leaving the set.
  • In Live: a one-note MIDI clip → Random (Choices = 3, Range fenced) → Scale (A minor) so any stray result is pulled back into key.
  • Why: this is bounded randomness in its purest form — surprise that always lands in tune.
Shape 06

Drift — a slow rule on a parameter

The rule in one sentence
  • "Over N bars, parameter X moves from A to B." A process, not an event — the Spiegel idea.
  • Home: a slow synced/unsynced LFO; Live automation drawn as a long ramp; a slow line/phasor in code.
Worked example
  • "Filter cutoff opens from 600 Hz to 3 kHz over 64 bars, then closes back over 32." A single sentence that shapes minutes.
  • Stack two drifts at different rates that never align — the combination never repeats exactly. That non-repetition is the hypnosis mechanism.
  • Why: drift is how a short rule-set fills a long piece without anything new entering.
Shape 07

Chance — the single coin

The rule in one sentence
  • "This fires P % of the time." The system rolls a die. The weakest, most expensive condition.
  • Home: Digitakt % condition; Live per-note chance; sometimesBy in Tidal; a Bernoulli gate in VCV.
Worked example
  • A single closed-hat flutter on step 15 at 25 %: rare, never the backbone, the one place you let chance author.
  • Why: use it last and once. If muting it doesn't make the groove feel less alive, it was decoration — leave it muted.

Principle

You will only ever need a handful of these shapes. A piece feels designed not because the rules are clever, but because each one is named, placed on purpose, and could be removed and defended.
