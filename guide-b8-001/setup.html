Spec & Voice · 04 — Setup

From spec to session template

A spec that never reaches the rig is a wish. This page is the bridge: how each line of the Engineering Spec becomes a concrete object in the workshop session — a track, a return, a clock setting, a mapped control. Build the template once, and every future piece in this voice starts from a session that already obeys the rules.

The workshop rig, named once

The Engineering Spec names the rig so a stranger could rebuild it. This is that rig. The System Spec does not mention any of it.

The signal path is the spec made physical

Constraint → Engineering line → Track / return / map → Template .als → Every future piece

If the template can violate a constraint, the constraint is not yet enforced. The work of this page is to make the rules structural — built into the session, not held in your memory.

Mapping the spec onto the session

Map 01

Element budget → track count

Build only the tracks the budget allows
  • If the constraint reads "no more than five sounding elements," create exactly five audio tracks and stop. The empty space below them in the session is the constraint, visible.
  • Name them after their role in the spec, not their gear: FOUNDATION, VOICE, SPACE, TEXTURE, ACCENT. A track named MicroFreak is engineering leaking into intention.
  • Resist the sixth track. Adding it is the precise moment the spec is broken — and the template should make that addition feel like a deliberate transgression, not a convenience.
Returns are part of the budget too
  • ⌘ ⌥ T for return tracks: A — REVERB (Hybrid Reverb, 100 % wet) and B — ECHO (Echo, high-passed at 250 Hz).
  • If the spec says "stock effects only," the return chain is your proof: nothing on it that did not ship with Live. Write the device list into the Engineering Spec so the ban is auditable.
Map 02

"One clock, one truth" → sync settings

Declare the master, enforce it in MIDI prefs
  • The Engineering Spec names exactly one clock master. On the workshop rig that is Ableton.
  • ⌘ , → Link, Tempo & MIDI. Enable Sync on the Digitakt II and MicroFreak output rows so Ableton sends clock and transport out.
  • On Digitakt II: Settings → MIDI Config → Sync — receive clock and transport, disable clock send. On MicroFreak: Shift + Tempo → source USB.
  • Verify: press play; both boxes start instantly and read the session tempo. Two clock masters is the most common silent violation of a spec — fix it before anything else.
Map 03

Parameter ranges → mapped controls

A range in the spec becomes a Min/Max on a map
  • The Engineering Spec specifies ranges, not just parameters. "Bass filter rides 600 Hz–3.5 kHz" becomes a literal Min/Max on a mapped Launchpad control.
  • ⌘ M for MIDI map mode. Click the device parameter, tap the Launchpad control, then type the range into the mapping's Min and Max fields.
  • Now the controller physically cannot overshoot into harshness. The constraint is enforced by the instrument, not by your willpower at the peak.
  • Map only the controls the spec permits — usually four: the load-bearing filter, its resonance, one send, one feedback. More maps than that and "all change is modulation" quietly becomes "all change is whatever I grab."
Map 04

"Movement is modulation" → a recording discipline

Printed vs drawn movement, decided in the template
  • Printed movement lives inside recorded audio — MicroFreak knob rides, Digitakt p-locks and LFOs. Drawn movement is Ableton automation on devices sitting on top.
  • If the spec bans new elements after a point, the template should hold long static takes plus automation lanes — and forbid duplicating a clip into a fresh section to fake an arrival.
  • Set record preferences once: ⌘ , → Record, Warp & Launch, WAV / 24-bit, and turn off Auto-Warp Long Samples so clock-synced takes stay untouched.
Save the template, not just the set
  • File → Save Live Set As, then File → Save Live Set as Default Set only if this voice is your main practice — otherwise keep it as spec_template.als in a dedicated folder.
  • The template is the Engineering Spec compiled. When the spec changes, you edit the template; when the gear changes, you edit both — and the System Spec stays untouched, which is the whole point.

The mapping at a glance

Spec line → session object
"Five elements maximum"Five named audio tracks; no sixth
"Stock effects only"Return + track device list, all native to Live
"One clock master"Ableton sends sync; boxes receive only
"Filter rides 600 Hz–3.5 kHz"Mapped control Min/Max set to that range
"No element enters after 3:00"Long static takes + automation; no duplicated clips
"No transition under eight bars"Grid snap on; automation ramps drawn 8 bars or longer
Decision check
  • Does every line of my Engineering Spec correspond to a concrete object in the template?
  • Is there exactly one clock master, verified by pressing play?
  • Can the template physically violate any constraint? If yes, enforce it structurally before building.
  • Are my tracks named after roles in the spec, or after gear?
