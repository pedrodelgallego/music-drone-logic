Lab Note B3-003
Polymeter & Phase · 04 — Rig & setup

The rig for disagreeing clocks

Polymeter and phase put unusual demands on the signal path. The Digitakt II is the polymeter engine — its per-track length is the single feature this whole study leans on. Ableton is the phase engine, achieved by duplicating a clip and detuning one copy's tempo. This page wires both, and is the reference the build assumes.

Who does what

Digitakt II — polymeter engine

  • PER TRACK scale mode: each track its own length
  • Kick stays 16; perc/texture set to 15, 14, 13…
  • The disagreement is internal; Ableton just records it

Ableton Live — phase engine

  • Duplicate a clip to two tracks, drift one's tempo
  • The clock master for both hardware boxes
  • Records the Digitakt polymeter as evolving audio
The division of labour

Polymeter happens inside the Digitakt, locked to Ableton's one clock. Phase happens inside Ableton, after recording, by drifting a duplicated copy. Never try to phase the hardware live against the clock — you would only be fighting the sync you carefully set up.

The signal path

Audio path

Digitakt II Main L/R → 2 × TRS → M4 In 3/4 + MicroFreak Out → TS → M4 In 1 → USB-C → Ableton → DT 990 Pro

Control + clock path

Ableton → MIDI clock + transport → Digitakt II + MicroFreak
One clock to rule them all

Exactly one device generates tempo: Ableton. The polymeter is a property of track lengths, not of multiple clocks — both Digitakt tracks run at the same 16th-note pulse, they simply wrap at different counts. If you ever see a second clock master, the realignment math breaks.

Configuring the Digitakt II for per-track length

Setup 01

PER TRACK scale mode

Find the setting
  • Press FUNC + PAGE to reach the SCALE menu (the page that controls pattern length and speed).
  • At the top, switch the mode from PER PATTERN to PER TRACK. This is the switch the whole Lab Note depends on: it lets every track carry its own LEN independently.
  • In PER TRACK mode the SCALE page now shows the selected track's length. Select a track first, then set its LEN.
Set the lengths
A worked polymeter assignment
T1 KICKLEN 16 — the fixed reference clock. Never changes.
T2 HATLEN 16 — also locked to the kick, for a stable backbone.
T3 PERCLEN 15 — the drifting voice. Realigns with the kick every 15 bars.
T4 TEXTURELEN 14 — a second, faster drift (7-bar realignment with the kick).
  • The CHNG (change length) parameter governs after how many steps the pattern advances to the next — leave it long or off so a short track simply loops within the pattern rather than triggering a pattern change.
  • Each track also has a SPD (speed) value. Leave every track at 1× for true polymeter. Changing speed turns it into something closer to polyrhythm — powerful, but a different study.
Verify before moving on
  • Press play in Ableton: the Digitakt starts, shows the project tempo, and stops on command.
  • Watch the step LEDs: T1 wraps at 16, T3 visibly wraps at 15 — its loop point walks left relative to T1 each bar.
  • If every track wraps together, you are still in PER PATTERN mode. Go back and switch to PER TRACK.

Configuring Ableton for clip phasing

Setup 02

Phasing a duplicated clip

The principle
  • Record one short, looping figure to a clip. Duplicate it to a second track with ⌘ D so both tracks hold the same audio.
  • Make the copy run at a fractionally different effective rate so it slides ahead of the original — by the end of a long span it has travelled one full pattern-length and rejoined unison. That slide is the phase.
  • The original stays warped to the grid (the anchor). The copy is the one you drift.
Three ways to create the drift
Pick one — they trade control for smoothness
Warp-marker nudgeOn the copy, switch Warp off for analog-style sliding, or place two warp markers and drag the end one slightly later. The clip plays a hair slow and creeps behind. Most "Reich-like".
Automated micro-delayA Delay/Echo on the copy with feedback at 0, automate its delay time from 0 ms to one pattern-length-in-ms over your chosen span. Precise and undoable.
Clip BPM offsetSet the copy's clip BPM field a tiny fraction below the project tempo (e.g. 123.7 against 124). It drifts continuously. Easiest to set, hardest to land back in unison exactly.
  • Keep the drift slow: a full phase cycle over 32–64 bars. If a listener can tell it's "speeding up", it is too fast — they should only notice the texture thickened and then cleared.
  • High-pass the drifting copy a little so it sits behind the anchor rather than doubling its weight; two identical low-ends phasing turns to mud.
Verify before moving on
  • Played together at the start, the two clips sound like one (true unison, no flam).
  • Halfway through the span the copy is audibly displaced and emergent patterns appear.
  • At the end of the span the two return to unison — if they don't, your drift rate doesn't divide the span evenly; adjust it on the palette page's table.

Track layout for this study

Ableton tracks

MF FIGUREMicroFreak mono via Ext. In 1 — the short figure you will later duplicate and phase
MF FIGURE (phase)Duplicate of the above; the drifting copy
MF DRONEMicroFreak mono — the steady bed under the drift
Return A / BReverb / Echo
  • Record long, edit later. Polymeter only reveals itself over many bars and phase over tens of bars — a 16-bar take shows you nothing. Record 4-minute passes so the full realignment cycle is captured as audio you can then cut on its natural seams.
  • Don't quantise the recorded polymeter. Auto-warp would try to snap the drifting part back to the grid, destroying the very thing you recorded. Keep Auto-Warp off, as in the studio reference.
