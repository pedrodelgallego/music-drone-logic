Lab Note B7-001
Parameter Riding · 04 — Setup & rig

The rig, the signal path, and the controller mapping

The decisions from page 03 now become a physical rig and a set of mappings inside Live. This page wires the control surface, sets up the four maps with their ranges, and builds a layout you can play without looking at the screen. Keep the ride map page open alongside it — that is the reference card this setup produces.

The performance rig

FaderFox EC4

  • 16 endless encoders with displays — the rides
  • Encoders pick up; values never snap when touched
  • Per-encoder labels you read at a glance

Launchpad Pro MK3

  • Grid of pads — mutes and section edits
  • Quantised clip / scene launching
  • The discrete-move surface, not the filter hand

Ableton Push (alternative)

  • Touch strip + pressure pads + labelled encoders
  • One expressive hand plus clip duties together
  • Use instead of EC4+Launchpad if you own one box

Ableton Live 12 Suite

  • Hosts the recorded takes from Lab Note B1-001
  • Receives the maps; records the performed ride
  • The only MIDI clock master for the rig
Two rigs, same map

The cleanest split is EC4 for the four continuous rides and Launchpad for mutes — one surface per kind of gesture. If you only own a Push, put the four rides on its encoders and the mutes on its pads; the mapping logic below is identical.

The signal path

Two flows, exactly as in the studio note: audio out to your ears, control in from your hands. Nothing about the audio path changes for performance — what changes is that control data now drives the parameters live instead of being drawn after the fact.

Control path

FaderFox EC4 → USB CC → Ableton MIDI map → Auto Filter / Send / Echo + Launchpad → quantised → Track mutes

Audio path

Recorded takes → Track devices (filter, sends) → Master → M4 → DT 990 Pro / PA
  • The EC4 sends CC, not notes. Each encoder transmits a continuous controller number on a fixed channel. You map those CCs to parameters in Live — nothing about the encoder itself knows what it controls.
  • Plug controllers into the computer directly, not through an unpowered hub, so control data stays tight and the audio clock stays clean.
  • The master stays off every surface. The only level near a performance hand is the four mapped rides; the master lives on Live's master fader or the venue mixer, untouched.

Enable the control surfaces in Live

Setup 01

Connect the EC4 as a generic MIDI controller

In Preferences
  • Plug the EC4 in via USB. Open ⌘ , → Link, Tempo & MIDI.
  • Leave Control Surface set to None for the EC4 — you want raw CCs, not a scripted surface that grabs control of devices.
  • Find the EC4's MIDI input row. Enable Remote (this is what lets its CCs drive mapped parameters). Leave Track off.
  • On the EC4 itself, confirm a setup/group where the four encoders you'll use send distinct CC numbers on one channel. Note them — you'll see them appear in Live's MIDI map.
Add the Launchpad for mutes
  • Plug the Launchpad in. Control Surface: Launchpad Pro MK3, Input/Output: LPProMK3 DAW.
  • Press Session on the Launchpad — pads light up confirming the link to Live's Session view, where your clips and mutes live.
  • If you prefer mutes on the Digitakt's own mute mode instead, you can skip the Launchpad entirely; the box handles its own track mutes.
Setup 02

Place the four devices the rides will touch

On the tracks
  • MF BASS: drop an Auto Filter (low-pass, 24 dB). Its Cutoff is ride 01.
  • Return B — ECHO: confirm the Echo device is present. Its Feedback is ride 02.
  • MF DRONE: the track's Send to Return A (Reverb) is ride 03. No new device — the send knob already exists in the mixer.
  • Mutes: ride 04 targets the DT DRUMS sub-mutes (or the Digitakt's per-track mutes). If your drums are one consolidated clip, split them into kick / hat / perc lanes first so each can mute independently.
Pre-set the safe starting positions
  • Bass Auto Filter Cutoff to its closed value (~600 Hz) — the set starts dark and opens.
  • Echo Feedback to its floor (~35 %) — never start a set with the tension valve already cracked open.
  • Drone reverb send to its dry end (~-12 dB) — space arrives later, it is not the opening state.
Setup 03

MIDI-map the four controls with their ranges

Map mode
  • ⌘ M to enter MIDI map mode — mappable parameters turn blue.
  • For each ride: click the parameter once (e.g. the bass Auto Filter Cutoff), then turn the EC4 encoder you've assigned to it. Live captures the CC and creates the map.
  • The new mapping appears in the MIDI Mappings browser on the left, each row showing the CC, the target, and editable Min and Max fields.
Set Min/Max — this is the whole job
Type these into the mapping's Min/Max columns
Bass Auto Filter — CutoffMin 600 Hz · Max 3.5 kHz
Echo (Return B) — FeedbackMin 35 % · Max 70 %
MF Drone — Send A (Reverb)Min -12 dB · Max -3 dB
Mute pads (Launchpad)Binary — map each pad to a track's mute; no Min/Max, but quantise the pad's clip launch to 1 Bar
  • The encoder's full physical travel now covers exactly the musical range. A careless turn can no longer reach a value that hurts — the safety lives in the map, not in your restraint.
  • ⌘ M again to exit map mode and test each one. Turn an encoder; watch its parameter move only within the window you set.
Setup 04

Label the surface so you can play blind

On the EC4
  • Name each encoder in the EC4's own display: BASS FLT, FEEDBK, VERB. The display name is your in-the-dark reference.
  • Put a strip of tape under each encoder with the same name and an arrow for "darker/brighter." In a dim room the tape beats the screen.
On the Launchpad
  • Colour the mute pads by track family — kick one colour, hats another, perc another — so a glance tells you the state of the groove.
  • Group the four ride encoders physically together on the EC4 (top-left bank) so your hand finds them without searching.
Decision check
  • Can I turn each encoder with my eyes closed and know what it does? If I have to look at the screen, the labels aren't doing their job.
  • Does every continuous map move only inside its musical range? Push each encoder to both stops and confirm.
  • Do the mute pads land changes on the bar, not the instant I press? If a mute lurches mid-bar, fix the launch quantise.
  • Is the master truly off every surface? Wiggle every control and confirm the master fader never moves.

Fast troubleshooting

Symptom → check
Encoder does nothingEC4 input has Remote enabled; the right CC was learned; map mode was exited
Parameter snaps when touchedYou mapped a fader/absolute control, not an endless encoder — re-map to an EC4 encoder set to relative
Ride reaches a harsh valueMin/Max not set, or set too wide — type the musical range into the mapping row
Mute lands off the barClip/scene launch quantise not set to 1 Bar in Live
Master jumps unexpectedlyAn encoder was accidentally mapped to master — delete that mapping immediately
Two encoders move one parameterDuplicate map — remove the stray row in the MIDI Mappings browser
