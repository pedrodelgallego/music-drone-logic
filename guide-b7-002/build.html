Lab Note B7-002
Live Resampling · 06 — The build

Build a performable resampling section

A full, performable set section that feeds on itself — captured live, transformed, returned, retired, and recovered if it goes wrong. Each step is small, each has a decision checkpoint, and every routing move and latency number is spelled out. Keep the setup page open in another tab — it is the wiring this page assumes — and the decisions page beside it, since this page only executes what you settled there.

  • Read this once before touching anything. The ten steps below run in order, and the order is the safety. Steps 01–03 are cold setup — clock, channels, latency — done in rehearsal, never on stage. Steps 04–08 are the live loop itself: source, capture, transform, return, feed. Steps 09–10 are recovery and capture-the-take. Skipping ahead to the fun part (the capture) before the clock and latency are locked is the single most common way this technique fails in front of a room.
  • The whole section is one idea repeated. You will perform the same loop — arm, capture on the bar, transform once, return under the source, retire — three or four times across 64 bars. Each pass replaces the last. The build is not in adding; it is in the discipline of letting one generation live at a time. If that sounds like the restraint from every other Lab Note, it is: do less, but make each element evolve.
  • Originality lives in the source. A resampled layer is your own output, in your own key, at your own tempo — there is nothing to copy and nothing to borrow. The only way this section sounds like someone else's is if your source does. Build an original groove first; the loop will keep it original.
Step 01

Confirm the rig and the clock

Set the session
  • Open the live set. Tempo 124, time signature 4/4. The whole loop is built on a clean bar count, so the grid must be correct before anything else.
  • ⌘ , → Audio. Sample Rate 48000 Hz, Buffer Size 256 samples. Write this buffer size down — your measured latency offset is only valid for this exact value.
  • ⌘ , → Record, Warp & Launch. Record Quantization → 1 Bar. Turn OFF Auto-Warp Long Samples — your captures are already on the grid; auto-warp would smear their transients.
One clock master
  • Ableton is clock + transport master. In Link, Tempo & MIDI, enable Sync on the output row of every hardware box you sync.
  • Digitakt II / Octatrack: clock receive on, transport receive on, clock send off. Exactly one device sends clock.
  • Verify before continuing: press play. Every box starts instantly and reads 124. Press stop; all stop. If either fails, fix it now — every later step assumes one locked clock.
Decision check
  • Is there exactly one clock master, with everything else slaved?
  • Does every box start and stop with the transport, on the bar?
  • Is the buffer size noted, so the latency measurement in Step 02 stays valid?
Step 02

Build the capture and return channels

Capture track
  • ⌘ T → audio track RS CAPTURE. ⌘ R to rename. Audio From → Resampling. This taps the full internal mix, post-master-fader.
  • Monitor → Off. The capture track must never monitor — monitoring a resampling input is an instant feedback path.
  • In Session view, the track's clip slots will hold each capture. Their record length is governed by the 1 Bar quantization from Step 01.
Return track
  • ⌘ T → audio track RS RETURN. This plays the captured clip back into the mix. Keep it separate from RS CAPTURE so the engine can never record its own output.
  • Build the return chain in this order: EQ Eight (high-pass ~200 Hz, ready but bypassed) → Limiter (ceiling -1 dBFS, fast release).
  • Pull the RS RETURN fader to -6 dB as a starting ceiling. This is the gain discipline made physical; you will lower it, never raise it, live.
Map the live controls
  • ⌘ M → MIDI map mode. Map a single pad to arm the capture clip slot, a second to launch the return clip, and a fader/encoder to the RS RETURN volume.
  • Map the high-pass bypass and the transform control (sample start, or Simpler transpose) you chose on the palette page. Map four controls and no more — capture, launch, return level, transform.
  • Exit map mode. Tap each control once to confirm it responds. A control you discover is unmapped during the set is a control you will fumble.
Decision check
  • Are capture and return on separate tracks, so the engine cannot sample itself?
  • Is RS CAPTURE monitor set to Off?
  • Is the limiter last in the return chain, ceiling at -1 dBFS?
  • Are exactly four controls mapped, and does each respond?
Step 03

Measure the round-trip latency

The click-capture test
  • Program a single click on the downbeat of an empty 1-bar pattern on the clock master.
  • Arm the RS CAPTURE clip slot. On the bar, capture exactly one bar. Stop.
  • Drag the captured clip onto the timeline. Zoom all the way into bar 1. The click transient sits a few milliseconds after the grid line — that gap is the round-trip latency.
  • At 48 kHz / 256 samples expect roughly 10–20 ms total. Read the exact figure off the ruler — say it measures 14 ms.
Apply the compensation
  • Global: enter the measured value (e.g. 14 ms) in ⌘ , → Audio → Driver Error Compensation.
  • Per-return: set a negative Track Delay of -14 ms on RS RETURN, so the replay fires early enough to land on the grid.
  • Re-test: capture the click again. It must now sit on the grid line. Do this once; it holds for the whole set as long as the buffer size from Step 01 does not change.
Why this matters more here than anywhere
  • A capture replayed over its source is the one case where a few milliseconds is audible as a flam or a smeared transient — the source and its echo are identical, so any offset between them comb-filters.
  • Measure once, correct, and forget it. An uncorrected loop will sound "almost tight," which is worse than obviously wrong, because you will chase it live instead of trusting the grid.
Decision check
  • Does the re-captured click sit exactly on the grid line?
  • Is the buffer size unchanged since the measurement?
  • Do you have the offset written down, so you can re-apply it after any audio-device change?
Step 04

Establish the source groove

Lay the bed the section feeds on
  • Run your live source — the Digitakt drums, the MicroFreak bass and drone — for at least 16 bars before you even think about capturing. The loop needs a source with character to feed on.
  • Find the densest, most characterful bar: the moment the perc and a long echo coincide. Note it. That is the bar you will capture, not a sparse intro bar.
  • Keep your hands off the master fader from here on. Resampling is post-master-fader; the master level is your capture level. Ride sends and channel faders instead.
Decision check
  • Has the source been running long enough to have a clear "best bar"?
  • Is the master fader parked at unity and untouched?
  • Do you know which role you want back — bed, counter-rhythm, or hook — before you arm?
Step 05

Arm early, capture on the bar

Arm in advance
  • One bar before the section you want to catch, arm the RS CAPTURE clip slot. Arming late is the classic failure: you decide to capture, then fumble while the moment passes.
  • Confirm the slot is set to record a fixed length — 4 bars for a counter-rhythm or hook, 8 bars for a bed. A clean count loops without a seam.
Trigger on the downbeat
  • Press the mapped capture pad at any time inside the bar. With Record Quantization at 1 Bar, the recorder waits for the boundary. The capture is on-grid even when your hand is not.
  • Let it run the full 4 or 8 bars and stop itself. Watch the clip's length confirm the exact bar count — not 4.1 bars, not 3.9.
  • Do one capture. Resist the urge to "grab a few while I'm here." A single deliberate capture is the material; a folder of half-takes is procrastination.
Decision check
  • Did the capture start exactly on a bar boundary?
  • Is the clip length a clean integer bar count?
  • Does the capture contain the dense, characterful bar you chose — not a sparse one?
Step 06

Apply one transform

Make it read as new — pick exactly one
One transform per capture — stacking them just makes mud
ReverseSwell-toward-the-beat. Double-click clip → Reverse. Best for a bed or a riser.
Pitch / octaveDown -12 st for a sub; up +12 st for a ghost line. Warp Mode Complex Pro. A hook — use once.
Sample-start chopDrag the start marker so the loop lands on an offbeat. A counter-rhythm — keep it under the source.
FreezeGrain Delay (Spray 60 ms, high Feedback) or a long reverb on a held slice. A bed — filter until it names nothing.
The transform test
  • Solo the transformed clip against the live source for four bars. If it sounds like the source with a delay on it, you have not transformed it — push the move further or choose a different one.
  • One move, then commit. A reversed, pitched, chopped, frozen capture is no longer in key; it is noise that happens to share a tempo.
Transform on hardware, in time, if you can
  • If your capture engine is the Digitakt II or Octatrack, perform the transform on the box rather than in Ableton: p-lock the sample start per trig, tune the pitch on the SRC page, or set the playback to reverse. Hardware transforms land on the clock automatically and read as a played gesture, not an edit.
  • If the transform is a filter or grain move, map it to one of your four controls and ride it in slowly across the bars the return is live — the transform itself can evolve, which is what keeps a single capture interesting for sixteen bars.
  • Whatever the route, the rule is unchanged: one transform. A control you ride is still one move; two stacked transforms is mud whether you played them or drew them.
Decision check
  • Does the replay read as a new voice, not a doubled mix?
  • Did you apply exactly one transform?
  • Does the transform match the role you decided on in page 03?
Step 07

Return it on the bar, under the source

Launch quantised
  • Move the transformed clip to RS RETURN. Set its launch quantization to 1 Bar so the replay drops in on a boundary, never mid-phrase.
  • Tap the mapped launch pad. The clip waits for the bar, then enters. With the -14 ms track delay from Step 03, its transients land on the source's transients.
Set the level and hold the ceiling
  • Bring the RS RETURN fader up to its -6 dB ceiling — no further. The capture already contains the full mix; it does not need to compete.
  • Watch the limiter's gain-reduction meter. Steady, occasional reduction is fine. Constant heavy reduction means the return is too hot — pull the fader, do not raise the ceiling.
  • If the capture is a bed, bypass-off the 200 Hz high-pass so it never doubles the source's low end. If it is a hook, leave the low end intact.

Decision check
  • Did the replay enter on a bar boundary, in time with the source?
  • Is the return at -6 dB or lower, with the limiter only catching peaks?
  • Could a stranger tell this layer is the set's own output? (If yes, transform harder.)
Step 08

Let the section feed on itself — safely

The one-capture-live rule
  • The mix now contains the source and the return. If you capture again now, the new capture contains the old one — levels compound, and the section turns to wall in four bars.
  • Retire before you re-capture. Before grabbing a second capture, fade the first return out (or mute RS RETURN), so the next capture samples a clean source again.
  • This is the discipline that separates a self-feeding section from a feedback accident: the loop is allowed to feed, but only one generation lives in the mix at a time.
Why one generation, in concrete numbers
  • Say the source sits at 0 dB reference and the return enters at -6 dB. Capture the combined mix and the new capture already carries the source plus a copy of itself; its loudest content is now hotter than the source alone. Return that at -6 dB and the second-generation copy is only -6 dB down from a signal that was already boosted — the effective level of the resampled content climbs with every round.
  • Three uncorrected generations and the resampled material is competing with the source; five and the limiter is pinned and the section is a wall. The limiter from setup stops it clipping, but it cannot make a wall musical.
  • Retiring the previous return before each capture breaks the chain. Every capture then samples the clean source at its known level, and the return ceiling means just what it says. This is the whole reason the section is built as a sequence of replacements, not a stack.
Build the section as a sequence of generations
A performable arc — each capture replaces the last, never stacks
Bars 1–16Source only. Establish the groove the loop will feed on.
Bars 17–32Capture the dense bar → reverse → return as a bed at -6 dB. The room deepens.
Bars 33–48Retire the bed. Capture again → sample-start chop → return as a counter-rhythm. New groove, same DNA.
Bars 49–56Retire the counter-rhythm. Capture → pitch up an octave → return as a single hook. The peak.
Bars 57–64Fade the return out entirely. The source carries the exit, retroactively bigger for having been fed.
Decision check
  • Is there ever more than one return generation live at once? (There must not be.)
  • Does each capture sample a clean source, with the previous return retired?
  • Does the section build by replacing layers, not by piling them up?
Step 09

The panic path

When the loop runs away
  • First move, always: pull the RS RETURN fader to -inf. One mapped fader kills the loop instantly without touching the source.
  • If the limiter is pinned and the sound is still building, you captured a mix that already contained a return — mute RS RETURN, let the source breathe for four bars, and start the next generation clean.
  • If a capture came back out of time, it is a clock or latency problem, not a level one. Stop the return, re-confirm one clock master, and re-check the Step 03 offset at the next break.
  • Rehearse this path until pulling the return fader is a reflex. The difference between a self-feeding section and a screaming feedback loop is one fader you can find without looking.
Decision check
  • Can you find and pull the return fader to silence without looking?
  • Does killing the return leave the source playing cleanly?
  • Have you rehearsed the recovery so it costs no more than four bars?
Step 10

Record the section and step away

Capture the performance, not just the loop
  • Arm Arrangement Record (F9) and perform the whole 64-bar section in one take. The captures, transforms, and retirements are arrangement moves — record them as they happen.
  • Do two passes. Keep the calmer one — the take where you trusted the grid and rode the return level slower than felt natural.
  • Export: File → Export Audio / Video, WAV, 48000 Hz, 24-bit, Normalize OFF. The headroom you protected with the gain ceiling is part of the sound.
  • Listen the next day, somewhere that is not the studio. Note one timestamp where the loop felt alive and one where it felt like a doubled mix. Change one thing. Perform it again.
Decision check
  • Did the full section survive one continuous take?
  • Across the whole take, did the return ever exceed the source? (It should not.)
  • Does the section sound like it generated its own material — or like a loop pasted over itself?
