Live Resampling · 04 — Setup

The rig and the signal path

The wiring reference the build page assumes. One capture engine, one clock master, one return channel with a gain ceiling — and a measured round-trip offset so the replay lands on the grid. Set this up cold, in rehearsal, before any of it has to work in front of a room.

Path A — Ableton resampling input

  • Create the capture track. ⌘ T for an audio track named RS CAPTURE. Audio From → Resampling. This taps the full internal mix post-master-fader — exactly the live output, nothing more.
  • Create the return track. A second audio/MIDI path named RS RETURN that plays the captured clip back into the mix. Keep capture and return separate so the engine never records its own output.
  • Set the clip record length. In Session view, give the capture clip slot a fixed length so the capture quantises to the grid, never to your reflexes. Record Quantization → 1 Bar.
  • Mind the post-fader tap. Resampling is post-master-fader, so the master fader level is your capture level. Leave the master at unity during a capture; ride sends and channel faders instead.

Path B — Digitakt II / Octatrack capture

  • Digitakt II. Route the source into the Digitakt's inputs (or sample its own main output) and use the sampling page to grab the buffer. The capture lands on a track you can immediately p-lock, pitch, and sequence — the tightest clock of the three paths.
  • Octatrack MKII. Assign a Recorder buffer to capture the incoming mix continuously; trigger the recording quantised, then play it from a Flex machine. The buffer and the playback share the same clock by design.
  • Feed the box, not a loop. Capture the source before the resample return re-enters it. Sampling a signal that already contains the previous capture is how the loop runs away.
Clock — exactly one master
AbletonClock + transport master. Sends clock out to all hardware; receives from none.
Digitakt IIClock receive on, transport receive on, clock send off.
Octatrack / MicroFreakSync source set to the same clock. No second device sends clock — ever.
Clock before compensation

A buffer recorded against free-running audio drifts and never loops clean — no latency offset can fix that. Lock one master clock first, confirm every box starts and stops with Ableton's transport, and only then measure the round-trip offset below.

Round-trip latency compensation

Output → converter → capture → converter → replay adds delay at every hop. Measure it once, in rehearsal, and bake the correction into the return so the captured transient lands on the grid instead of a few milliseconds behind it.

Measure

Find the round-trip offset

The click-capture test
  • Program a single click on the downbeat of an empty 1-bar pattern, clocked to the master.
  • Arm the capture engine and capture that one bar, quantised to 1 Bar.
  • Drop the captured clip on the timeline and zoom all the way into bar 1. The click transient will sit a few milliseconds after the grid line. That gap is the round-trip latency.
  • At 48 kHz with a 256-sample buffer, expect roughly 5–12 ms per direction plus converter latency — a typical total of 10–20 ms. Measure yours; do not trust the estimate.
Apply the correction
  • Ableton, global: enter the measured value in ⌘ , → Audio → Driver Error Compensation so monitored captures align automatically.
  • Per replay, manual: nudge the replay clip start earlier by the measured offset (e.g. drag it left ~15 ms, or set a negative track delay on RS RETURN).
  • Digitakt/Octatrack: use the sample start trim or track micro-timing to pull the replay a few ticks early until its transient lands on the source's transient.
  • Re-run the click test after correcting. The captured click should now sit on the grid line. Do this once; it holds for the whole set as long as the buffer size doesn't change.

The return channel and its ceiling

RS RETURN chain — in this order
1 — Channel faderReplay enters at least -6 dB below the source. The capture is the whole mix; it does not need to match the live level.
2 — EQ / filter (optional)High-pass around 200 Hz if the capture is a bed, so it never doubles the source's low end.
3 — LimiterCeiling -1 dBFS, fast release. The hard backstop that catches a runaway loop before it hits the master.
Gain ceiling on the loop

The limiter on RS RETURN is not for sound — it is a safety device. If a capture ever contains a previous capture, the level compounds; the limiter holds the line at -1 dBFS while you pull the fader. Set it now and never raise the return's headroom live.

Pre-set checklist

  • One capture engine chosen, wired, and rehearsed.
  • Capture track taps the live mix; return track is a separate channel.
  • One clock master; every box confirmed to start/stop with the transport.
  • Record quantization set to 1 Bar on both capture and replay.
  • Round-trip offset measured with the click test and corrected.
  • Return fader at -6 dB or lower, with a -1 dBFS limiter last in the chain.
