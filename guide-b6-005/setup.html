Lab Note B6-005
The Score · 04 — Setup

A rig that realizes a score honestly — twice

The score is written away from the rig. The rig's only job is to realize it — to perform the rules and print the result — and to do it twice under conditions identical in everything except the score's free zones. If R1 records hotter than R2, or against a grid R2 didn't use, the delta lies. This page sets the rig so the only thing that varies between takes is what the score allowed to vary.

The two flows: written and sounding

Before any cable, hold the shape in your head. The score lives on paper or in a text file or a drawing app. The rig turns one reading of it into audio. Nothing about the rig should make a musical decision the score didn't delegate to it.

the score (paper / text / drawing) → read → you, as realizer → perform → MicroFreak + Digitakt II + sources → MOTU M4 → Ableton print track → render → R1 / R2 WAV

  • Sources are open by design. The score may call for "any source not yet heard" — that can be the MicroFreak, a Digitakt sample, a field recording, or a struck object miked into the M4. Keep input 1 free for whatever you grab; load samples and field recordings onto Digitakt tracks ahead of time so they're available without browsing mid-take.
  • One print track per realization. Record each realization to a single stereo track. A realization is a performance, not an edit — resist comping it together from fragments.

The decision the score forces: grid or no grid

This is the one rig setting the score itself dictates. Set it from your fix/free choice on the timing dimension, made on the Decisions page.

Clock / sync follows the score
Score fixes timing (event-list with durations, a pulse, a process tempo)Work on the grid. Tempo 120 or whatever the score states. Ableton is clock master; Digitakt II receives clock + transport; MicroFreak source Shift+Tempo → USB. Both realizations use the identical tempo.
Score frees timing (graphic, text response rules, free durations)Work gridless. Right-click the timeline ruler → Time Ruler Format → Time so positions read m:ss. No metronome. R2's timing can then genuinely differ from R1's without the grid quietly forcing them into agreement.
Why this matters for the delta

If your score frees timing but you record both takes locked to a click, the grid becomes a hidden D rule you never wrote — both realizations will share a rhythm the score never specified, and your delta map will credit a "rule" that doesn't exist. Match the clock to the score, or the delta is a lie.

Project setup in Ableton

One set, two arrangements

  • Save as B6-005-score. Keep R1 and R2 in two named arrangements (or two saved sets) so neither overwrites the other.
  • Audio prefs: MOTU M4 in and out, 48 kHz, buffer 256. Input config: mono 1 and stereo 3/4.
  • Record prefs: WAV, 24-bit. Turn off Auto-Warp Long Samples — a realization is untouched audio.

Controls held constant

  • Same gain target for both takes — peaks ≈ -12 dBFS. A loudness difference in the delta should be a choice the score allowed, not a record-level accident.
  • Same monitoring level, same room, same print format, same duration window.
  • Everything except the score's free zones must match across R1 and R2.

Gain plan — set once, identical for both

Set in this order, then leave it for both realizations
1 — Source level (MicroFreak ~70–80 %, Digitakt main ~12 o'clock)Healthy line level without clipping the M4's fixed 3/4 inputs.
2 — M4 Input 1 gain (for whatever is on input 1)Meter green, occasional amber, never red. Note the setting so R2 matches it.
3 — Ableton print-track meterPeaks ≈ -12 dBFS. This is the number to match between takes.
4 — Phones knobComfortable, then do not touch — this is your A/B reference level for reading the delta.
Rig ready check
  • Can I record a 2–5 minute take to one stereo track and render it 24-bit WAV named R1?
  • Is the clock set to match the score — grid if timing is fixed, gridless m:ss if timing is free?
  • Have I written down the gain settings so R2 can be recorded identically?
  • Are the sources the score's free zone allows all reachable without browsing mid-take?
  • Is there a paper / text / drawing surface for the score itself, separate from the rig?
