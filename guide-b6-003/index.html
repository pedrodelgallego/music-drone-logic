Lab Note B6-003
Chance Operations · 01 — Overview

Letting chance decide the structure — inside a boundary you built

A deep reference for using dice, coins, and algorithms as a composer of structure — never of foundation. The lesson is Cage's, not the dice's: the radical act is authoring the field of allowed outcomes, then refusing to make the pick yourself. The track stays fully original; chance only chooses among options you would each have been happy to keep.

The goal

A chance-structured original

  • A 6–8 minute track whose form is determined by chance procedures you designed
  • Made with Ableton Live 12 Suite, Arturia MicroFreak, Elektron Digitakt II, and VCV Rack
  • Fully original; every outcome is one you composed the boundary for

Fundamentals you'll practice

  • Indeterminacy theory: outcome undetermined, field fully composed
  • Bounded chance on Digitakt trig conditions, Live per-note chance, VCV Bernoulli gates
  • Logging a procedure, running it many times, and curating what survives
Core objective

Understand how a chance operation becomes a composer only when you have already decided exactly what it is allowed to compose.

Why this exercise matters

The beginner instinct

  • Call any randomizer "generative" because it is unpredictable
  • Let chance decide whether core elements exist at all
  • Keep the first roll because re-rolling is effort

The opposite discipline

  • Decide what is never random — kick, key, form — before any roll
  • Compose a tight field where every result is usable
  • Run the procedure many times and curate instead of settling

The skill transfers far past techno: bounded indeterminacy is how generative scores, modular patches, algorithmic art, and live coding stay surprising without falling apart. You are not learning to add randomness — you are learning to relocate authorship to the boundary and let the coin work below it.

Principle

Chance is a composer only when you have already decided what it is allowed to compose. Otherwise it is a noise generator wearing a philosophy.

How this Lab Note is laid out

Theory, then rig

  • 02 Listen — Cage, Mozart's dice game, and tape collage as case studies
  • 03 Decisions — chance vs randomness, indeterminacy, the boundary
  • 04 Setup — the signal path from dice/coin/algorithm into the box
  • 05 Procedures — chance operations with worked, logged outcomes

Then build and reflect

  • 06 The build — the full step-by-step chance-structured track, procedure logged as you go
  • 07 Reflection — separate what chance decided from what you decided
  • Curation is the second composition pass, not an afterthought
  • The rig: Ableton Live 12 Suite + Max for Live, Arturia MicroFreak, Elektron Digitakt II, MOTU M4, and VCV Rack (free modular) for the Bernoulli-gate path. A physical six-sided die and a coin are real tools here, not metaphors.
  • The free code paths, named throughout: VCV Rack for a coin in the control voltage, and TidalCycles / SuperCollider for writing the chance function as code that chooses among a list you wrote.
  • The throughline: at any moment you must be able to write down the complete, finite list of things chance is currently choosing between. The day you can't, you've crossed from indeterminacy into noise.
