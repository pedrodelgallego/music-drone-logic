Lab Note B6-003
Chance Operations · 06 — The build

Build a track whose form is decided by chance you designed

This is the deep page. We build one complete 6–8 minute track end to end, and at each step we draw the line between what is decided and what is rolled, then log every roll. Keep the setup page open for wiring and the procedures page open for the six chance operations this build calls by name. Nothing here is random until we say it is — and when we say it is, we write down what happened.

Step 01

Write the constitution — what chance may never touch

Decide everything structural, on paper, first
  • Open a text file b6003_constitution.txt next to the project. Before any gear, write the fixed law of the track. This is the human composition; everything later is execution.
  • Tempo 124 BPM, time signature 4/4, key A natural minor, root A1. Eight-minute form with a single peak — these are inherited, fixed, and never rolled.
  • The kick is four-on-the-floor and exists in every bar of the body. Chance is forbidden from deciding whether the kick plays.
  • The three notes the bass may use: A1, C2, G1. The three steps it may land on: 1, 7, 11. Write these down — they are the fence for Procedure D.
Declare where chance is allowed to act
  • List the four hand-offs explicitly, so the file is the complete answer to "what is chance choosing between?":
  • 1 — which of the three notes the bass plays each bar (Procedure D).
  • 2 — whether the rare perc trig fires, at 35 % (Procedure B).
  • 3 — which of two texture samples loads each cycle (Procedure C).
  • 4 — the order of six pre-built 16-bar section variants across the body (Procedure A).
Set the re-roll budget
  • Write the rule: best of three for each chance decision. You will run every procedure at least three times and keep the run you most want to hear again.
  • This single line is what separates indeterminacy from settling. Without it, chance is just an excuse to stop deciding.
Decision check
  • Is every structural element on the "decided" side of the line? Kick, key, tempo, form, peak — none of them may be rolled.
  • Can a stranger read this file and list exactly what chance is choosing between? If not, rewrite until they can.
  • If you deleted the four chance hand-offs, would a complete (if static) track still be described? It must.
Step 02

Set up the project and confirm one clock

Ableton project
  • ⌘ N, Tab to Arrangement. Tempo 124, time ruler set to m:ss so the form plan reads in minutes.
  • Tracks (⌘ T): DT DRUMS, MF BASS, MF DRONE, ATMOS. Returns (⌘ ⌥ T): A — REVERB, B — ECHO.
  • Save as b6003_chance.als in a dedicated folder. Keep b6003_constitution.txt and a new b6003_rolllog.txt beside it.
Clock — one master, as always
  • In ⌘ , → Link, Tempo & MIDI, enable Sync on the Digitakt and MicroFreak output rows. Ableton is the only clock master.
  • If using the VCV path for Procedure D, slave VCV to Ableton over Link (VCV's Hub/clock module set to follow Link), and route its MIDI out to the MicroFreak.
  • Verify before any roll: press play. Digitakt reads 124, MicroFreak's synced LFO follows, VCV's clock light blinks in time. A chance take recorded against a drifting clock cannot be reproduced or repaired.
Ableton (clock master) → Link / MIDI → VCV chance engine → MIDI note → MicroFreak bass + analog audio → M4 → Ableton
Step 03

Build the fixed drum foundation

The part chance never touches
  • On the Digitakt, project CHANCE LAB, pattern A01. Assign T1 KICK, T2 HAT, T3 PERC, T4 TEXTURE.
  • Kick on 1, 5, 9, 13 — fixed, every bar, no condition. This is the spine the whole track hangs on.
  • Hat on 3, 7, 11, 15, plain trigs for now. Tune the kick's fundamental to A so it agrees with the bass.
Now place the first authored chance — Procedure B
  • On T3 PERC, one trig on step 8. Hold it, set the trig condition to 35 %. This trig plays sometimes — the only undetermined thing in the drum foundation.
  • Everything else on the drums is decided. Confirm: kick certain, hat certain, perc the single coin.
Decision check
  • Solo the drums for two minutes: is the groove complete without the 35 % trig ever firing? It must stand alone — the coin decorates, it doesn't carry.
  • When the trig lands, is it a pleasant surprise rather than a missing-puzzle-piece? If you feel its absence as a gap, it's load-bearing and the condition is wrong.
Step 04

Build the chance bassline — Procedure D, logged

The MicroFreak patch (decided)
  • Shift + Type to init. Oscillator Wavetable, monophonic, root at A1. Cutoff at 11 o'clock, short envelope — the standard hypnotic bass voice.
  • The sound is fully composed. Only the note choice is handed off.
Wire the chance engine — pick one path
  • VCV path: random voltage → sample-and-hold clocked once per bar → quantizer set to A-minor with only degrees A, C, G enabled → MIDI to MicroFreak. The quantizer is the fence; without it this is Procedure F.
  • Code path: Prand([0,3,7], inf) in SuperCollider clocked per bar, MIDI out to the MicroFreak.
  • Analog path: roll a die per bar — 1–2 = A1, 3–4 = C2, 5–6 = G1 — and play the logged sequence in by hand.
  • Whichever path: the bass plays only on steps 1, 7, 11. Step choice is decided; note choice is rolled.
Run it three times — the logged rolls
Best-of-three for the 8-bar bass seed (step order: bar.1 / bar.7 / bar.11)
Run 1A·A·G / C·A·G / A·C·C / G·A·A — lands on root too often, feels safe. Reject.
Run 2G·C·A / A·G·C / C·C·A / G·A·G — strong C2 tension mid-phrase, never fully resolves. Keep.
Run 3C·C·G / C·A·C / A·G·C / C·C·A — too much C2, reads as a different key centre. Reject.
  • Log all three runs verbatim in b6003_rolllog.txt, mark Run 2 as kept, and note why. The rejected runs are part of the score — they prove you curated.
Record the kept run
  • Arm MF BASS, monitor Auto. Record 4 minutes of Run 2 looping. Hands off the knobs — this is the static raw material; movement comes later from automation.
  • If using the live VCV/code path, the engine keeps re-rolling as it runs — that's fine, but you've committed to Run 2's character; let only the per-bar micro-variation stay live. Save the patch as LAB CHANCE BASS.
Decision check
  • Is the kept run one of several you'd have been happy with — or did you keep the first thing that played? Best-of-three is the law.
  • Does the bass leave the downbeats to the kick? Steps 1, 7, 11 should never bury the four-on-the-floor.
  • Remove the chance note-picker and play a single fixed note — is the part still recognisably this track? If yes, the fence is doing its job.
Step 05

Build the fixed drone and the chance texture — Procedure C

The drone (decided)
  • Init a fresh MicroFreak patch. Paraphonic on, hold A1 + E2, long attack and release. Record a 4-minute hands-off take into MF DRONE. EQ HP at 120 Hz → reverb send. The drone is the fixed room the chance lives inside — never rolled.
Author the two texture samples
  • Resample two short MicroFreak fragments into the Digitakt, both in A-minor, both matched in level and decay. Load them into slots the texture trig can reach. Vet each one solo against the drone — if either fails, it doesn't go in the field.
Hand the choice to a coin — Procedure C, logged
  • Digitakt way: conditional sample lock — pattern A locks slot A on the texture trig; copy to pattern B, lock slot B; alternate with a per-cycle die roll (odd = A, even = B).
  • VCV way: a guaranteed gate into a Bernoulli Gate at 0.5, outputs triggering sampler voices A and B. Rhythm fixed, timbre rolled.
  • Logged 16-cycle run: A B A A B A B B A A B A A B A B — 9×A, 7×B, with a B-run around cycle 7 that gave the section unplanned movement. Kept on the first run because every cycle was usable; logged anyway.
Decision check
  • Are both texture samples ones you'd keep on their own? If one is weaker, the coin will surface it half the time — fix the field, not the odds.
  • Does the texture sit under the drone, never fronting it? Target around -20 dB, as in the studio reference.
Step 06

Pre-build six section variants — the charts for Procedure A

Compose the field — six 16-bar variants
  • This is the composition work, the equivalent of Cage's charts or Mozart's 176 bars. Build six 16-bar variants of the body, all sharing the fixed kick, drone, and key. They differ only in things chance is already touching: bass density, whether the perc condition is active, which texture is favoured.
The six charts — each one keepable on its own
V1Root-heavy bass, perc off, texture A favoured — the anchor
V2C2 lift in the bass, perc 35 %, textures even
V3Longest gaps, sparsest bass, perc off — the breath
V4G1 emphasis, perc off, texture B favoured
V5Full bass field, perc condition active, hat retrig every 8 bars
V6C2/G1 alternating, perc 1:2, texture coin at 0.5
  • Render each variant to its own audio clip in Ableton so a die roll can drop them on the timeline directly. These are your numbered bars.
Roll the order — Procedure A, logged
Body order, eight 16-bar slots, die per slot (no consecutive repeats)
Slot 1 → 4V4
Slot 2 → 4 → re-roll → 1V1
Slot 3 → 6V6
Slot 4 → 2V2
Slot 5 → 5V5 (sits at the peak — energy from the chart, not from new elements)
Slot 6 → 1V1
Slot 7 → 3V3 (the strip-back breath)
Slot 8 → 6V6
  • Log the full sequence and the one re-roll. Note the happy accident: V5 landing on slot 5 put the densest chart exactly at the peak — chance handed you the structure, inside a fence where any order would have worked.
Decision check
  • Is every one of the six variants a section you'd keep? If V3 is boring solo, it's not a valid chart entry — fix it before rolling.
  • Does the rolled order still respect the inherited form — intro before body, single peak, outro? The die orders the body; it never moves the peak.
  • Did you roll the order more than once and choose? Best-of-three applies here too — you kept this sequence over two others (logged).
Step 07

Assemble the timeline and place the fixed frame

Lay the decided frame first
  • Add locators: INTRO (0:00), BODY (1:00), PEAK (4:30), OUTRO (6:30), END (8:00). These positions are from the constitution — not rolled.
  • Intro: drone + texture only, nothing rhythmic, for the first minute. Outro: drone carries the exit, kick out around 7:00, slow master fade.
Drop the rolled body in the logged order
  • From bar 32 onward, place the eight variant clips in the Procedure A order: V4, V1, V6, V2, V5, V1, V3, V6. Cut on kick downbeats, 4-beat crossfades at every seam.
  • The recorded chance bass (Run 2) and the 35 % perc are already printed into those clips — the chance is baked, the order is rolled, the frame is fixed.
Decision check
  • Play the seams: does each variant boundary land on a downbeat without a click or lurch? Re-cut any that don't.
  • Does the PEAK section get its energy purely from V5's density and the automation, with nothing new entering? That's the thesis of the genre and the test of the build.
Step 08

Draw the automation — the human hand returns

Movement is decided, not rolled
  • Chance shaped the notes, the perc, the texture, and the order. The macro arc — the slow open and close — is yours, drawn by hand. Chance never automates the build's emotional curve.
  • Auto Filter (low-pass, 24 dB) on MF BASS: cutoff ~600 Hz at 0:00, climbing to ~3.5 kHz at the 4:30 peak, back to ~800 Hz by 6:30. No ramp shorter than 8 bars; S-curves, not straight lines.
  • Drone volume and reverb send bloom at the peak, recede after. Echo feedback briefly self-regenerates at 5:00. All decided gestures.
Decision check
  • If you can hear the automation move, it's too fast — same rule as always.
  • Does the peak arrive because things opened (your automation) over a chart that happened to be dense (chance)? Two layers of authorship, neither pretending to be the other.
Step 09

Curate — the second composition pass

Listen to the whole run as one object
  • Play the full eight minutes start to finish, twice, before changing anything. Curation is listening, not tweaking.
  • Mark any moment where chance produced something you'd not sign your name to. There should be very few, because the fences were tight — but if one exists, that's a fence failure, not a roll failure. Tighten the boundary and re-record that variant.
Decide what to keep, with the log open
  • Read b6003_rolllog.txt alongside the audio. For each kept run, confirm you can still defend the choice. For each rejected run, confirm you'd still reject it.
  • Mix pass as in the studio reference: kick owns 40–80 Hz, bass 80–180 Hz, drone HP at 120 Hz, texture HP at 220 Hz, master peaks ≤ -3 dBFS, no limiter.
  • Export b6003_chance_v1.wav, 48 kHz / 24-bit, normalize off. Collect All and Save. The track now exists as three recoverable layers plus a roll log that is its score.
Decision check
  • Could you reproduce this exact track from the constitution plus the roll log? If the log is complete, yes — that's the proof the chance was authored, not arbitrary.
  • Did curation mean choosing between good runs, or rescuing bad ones? Only the first means the fences were right.
  • Remove all four chance hand-offs and play the constitution as a static track — is it coherent? It must be. Chance decorated structure; it never was the structure.
What you just proved

The track's form was decided by chance — which note, which perc, which texture, which order — yet every second is one you authored, because you built every fence the dice rolled inside. That is Cage's architecture: authorship relocated from "which" to "which field of."
