Metering · 04 — Setup

Build a measurement channel

A meter you have to go find is a meter you won't use. This page builds a permanent measurement chain that lives on your master from the start of every project — Spectrum, a loudness meter, a correlation meter, a tuner, and a mono switch — plus the one rule about where meters go so they read the truth.

Ableton Live 12 Suite

  • Ships Spectrum, Tuner, Utility, and channel meters
  • Master meter shows peak and (in Suite) integrated LUFS
  • The whole measurement bench lives here

MOTU M4

  • Hardware input meters and a clip LED per channel
  • Your first meter — before the signal even reaches Live
  • Sets recording level so nothing clips at the door

Elektron Digitakt II

  • The loop source you will measure end to end
  • Its main-out level is set by reading the M4 meters
  • Drums give peak meters a real transient to read

Arturia MicroFreak

  • The drone you tune against Live's Tuner
  • A sustained source for reading RMS and spectrum
  • Mono into the M4 — width is created, not recorded

Beyerdynamic DT 990 Pro

  • Open-back, honest about harshness you read on Spectrum
  • Watch its own 7–10 kHz bump — it is the headphone, not the mix
  • Exaggerates width; trust the correlation needle over it

The measurement rack

  • Spectrum + loudness + correlation + Utility, last on the master
  • A Tuner on a spare track for pitch work
  • Saved into the default template — there from project one

Why meter placement is the whole game

A meter reads whatever sits before it. Put Spectrum before a limiter and it shows you a balance you no longer have. Put the correlation meter before a widener and it never sees the width. The rule is simple: measurement devices go last, so they read exactly what leaves the system.

The master measurement chain

Mix bus → Spectrum → Loudness meter → Correlation → Utility (mono switch) → out
DeviceWhat it readsWhere it goes

UtilityThe mono switch — Width 100 % → 0 %Last in the chain, your ground-truth mono fold
One chain, read-only, never moved

Nothing in this chain shapes the sound except the final Utility, and that only when you toggle the mono test. The meters are passive. If a reading ever looks wrong, the first question is always "is a level or width device sitting after my meters?" — move them back to the end and re-read.

The first meter: the M4 inputs

MicroFreak Out → TS → M4 In 1 + Digitakt II Main L/R → 2 × TRS → M4 In 3/4 → USB-C → Ableton
Set record level by reading the hardware meter — before Live ever sees it
  • Play the loudest part of your loop. Watch the M4's input meter for that channel: the green / amber ladder should ride healthily and the red clip LED must never light.
  • If the clip LED flickers, turn the M4 input gain down — not the level in Live. Clipping at the converter is permanent; no fader in Live can undo it.
  • Aim for peaks that sit a comfortable way below the top of the M4's ladder, leaving headroom. A clean input is the only honest starting point for every meter downstream.

  • Keep monitoring volume fixed. Loudness judgements only mean something if your listening level is constant. Mark a position on the M4 monitor knob and return to it for every reading.
  • Sample rate matters for true peak. Work at 48 kHz. A true-peak meter needs the file's real rate to predict inter-sample overshoot correctly.

Save it into the template

Build this chain once, then File → Save Live Set as Default so every new project opens with the meters already on the master. A measurement bench that's already there is one you'll actually use.
