Lab Note B5-001
Delay · 05 — Delay recipes

Four delays, four jobs

Concrete starting points for the throws you will use in the build. Each recipe is a complete Echo setting plus the gesture that plays it. Dial them, then mutate — the numbers are a floor to stand on, not a sound to copy.

Recipe 1 — the dub throw

Recipe 01

Spike the send, let it trail for bars

The Echo setting (Return A)
Long sync'd ping-pong
Sync / timeOn, 3/16
Feedback60 % — six to eight audible repeats
StereoPing-Pong
Feedback HP / LP250 Hz / 4 kHz
Dry/Wet100 %
The gesture
  • Keep the send at -inf for the whole bar, then spike it to 0 dB for a single snare, stab, or word, then drop it back to -inf.
  • Only that one hit enters the delay line. With 60 % feedback it trails away for bars while the dry part carries on clean — the signature dub move.
  • Automate the send, not the feedback. Feedback stays set; the throw is purely an on/off of the path.
On the rig
  • Ableton: draw send automation on the source track — flat at -inf, a one-step spike to 0 dB on the chosen trig, back to -inf. Or MIDI-map the send to a fader and perform it.
  • Digitakt II: parameter-lock the delay send on a single trig. Everything else sends nothing. The throw is baked into the pattern, repeatable forever.
  • Compact mixer: if the delay is on an aux send, ride the aux knob by hand — the most tactile version of the same idea.

Recipe 2 — the slap

Recipe 02

A single short repeat that thickens

The Echo setting (Return B)
Free-time mono slap
Sync / timeOff, 110 ms (try 80–130 ms)
Feedback12 % — one clear repeat, maybe a ghost of a second
StereoMono, panned a touch off-centre opposite the dry source
Feedback HP300 Hz
Dry/Wet100 %
What it is for
  • A slap is felt, not heard as an echo — it fattens a stab or vocal and pushes it slightly back in the room.
  • Because it is short and mono, it is the mono-safe width source: pan the dry left and the slap right and the part feels wide without ping-pong that collapses.
  • Keep feedback low. A slap with high feedback stops being a slap and becomes a flutter — a different, busier effect.

Recipe 3 — the ping-pong wash

Recipe 03

Wide repeats that build a space

The Echo setting
Long ping-pong, used as ambience
Sync / timeOn, dotted 1/8 (363 ms at 124 BPM)
Feedback50 %
StereoPing-Pong, Width ~70 % (not full — leaves a centre that survives mono)
Feedback HP / LP300 Hz / 3 kHz — dark, washy repeats
ModulationSmall — a slow wobble blurs the repeats into a cloud
How to use it without making mud
  • Feed it sparse, sustained material — a melodica-style line, a chord stab — not the kick or busy percussion.
  • The dotted-eighth makes the repeats stagger against the beat, so a held send creates a rolling, self-weaving texture that never lands square.
  • Pull the Width to ~70 % so the centre stays populated; a 100 % ping-pong is a hole in mono.

Recipe 4 — runaway feedback, filtered

Recipe 04

Push feedback to the edge of self-oscillation

The Echo setting
The transition / riser delay
Sync / timeOn, 3/16
Feedback — resting55 %
Feedback — peak (automated)85–95 % for 8–16 bars, then back to 55 %
Feedback HP / LP250 Hz / 4 kHz — the filter is what keeps the runaway from screaming
Saturation in loopA touch of Echo's drive — repeats compress and dirty as they pile
The move
  • This is the only time you automate feedback rather than the send. Ramp it up over 8–16 bars and the existing repeats regenerate into a swelling wash — a riser made entirely of echo, no new element added.
  • The in-loop low-pass is essential: without it, the runaway turns into a bright howl. With it, the swell stays warm and darkens as it intensifies.
  • Pull feedback back down before it fully self-oscillates, on a bar line, into a stripped section — the drop lands because the echo suddenly stops feeding itself.

Choosing per source

Which recipe for which part
Snare / rim / clapRecipe 1 — throw it into the long tail on chosen bars
Synth stab / chordRecipe 2 slap to fatten, or Recipe 3 wash when it should bloom
Lead / melodica-style lineRecipe 3 — the delay becomes a second voice answering the phrase
Whole-track transitionsRecipe 4 — feedback swell into a drop, sparingly, once or twice in the arrangement
Kick / sub bassNone. Keep them dry — the feedback HP exists precisely to keep them out of the echoes
Palette check
  • Does each recipe filter its feedback so the tail darkens? Flat feedback is a bug, not a sound.
  • Are you throwing (send automation) for Recipes 1–3, and only automating feedback for Recipe 4?
  • Is anything low-frequency being sent to a delay? It should not be — solo the return and listen for kick or sub leaking in.
  • If you removed every dry hit and left only the returns, would the recipes still imply the groove? In a dub track, they should.
