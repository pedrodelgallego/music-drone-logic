Lab Note B5-001
Delay · 04 — Setup

The delay rig

Two ways to build the architecture: Ableton's Echo on return tracks (the studio default for this course), and a compact hardware mixer with an aux send (the dub-engineer way, where the send knob is literally in your hand). This page wires both, and shows how the Digitakt II's own delay slots in.

The signal path you are building

Dry parts stay dry. A copy of each part is sent to a shared echo return, which runs 100 % wet, filters its own feedback, and returns to the master. The send is where you perform; the return is where you shape the tail.

Dry source → send A → Return A: Echo (100% wet) → HP/LP feedback → Return fader → Master

Option A — Ableton Echo on a return (course default)

Setup A

Build two delay return tracks

Create the returns
  • ⌘ ⌥ T twice for return tracks. Rename with ⌘ R: A — DUB ECHO and B — SLAP.
  • A third return for reverb is useful (see the block-5 reverb note), but the two delays are the spine of this track.
  • Drop an Echo on each. Both run Dry/Wet 100 % — the send is the mix, always.

Return B — the tight slap
Echo on Return B
SyncOff — free time, 90–130 ms (a slap, not a rhythm)
Stereo modeMono, panned slightly off-centre

  • Return B is mono-safe by design — it is the width insurance for when Return A's ping-pong is the only thing carrying stereo.
Print discipline — the return is a bus you can commit
  • Because the echo lives on a return, you can record it as audio: route the return's output to a new audio track, ⌘ ⇧ R, and capture a pass of pure repeats.
  • A printed tail can be reversed, re-pitched, time-stretched, or chopped — moves you cannot make on a live return. Use this when the delay becomes a texture in its own right (page 06, Step 08).

Option B — compact mixer with an aux send (the dub way)

Setup B

Aux send to an outboard or plugin delay

The hardware version of the same idea
  • On a small mixer (or the Digitakt II thought of as a mixer), each channel has an aux/FX send knob. That knob is the throw — turning it up sends a copy of that channel into the delay.
  • The delay's output returns to two spare channels (or the mixer's FX return). The return channel's fader is the return level; the aux knob is the send level. Same two-knob distinction as page 03, now physical.
  • This is exactly the King Tubby model — the engineer plays the aux knobs live, and the performance is the mix.
Wiring it through the MOTU M4
If using outboard delay alongside the M4
Mixer aux out→ delay input (pedal, rack unit, or a hardware echo)
Delay output→ M4 spare line inputs, recorded to a dedicated ECHO RETURN track
Send controlThe mixer's aux knob, ridden by hand — printed as audio in real time
CaveatOne clock only. If the delay is sync'd hardware, feed it MIDI clock from Ableton; otherwise run it free-time as a slap.
  • For this course, Option A is simpler and fully recallable. Use Option B when you specifically want the tactile, performed throw — and you will hear the difference, because a hand-ridden send breathes in a way drawn automation rarely does.

One throw, one place

Don't send the same hit into both the Digitakt's delay and Ableton's Echo at once — you get two overlapping tails at different times and the rhythm smears. Decide per sound which delay owns it.

Gain & headroom for delay

Where level lives in a delay-heavy mix
Dry tracksPeaks around -12 to -6 dBFS — same as any take; leave room for the wet path to add up
Send levelPer track, set by ear during the throw — flat at -inf when not throwing
Return A faderStart -10 dB; this is presence, not length
Return B faderStart -12 dB
Master peak≤ -3 dBFS — high feedback can bloom; watch the meter through the loudest throw
  • High feedback is a headroom risk. A throw at 65 % feedback regenerates — its peak arrives a second after the hit, not on it. Test the loudest throw and leave 3 dB of master headroom for the bloom.
  • Filter the feedback, not just the return. An EQ after the Echo filters the sound once. The HP/LP inside Echo's feedback path filters every pass — that is what makes the tail darken and die instead of piling up. This is the whole reason dub delays sound the way they do.
Setup check
  • Both Echo returns at Dry/Wet 100 %? If either is below 100 %, you wired a send like an insert.
  • Is the in-loop HP/LP engaged on Return A — not just an EQ after it?
  • Flip Return A to mono for ten seconds: does Return B keep the width alive? If both collapse, you have no mono insurance.
  • Is exactly one delay owning each sound — no hit feeding two delay lines at once?
