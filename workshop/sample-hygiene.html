Sample Library & Release Hygiene

How to keep your sample library organised, how to know whether a sample is safe to release, and what counts as original material in a portfolio.

The stance

Every sample in a release candidate needs a clear answer to two questions: where did this come from, and can I release it? If you can't answer both, the track is not ready to release — even if it sounds finished.

Sample legality and clearance for released work

A sample is any recording you did not make yourself. Using one in a released track may require permission from the copyright holder — even if you have heavily processed it.

Original recordingsYou made the recording yourself (field recording, direct instrument capture, printed synthesis). No clearance needed. These are always safe.
Sample packs — royalty-freePacks labelled "royalty-free" or "no clearance needed" let you use the sounds in released work without payment or permission. Read the licence anyway — some restrict commercial use.
Sample packs — all rights reservedSome packs are sold for personal/production use only. Using them in a released track may violate the licence. Check before you release.
Recorded music (other artists)Recording a loop, riff, or any portion of another artist's released track and using it in your own requires a sample clearance — written permission from the rights holder. Without it, release is a legal risk regardless of how much you process the sample.
Field recordings of peopleRecordings of identifiable voices in public or private spaces may require consent in some jurisdictions. When in doubt, record your own voice or use ambience where no individual is recognisable.
  • Processing does not clear a sample. Pitching, reversing, time-stretching, or heavily filtering a copyrighted recording does not remove the copyright. The source material still needs clearance.
  • Creative Commons licences vary. CC0 and CC-BY are generally safe for commercial use. CC-NC (Non-Commercial) means you cannot use the sample in a paid release. Always check the exact licence code.

Library organisation

A disorganised library slows down every session. You spend time searching instead of making. The goal is to find any sample in under 30 seconds.

Folder structure

Keep all samples in one root folder. A simple structure that scales:

  • Samples/
    • field-recordings/ — your own location recordings
    • synthesis-prints/ — hardware and software synth prints by date
    • packs/[Pack Name]/ — third-party sample packs, one folder per pack
    • vocals/ — voice recordings
    • foley/ — objects, surfaces, mechanical sounds
    • archive/ — old or unsorted samples you are not actively using

Live Browser: Places, labels, and collections

In Live's Browser, the Places section lets you add any folder as a shortcut. Add your Samples/ root so you never navigate from scratch.

Colored labelsRight-click any folder or file in the Browser to assign a colour. Use colour consistently: e.g. green = cleared for release, orange = check licence, red = personal use only.
CollectionsLive's Collections (the coloured dots in the bottom-left of the Browser) let you tag any item across folders. Use one collection for "current project," another for "favourite textures," another for "needs clearance check."
SearchLive's Browser search (the magnifier) searches by file name within the selected Place. This is why file naming matters — see below.

Naming and metadata

Name files so they describe what they are without you needing to play them. A good name has: source / character / key or pitch / date.

Examples:

  • field-rain-heavy-roof-2025-02-14.wav
  • mf-pad-slow-atk-Aminor-2025-03-01.wav
  • dtii-kick-punchy-120bpm-2025-03-05.wav
  • pack-spitfire-labs-felt-piano-C4.wav

Keep names lowercase with hyphens, no spaces. This makes them safe in any operating system and easy to read in a file list.

Collect All and Save

When a project uses samples from multiple locations across your drive, those samples can move or be deleted. Collect All and Save (File → Collect All and Save) copies every sample the project references into the project's own Samples/ folder. After this, the project is self-contained.

Do this for every project you consider finished. The collected samples inside the project folder are the definitive set for that track — do not move or rename them.

Sample-rate consistency

Your project runs at one sample rate (most likely 48 kHz). When you drop a sample at a different rate (e.g. 44.1 kHz) into a Live Set, Live converts it in real time. This is fine for most work, but it uses CPU and can introduce subtle artefacts.

  • Check the sample rate of any downloaded pack: right-click the file and view its properties, or look at the file info in Live's Browser detail pane.
  • If you record field audio on a portable recorder, set it to 48 kHz before you start. This matches your Live project and avoids any conversion.
  • When exporting prints from Live, always export at your project sample rate (48 kHz), not at the file's original rate.

Destructive vs non-destructive edits

A destructive edit permanently changes the audio file on disk. A non-destructive edit changes how the file is played back, without touching the file itself.

Non-destructive (preferred)Clip envelope, warp markers, transposition, gain, start/end points in Live's clip view. These changes only exist inside the project — the original file is untouched. You can always return to the original.
Destructive (be careful)Editing in an external audio editor (like Audacity), normalising a file to disk, or using Live's "Consolidate" function. The file on disk changes. Keep a copy of the original before doing this.

The rule: never destructively edit your only copy of a sample. Always keep the original, even if you think you won't need it.

What counts as original material in a portfolio

When you present work in a portfolio, "original" means you made every sound yourself — or used third-party sounds with a licence that permits that use.

Fully originalAll sounds are your own recordings or synthesised by you (hardware or software). No third-party samples. Strongest position for a portfolio.
Original with licensed samplesYour arrangement, processing, and composition are original. Third-party samples are royalty-free and licensed for this use. Acceptable — be ready to name the packs used.
Cover / remix with clearanceYou have written permission to use the source material. Acceptable, but note it clearly in the portfolio as a remix or cover, not original composition.
Uncleared samplesDo not include in a public portfolio or release. Risk of takedown and legal exposure.

Proof gate

What to prove

For every release candidate: trace every sample to its source, confirm the licence, document the processing chain, and confirm a printed audio file exists. Complete this table for the track before you release it.

  • Every sample in the project is listed in the table above.
  • Every source is identified. No "unknown" entries remain.
  • Every licence is confirmed as compatible with the intended release (commercial / non-commercial).

  • Collect All and Save has been run. All samples are inside the project folder.
  • No sample in the project is labelled "unconfirmed" or "check needed."
