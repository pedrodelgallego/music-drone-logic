Replicate: Florian Hecker — “Acid in the Style of David Tudor”

Copy the reference's working mechanism, not its style. A piece that does not process a recording at all: the sound is the audible output of a synthesis process run at the edge of control, where parameter motion is the music. Build a short original study with Live 12, Digitakt II, MicroFreak, and the MOTU M4, then A/B it against the reference at matched loudness.

The point

Before you start
  • Live 12 set saved as REP-FLORIAN-HECKER-ACID.
  • Digitakt II connected by USB for sync and by main outs to MOTU M4 inputs 3/4.
  • MicroFreak output connected to MOTU M4 input 1 when the lesson needs a synth source.
  • MOTU M4 headphones at the Lab 001 mark; Live tracks set to Monitor Off while recording hardware.
  • Reference on a Live audio track named REF, with Utility inserted for loudness matching.
  • What you will make: a 60-90 second original study where the processor chain changes the identity of one source.
  • What you will turn off: extra layers, decorative effects, and level jumps that hide what the processor is doing.
  • What you will prove: the same transformation behavior — the synthesis process is the composition — remains audible after you change the source.
  • What counts as success: remaining differences are source choices or rig limits, not a missing mechanism.
Reference pass · 10 minutes

Listen before you build

Listen to one clear section of Florian Hecker — “Acid in the Style of David Tudor”. Hecker builds the entire piece out of computer synthesis driven hard — no field recording, no instrument, no sample is being treated. The “acid” is not a 303 line but a reference to restless, self-modulating parameter motion, and the “Tudor” is the idea that a complex electronic process, once set running, composes itself. The transformation is the source. Write one short answer for each question before you touch the gear.

  1. What is the smallest repeated behavior you can point to?
    Answer

    Here it is a synthesis engine left running while one or two parameters are driven continuously, so the timbre never settles and the motion of the controls is the only “melody” there is. Name the behavior, not the mood.

  2. Which device or path could create that behavior in your studio?
    Answer

    In this rig: MicroFreak oscillator + Matrix modulation, or an all-software Operator/Wavetable patch in Live, with parameter automation as the score. If the mechanism seems to live everywhere, you have not isolated it yet.

  3. What should stay boring so the mechanism remains audible?
    Answer

    Keep the note material — hold one pitch, or a tiny two-note cell — so every change the ear hears is a change in the synthesis process, not a change in melody. A good replication drill keeps the supporting material plain so the real mechanism can be heard clearly.

  4. What would prove you copied the system rather than the style?
    Answer

    your study should sound like a process being driven, not a sound being played; if it only works when you copy Hecker's exact timbres, you copied the surface and not the self-generating mechanism.

Steps

  • Button map for the steps below: use these controls whenever the lesson names the device.
  • Live 12: ⌘ T creates an audio track, ⇧⌘ T creates a MIDI track, ⌘ R renames the selected track, ⌘ , opens Settings, ⌘ M enters MIDI Map mode, and ⌘ J consolidates the selected clip.
  • Digitakt II: [TRK] + [TRIG 1-16] selects the active track. Press [RECORD] for grid recording, press numbered [TRIG] keys to place or remove hits, and press [PLAY] to hear the pattern.
  • Digitakt II sound and timing: press [SRC] for sample/source settings, [FLTR] for filter, [AMP] for envelope/level, [LFO] for modulation, [FX] for sends, and [TEMPO] for tempo and swing.
  • Digitakt II sync: press [SETTINGS] for MIDI Config → Sync. Use [YES] and the arrow keys to turn clock and transport receive on or off.
  • MicroFreak: use the Preset encoder to choose a plain starting sound, Type to choose the oscillator, Cutoff and Resonance for the filter, Matrix for modulation routing, and Save only after the study behaves correctly.
  • MOTU M4: MicroFreak usually enters input 1. Digitakt II main L/R enter inputs 3/4. Inputs 3/4 have no front-panel gain knobs, so set their record level from the Digitakt II Main Volume.
  • Sound target: use simple source material so the transformation behavior is exposed.
  • Recording target: peaks around -12 dBFS in Live before any processing.
  • Copy target: a solo flip between REF and your print makes the shared behavior obvious.
  • Variation rule: change one variable per pass and write it down.
Phase 1 · ~20 min

Put the reference in Live and write the hypothesis

Do this
  1. Drop the reference onto a track named REF.
  2. Loop the clearest 30-60 seconds.
  3. Add Utility to REF and lower it until it is comfortable.
  4. Write: “The mechanism is ___.” Use the listening tag if you need a first guess: the synthesis process is the composition.
  5. Write the one thing you will not copy: sound, melody, sample, room, mix polish, or performance style.
  6. Create tracks named SOURCE, SYSTEM, and PRINT.
Expected result

You have a looped reference, three working tracks, and one written hypothesis before any programming starts.

Phase 2 · ~35 min

Build the transformation chain in Live

Build the smallest system that can prove the hypothesis. Keep the material plain. The point is to make the mechanism — the synthesis process is the composition — audible, not to finish a track.

Do this
  1. Choose one plain synth voice — a single saw or a simple two-operator FM patch — and hold one pitch. Do not write a melody.
  2. Plug MicroFreak output into input 1. Set the M4 gain so Live peaks around -12 dBFS.
  3. Press ⌘ T, name the track SOURCE, set input Ext. In 1, Monitor Off, and record 30-60 seconds.
  4. Duplicate the track and name it CHAIN. Mute SOURCE so it stays as the clean proof.
  5. Add the transformation that generates the piece: route fast modulation into timbre. On the MicroFreak, send the LFO and an envelope through the Matrix into Type, Wave, and Cutoff at audio-adjacent rates; in Live, automate Operator's ratios and feedback or Wavetable's position so the spectrum is always in motion.
  6. Treat the parameter automation as the composition. Draw the motion you want — a slow climb, a sudden break, a self-oscillating wobble — and let that curve, not any note, decide the form.
  7. Put Utility before and after the chain. Pre-Utility sets drive into the chain; post-Utility brings the level back down so loudness does not fake the result.
  8. Print SOURCE and CHAIN at matched loudness so the processor, not volume, wins the comparison.
Expected result

A working study where the synthesis process is the composition appears without needing extra decoration.

Phase 3 · ~15 min

Route the studio so the print is clean

Make the signal path boring and dependable. If the recording path is confusing, you will misdiagnose the mechanism.

MicroFreak · input 1 + Digitakt II · inputs 3/4 → MOTU M4 → USB → Live 12 · Monitor Off
Do this
  1. Plug MicroFreak into input 1 if the study uses it. Set the input gain while watching Live.
  2. Plug Digitakt II left/right into inputs 3/4.
  3. Set hardware tracks to Monitor Off. This prevents doubled monitoring.
  4. Arm one record track at a time and check peaks. Aim for -12 dBFS, not “as loud as possible.”
  5. If inputs 3/4 are too quiet or too hot, adjust Digitakt II Main Volume and write the position down.
  6. Record 60-90 seconds of the system.
  7. Press ⌘ J to consolidate the best section into a clean clip named PRINT-A.
Phase 4 · ~20 min

Make the system map and the A/B proof

Draw the mechanism, then test the print against the reference. The map is not decoration; it is how you find what failed.

Do this
  1. Put REF and PRINT-A on adjacent tracks.
  2. Use Utility to match loudness before judging.
  3. Listen to 10 seconds of REF, then 10 seconds of PRINT-A.
  4. Write one sentence: “The first thing that gives mine away is ___.”
  5. Fix only that one thing and record PRINT-B.
  6. Keep both prints. The difference between A and B is the lesson.

Phase 5 · ~15 min

Diagnose one miss and write the rule

Do this
  1. Use the page's current rig for this phase and keep REF from Florian Hecker — “Acid in the Style of David Tudor” one solo away.
  2. Carry out the diagnosis with the existing devices, changing only what proves extreme computer synthesis where the transformation process itself, not a source recording, generates the entire piece.
  3. Record a short proof pass before comparing; do not judge from a live monitor path only.
  4. Write the one setting, edit, or gesture that made the biggest difference.

Final check

Play the reference and your best print at matched loudness, then answer these in the logbook:

  • What mechanism did you copy?
    Answer

    Name the mechanism, not the genre: here, extreme computer synthesis where the transformation process itself, not a source recording, generates the entire piece.

  • Where does that mechanism live in your setup?
    Answer

    Point to the device or path: MicroFreak oscillator + Matrix modulation, or an all-software Operator/Wavetable patch in Live, with parameter automation as the score.

  • What did you deliberately not copy?
    Answer

    The answer should be material: melody, sample, exact sound, mix polish, or artist style. The workshop copies systems, not identity.

  • What rule carries into the related lab?
    Answer

    Write it as a usable instruction. Example: “If the synthesis process is the composition disappears when the source changes, I copied the sound, not the system.”

  • Logbook — the miss list, three lines: Matched: ___ · Missed: ___ (best guess why) · Change next: ___
  • System diff sentence: “I thought the system worked like ___; after building it, I discovered it worked like ___.”
