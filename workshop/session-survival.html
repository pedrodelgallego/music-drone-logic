Session Survival & Export Discipline

How to keep a Live project safe over the long term, export it correctly, and know exactly what you'll have left if everything else disappears.

The stance

A project that only lives on your computer is not finished — it is at risk. The goal of every session is to leave with a state you could reopen on a different machine, on a different day, and still make sound.

Templates and the Default Set

A Default Set is the project Live opens when you press File → New Live Set. If you leave it empty, every session starts from scratch. Instead, build a template that matches your rig.

  • Route your standard inputs (M4 channels 1–2, 3–4) and outputs (1–2 to monitors) in the template so you never rebuild them.
  • Add your most-used instrument and effect devices pre-loaded and named.
  • Save it with File → Save Live Set as Default.
  • Keep a second copy named _template-YYYY-MM-DD.als in a _templates/ folder. Update it every few months.

Backups

Live saves automatic backups in a hidden Backup folder inside your project folder. They are time-stamped snapshots. You can raise the count in Preferences → File & Folder → Maximum Backup Files. Set it to at least 10.

Automatic backups are not a substitute for a real backup strategy. Use at least two of these three:

Cloud sync
iCloud Drive, Dropbox, or similar. Automatic, continuous, survives hardware failure. Make sure the sync folder is your projects root.
External drive
A physical hard drive kept off-site or at least unplugged. Run a copy after every major session. Works even when your internet is down.

Collect All and Save

Collect All and Save (File → Collect All and Save) copies every sample the project references into the project's own Samples/ folder. After running it, the project is self-contained — it no longer depends on samples sitting in other locations on your drive.

When to run it

Run Collect All and Save before you: archive a finished track, move a project to another drive, share a project with anyone, or back up to an external drive. If you skip this step, missing-sample warnings are almost guaranteed.

Versioning — dated save-as

Live's autosave keeps short-term history. For longer-term milestones, use a dated save-as. The pattern is:

ProjectName-YYYY-MM-DD.als

Save a new version: after a major structural change, before you try something risky, when you reach a state you want to be able to return to. Keep all versions in the same project folder. Do not delete old versions until the track is fully released and archived.

Freeze, flatten, and print decisions

These three operations all turn something editable into audio, but they mean different things.

FreezeTemporarily renders a track to audio so it plays without CPU load. The original devices and clips are still there — just hidden. You can unfreeze at any time. Use it on CPU-heavy tracks while you work on other parts.
FlattenPermanently replaces a frozen track's clips with audio files. The devices and MIDI are gone from that track. You cannot undo this after saving. Use it when you are certain a sound is finished.
Print to audioRecord the track's output into a new audio track while the original runs. You keep both. The safest option: the MIDI/device track survives, and you have an audio capture for mixing or export.
  • Always print rather than flatten unless you are consciously reducing project size and you are certain about the sound.
  • When you print hardware (MicroFreak, Digitakt II), that print is irreplaceable — the hardware state will change. Name the audio file with the patch name and date: MF-Dronepatch-2025-03-01.wav.

Exporting stems and a master

A stem is a single element of the mix exported as its own audio file (kick, bass, synth pad, etc.). A master is the full stereo mixdown. Both are exported with File → Export Audio / Video.

Settings that matter

Sample rateMatch your project rate. For most work: 48 kHz. Do not upsample on export — if the project ran at 48 kHz, export at 48 kHz.
Bit depthExport at 24-bit for stems and archive masters. 16-bit is only for final delivery (CD, some streaming). Higher bit depth = more headroom for later processing.
NormalizeLeave this off. Normalizing changes the level relationship between your stems and makes them harder to use together later. Set your mix level by ear, not by automatic normalization.
True-peak ceilingIf your mastering limiter has a true-peak setting, use −1.0 dBTP as a ceiling for streaming delivery. For stems and working files, leave the limiter off and let the mix breathe.
DitheringTurn on dithering only when reducing bit depth (e.g. 24-bit project → 16-bit delivery). Use POW-r 1 for most music. Do not dither when staying at 24-bit.

Stem export procedure

1
Solo each track group in turn (or use track activation). Set the export range with the Loop Brace to cover the full arrangement.
2
Export each stem with the same settings (48 kHz / 24-bit / normalize off). Name files consistently: TrackName-StemRole-YYYY-MM-DD.wav.
3
Export the stereo master last, with the full mix active. Keep the master export in the same folder as the stems.
4
Import all stems back into a blank Live Set and check that they sum to the same sound as the master. If they don't, something was muted or routed wrong.

Recovery from missing samples and plugins

When you open a project and see missing-file warnings, do this in order:

1
In Live's missing-file browser, click Search. Live will look for files with the same name across your drives. Accept any matches that are obviously correct.
2
If search fails, manually locate the file using Browse for File. Check your external drive and cloud sync folder first.
3
If the sample is gone entirely, check whether you have a printed audio version of the track it was part of. The print is a valid substitute — replace the missing clip with the print.
4
For missing plugins: install or reinstall the plugin, then reopen the project. If the plugin is no longer available, you must replace it with something equivalent, or use the printed audio.

The print map

A print map is a plain-text document listing every track in the project and its status. It answers: if I opened this project on a clean machine, what would I have? Keep it next to the project file.

Track nameTypeStatusNotes
KickAudioPrinted — self-containedDigitakt II pattern 1, printed 2025-03-01
Bass droneMIDI + deviceDevice state only — requires pluginOperator patch, no print yet
Pad layerAudioPrinted — self-containedMicroFreak Dronepatch-A, printed 2025-03-01
FX return AMIDI + deviceDevice state only — requires pluginLive's Echo, built-in, always available

Four possible states for any track:

Audio — self-containedAn audio clip with its sample collected inside the project folder. Safe on any machine.
MIDI + deviceRequires the plugin or Live instrument to be installed. The sound is not saved as audio — it is re-synthesised every time.
Hardware stateDepends on a hardware device (MicroFreak, Digitakt II) being connected and set to a specific patch. The patch may have changed since the session.
GoneMissing sample or uninstalled plugin with no print. The track is silent until you find a replacement.

Proof gate

What to prove

This gate certifies that your project is genuinely portable and your export is trustworthy. Run it on every track before you call it finished.

  • Run File → Collect All and Save. No warnings appear.
  • Copy the project folder to a different location (or a second drive). Open it from that new location.
  • No missing-sample warnings on open. Every track plays correctly.
  • Export stems (48 kHz / 24-bit / normalize off). File names follow the naming convention.
  • Export the stereo master with the same settings and a −1.0 dBTP ceiling if mastered.
  • Import stems into a blank Set and confirm they sum correctly against the master.
  • Write or update the print map. Every track has a status: audio self-contained / MIDI + device / hardware state / gone.
  • Create a dated save-as version: ProjectName-YYYY-MM-DD-final.als.
