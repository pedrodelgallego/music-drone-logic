Shared workshop references

Written once, used by every lab. These pages are stubs in the scaffold; the assessment frame below is already final.

The two specs

Every track lab and capstone opens with both, before any gear is touched. Year 1: provided, you verify. Year 3: you author them blank.

System Spec — composition-facing

  • Inputs — what raw material enters the system
  • Transforms — what changes it, in what order
  • Feedback / variation — what keeps it alive
  • Constraints — what prevents clutter
  • Human role — performed, decided, or curated

Engineering Spec — path-facing

  • Signal path · gain plan · clock and sync
  • Latency · stereo and phase · noise budget
  • Print strategy — what is committed, when, why
  • Failure mode — what will go wrong, how to diagnose it
  • Safety — feedback, headphone level, field, live

The fault ritual

Predict the symptom→ Create the fault→ Observe→ Diagnose the location→ Fix→ Write the rule

One controlled breakage per track lab, bench lab, and capstone — never punitive. Fault classes: gain, monitoring, clock, phase, routing, latency, noise, state, performance.

The lab report

Filed for every track lab, étude, and capstone
  • Rendered piece — 24-bit WAV, −3 dBFS
  • Routing diagram — drawn before patching, corrected after; the diff is the lesson
  • Recording plan — what gets printed, when, at what stage
  • Patch notes — the settings that mattered
  • Session notes
  • Reflection answers, citing the lab’s listening assignment
  • One removal decision — what was cut, and why
  • Fault entry — symptom, cause, fix, prevention rule
  • Print map — what exists as audio, MIDI, automation, patch state, memory
  • System diff — “I thought it worked like X; I discovered it worked like Y.”

The seven layers — the graduate frame

Used at capstone reviews, L8, and the final defense: describe any system through material, signal, control, time, decision, human, and archive layers.
