CC-Mapping Lab

A MIDI CC is a numbered control message your controller sends when you move a knob or push a pad. This lab walks you through mapping one control end-to-end — to a Live Macro, to the MicroFreak, and to the Digitakt II — and proves that the movement records and recalls correctly.

What is a MIDI CC?

CC stands for Control Change. It is a MIDI message with two numbers: a CC number (0–127, which identifies the control) and a value (also 0–127, which says how far you turned the knob). Your controller sends these messages over a MIDI channel (1–16). The receiving device uses the CC number to know which of its parameters to move.

The three targets

You can send a CC message to three different places in your rig. The mapping method is different for each.

Live MacroMapped inside Live using MIDI Map mode (⌘ M). Live intercepts the CC and applies it to the Macro. The movement is recorded as automation inside Live's session or arrangement. Everything stays in the computer.
MicroFreak parameterThe MicroFreak responds to MIDI CC on whatever channel you set in its MIDI settings. The controller sends the CC directly — Live can pass it through, or you can connect the controller to the MicroFreak without going through Live at all. Check the MicroFreak's MIDI CC table in its manual to find which CC number controls which parameter.
Digitakt II performance moveThe Digitakt II can receive MIDI CC to control its track levels, mutes, and some performance parameters. Send CC on the Digitakt's auto-channel or a specific track channel. Check the Digitakt II MIDI CC table for the correct numbers.

Mapping to a Live Macro

1
Create a Rack if you do not have one. Select a device in Live. Right-click → Group into Instrument Rack or Group into Audio Effect Rack. Eight Macro knobs appear.
2
Assign a device parameter to Macro 1. Right-click any knob inside the Rack → Map to Macro 1. Set a min and max range in the Macro map panel so the full knob throw maps to a useful range.
3
Enter MIDI Map mode. Press ⌘ M. The interface turns blue.
4
Click Macro 1, then move your controller knob. Live shows the CC number and channel it detected. Press ⌘ M again to exit. The Macro is now mapped.
5
Test it. Move the physical knob. The Macro should respond in real time.

Mapping to the MicroFreak via MIDI

The MicroFreak receives MIDI from an external source. Your controller sends CC directly to it — Live can be in the middle as a pass-through, or you can use a separate MIDI cable or USB connection.

1
Find the CC number for the parameter you want. Open the MicroFreak manual and look at the MIDI CC implementation chart. For example, Cutoff is typically CC 74.
2
Set your controller knob to send that CC number on the MicroFreak's MIDI channel. In your controller's editor software or the Launchpad's Custom mode, assign the CC number and channel to the knob. The default MIDI channel on the MicroFreak is channel 1 — confirm this in MicroFreak → Utility → MIDI Channel.
3
Route MIDI from your controller to the MicroFreak. If going through Live, enable the controller's MIDI output in Live → Preferences → Link / Tempo / MIDI. Create a MIDI track, set its output to the MicroFreak, and arm it. Or connect the controller directly to the MicroFreak's MIDI in (bypassing Live).
4
Test it. Move the knob. The MicroFreak's parameter should change. You should hear a change in the sound.

Mapping to the Digitakt II

The Digitakt II responds to MIDI CC on its auto-channel (which controls the currently selected track) or on individual track channels. Check the Digitakt II manual's MIDI CC implementation table for the exact CC numbers for track level, mute, and performance parameters.

1
Find the CC number. The Digitakt II manual lists which CC numbers control which parameters. CC 7 is usually volume; performance moves like filter cutoff are on other numbers per track.
2
Set your controller to send that CC on the Digitakt's auto-channel. The auto-channel follows whichever track is selected. This is the most flexible option for live control.
3
Route MIDI to the Digitakt. Connect the Digitakt's MIDI in to your interface. In Live, enable the Digitakt as a MIDI output in Preferences. On a MIDI track, set the output to the Digitakt and the channel to the auto-channel.
4
Test it. Select a Digitakt track. Move the controller knob. You should hear or see the Digitakt respond.

Mapping in Live vs sending CC to hardware — the key difference

  • Mapping in Live means Live intercepts the CC and applies it to a software parameter. The movement can be recorded as automation in your Live Set. When you recall the Set, the automation replays automatically. Nothing is sent to hardware unless you explicitly route it.
  • Sending CC to hardware means the CC message travels out of your computer to a physical device (MicroFreak, Digitakt). The hardware responds in real time, but Live does not automatically record what you did. To record hardware CC movements, you need a MIDI track set to record the input, or you record the audio output of the hardware instead.

Proving recordability and recall

A mapping is only useful in a live set if it records and recalls. Here is how to prove it.

1
Map a control to a Live Macro (see above). This is the easiest target to prove recall, because automation lives inside the Live Set file.
2
Arm automation recording. Click the Automation Arm button (red circle with A) in Live's toolbar. Make sure you are recording into Arrangement view or a clip envelope.
3
Play and move the control. Start playback. Sweep the knob several times over a minute or so. Stop.
4
Check the automation lane. Open the Arrangement view. Press A to show automation. Select your Macro in the automation lane selector. You should see a line that follows your knob movements.
5
Save the Set. Close Live. Reopen it.
6
Play back the arrangement. The Macro should move by itself, following the automation you recorded. If it does, the mapping recalls correctly.

Proof gate

You pass this lab when you can answer yes to all four checks.

Key facts to write down

For each mapping you make: the CC number, the MIDI channel, the physical control, and the parameter it moves. You will forget these. A single table in your notes prevents confusion during a performance.
