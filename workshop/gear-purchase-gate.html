Gear purchase gate

A short test every new piece of gear must pass before it joins your rig. It exists to stop the most expensive habit in music-making: buying a box to feel productive instead of to solve a named problem. Run it before you buy, and again before you build a lesson around the gear.

The stance

Your current rig is already complete. New gear is not an upgrade by default — it is a new failure surface (gain, clock, noise, latency, recall, state) that must earn its place by solving a problem the rig genuinely can't.

The five questions

A purchase passes only if it answers all five. Any "no" or hand-wave is a fail.

1 · What problem does it solve?It must name a current bottleneck in a real project — not an imagined future style. "I want to make X and the rig can't" beats "it would be cool to own."
2 · Which path changes?It must clearly update one of: the audio path, control path, clock path, monitoring path, or print path. If you can't draw the change on your Studio Signal Map, you don't understand the purchase.
3 · What does it replace or simplify?If it only adds options without removing or simplifying anything, it fails. More choices is not more music.
4 · What proof will certify it?You commit to a 30-minute integration test that ends in a recorded print: the gear in the chain, levels checked, one finished result.
5 · What new failure does it create?Name the new fault it introduces — a gain standard, a second clock, a noise source, added latency, a recall problem, a new state to remember. If you can't name it, you can't manage it.
The 30-day no-buy test

Still want it after answering all five? Wait 30 days. Spend that month pushing the current rig at the same problem. Most "needs" evaporate; the real ones get sharper and pass the gate with a concrete answer to question 1.

Buy order, if the gate passes

When a purchase is justified, this is the order that adds the most new capability per dollar and per unit of new complexity. Each has a free software substitute you already own — listed so you can practise first.
