Digitakt Project Hygiene & Performance Patterns

The Digitakt II holds a lot of state — projects, patterns, kits, and a sample pool — and it is easy to lose track of what is saved where. This page covers the basic structure, how to keep it organised, and how to use mutes and fills to design a live set section by section.

Why hygiene matters

If you do not know what is saved in which pattern, you will reach for something during a performance and find a different sound or an empty slot. Good project hygiene means every pattern has a purpose, every sample has a home, and every state can be recovered after a power cycle.

Project structure — what lives where

The Digitakt II stores its state in a hierarchy. Understanding this hierarchy is the foundation of good hygiene.

ProjectThe top-level container. One project holds up to 16 banks of 16 patterns (256 patterns total), its own sample pool (the samples loaded into RAM for that project), and all kits. When you switch projects, everything changes — patterns, kits, and loaded samples.
Pattern bankA group of 16 patterns, labelled A–P. Each bank is a section of your live set or a stage of development. A common split: Bank A = intro patterns, Bank B = main loops, Bank C = variations and breakdowns.
PatternOne sequence of up to 64 steps across eight tracks. Each pattern has its own tempo, length, and swing. A pattern also stores which kit it is linked to (or you can use pattern-level parameter locks on top of the kit).
KitA snapshot of all eight tracks' sound settings — which sample is loaded on each track, and all the synthesis and effect parameters for each. A kit can be shared across multiple patterns or unique to one. Changing a kit affects every pattern that uses it, unless you save a new kit first.
Sample poolThe samples the project has loaded into RAM from the +Drive (internal storage). The pool can hold up to 128 MB of audio. Loading a sample into the pool does not use it — you still assign it to a track manually.
+DriveThe Digitakt's internal storage — up to 1 GB of samples and 128 project slots. Samples on the +Drive are not in RAM until you load them into a project's sample pool. Projects are backed up to and restored from the +Drive.

Kit and sample-pool discipline

The most common Digitakt mistake is editing a kit without saving it first — your changes overwrite the shared kit and change every pattern that uses it. Follow these rules.

  • Save a kit before you edit it. Press FUNC + SAVE/RELOAD → Save Kit. Give it a name that describes the sound (e.g. KICK-A-TIGHT or PAD-OPEN). Now your edits go into the new kit and the old one is untouched.
  • Give each pattern its own kit if the sounds differ. Patterns that share a kit will always change together. If you want a variation to sound different, it needs its own kit.
  • Name your samples when you load them. The +Drive shows file names. Short, descriptive names (e.g. CLAP-DRY, SUB-808) make it possible to navigate quickly under pressure.
  • Keep the sample pool trim. Only load what the project actually uses. A bloated pool makes scrolling slow and makes it harder to see what matters.

Mutes and fills as performance design

Mutes silence one or more tracks without stopping the pattern. They are the primary performance tool on the Digitakt — hold a track button and press MUTE, or set up mute states in advance. Design your patterns so muting one or two tracks creates a distinct section feel.

Fills (also called Fill mode or Trig Conditions with fill) play a set of steps only when you hold the YES button. Use fills for one-bar variations that drop in when you choose — a snare roll, a cymbal crash, a melodic accent. They are pre-composed but performer-triggered.

Mute for dramaDrop the kick and bass together for a breakdown. Bring them back for impact. Design every pattern so muting the low tracks gives you a breakdown without changing patterns.
Mute for textureRemove one melodic track to thin the texture. Restore it gradually or all at once. The same pattern sounds different depending on which tracks are active.
Fill for punctuationProgram a fill on a percussion track that only fires when you hold YES. Hold it for the last bar of a section as a signal to the listener that something is about to change.
Fill for transitionUse a fill that plays a short melody or noise burst. Trigger it as you switch patterns so the transition has a musical event rather than an abrupt cut.

Backups and exports

The Digitakt can lose power. Kits can be overwritten by accident. Back up regularly.

1
Save the project to +Drive. Press FUNC + SAVE/RELOAD → Save Project. The project writes to the +Drive slot. Do this after every session.
2
Back up to a computer using Elektron Transfer. Open Elektron Transfer on your Mac. Connect the Digitakt via USB. Use the Backup function to export the full project as a .dtprj file. Store this on your computer after each significant session.
3
Export samples if you created new ones. If you resampled audio into the Digitakt during a session, export those audio files via Elektron Transfer → Sample Manager before the next session. Resampled audio lives in the +Drive but is not automatically backed up to your computer.

Moving from one pattern to a live-set section

A pattern is one loop. A live-set section is a stretch of time — intro, build, peak, breakdown, recovery. Here is how to map patterns to sections.

One bank per sectionBank A = intro patterns (sparse, minimal). Bank B = main body (full groove). Bank C = variations (different mute states, subtle changes). Bank D = breakdown and recovery. This makes it easy to navigate with the bank buttons during a performance.
Pattern chainingThe Digitakt can chain patterns to play in sequence automatically. Set the pattern length and chain them in Pattern → Pattern Settings → Pattern Chain. Use chaining for a section that needs several loops in fixed order — for example, an intro that runs 4 bars of A1, then 8 bars of A2.
Manual switchingFor live performance, switch patterns manually by pressing the pattern buttons. The Digitakt queues the next pattern and switches at the end of the current loop. No gap, no click.

Documenting Digitakt state in the print map

Your print map (the document that describes your live set) must include the Digitakt state. Write down what lives where so you can reconstruct the set after a gap or recover from a crash.

What to documentWhere to write it
Project name and +Drive slot numberTop of your print map
Which bank maps to which section of the setSection map in your print map
Which kit each pattern usesPer-pattern notes or a table
Which samples are in the pool and which tracks use themPer-track column in your print map
Mute states for each section (which tracks are muted)Per-section column or colour-coded grid
Fill assignments (which track, what it plays)Per-pattern notes
  • Project memory (what is saved in the project file): all patterns, all kits, all parameter locks, all mute states, the sample pool list, and the project tempo and settings.
  • Kit memory (what is saved in a kit): the sound settings for all eight tracks — sample assignment, filter, amplitude, effects, and MIDI settings.
  • Pattern memory (what is in a pattern): the step sequence, trig conditions, parameter locks, pattern length, and which kit the pattern is linked to.
  • Not saved automatically: changes to a kit are in RAM until you explicitly save the kit. If you power off without saving, those edits are lost.

Recovery test

After completing the proof gate, power cycle the Digitakt. Reload the project from +Drive. Confirm all five patterns play correctly and all kit sounds are as expected. If anything is missing, you have found a gap in your save workflow — fix it now, not during a performance.
