Replicate: The Upsetters — Blackboard Jungle Dub

Copy the reference's working mechanism, not its style. Build a short original study with Live 12, Digitakt II, MicroFreak, and the MOTU M4, then A/B it against the reference at matched loudness.

The point

Before you start
  • Live 12 set saved as REP-THE-UPSETTERS-BLACKBOARD-JUNGLE-.
  • Digitakt II connected by USB for sync and by main outs to MOTU M4 inputs 3/4.
  • MicroFreak output connected to MOTU M4 input 1 when the lesson needs a synth source.
  • MOTU M4 headphones at the Lab 001 mark; Live tracks set to Monitor Off while recording hardware.
  • Reference on a Live audio track named REF, with Utility inserted for loudness matching.
  • What you will make: a 60-90 second original study where the routing and space are the main instrument.
  • What you will turn off: uncontrolled wet paths, doubled monitoring, and stereo width that fails in mono.
  • What you will prove: the same spatial behavior remains audible after you change the dry material.
  • What counts as success: remaining differences are source choices or rig limits, not a missing mechanism.
Reference pass · 10 minutes

Listen before you build

Listen to one clear section of The Upsetters — Blackboard Jungle Dub. Lee Perry and Tubby together — the loop pushed until it stops being an effect and becomes an unstable weather system over the band; the wildest desk behavior in the ten. (Perry's Super Ape is homed in Block 1; this is the sibling study.) Write one short answer for each question before you touch the gear.

  1. What is the smallest repeated behavior you can point to?
    Answer

    Do not name the mood first. Name the behavior: a timing lean, a processor changing with level, a return path blooming, a rule producing events, a hand riding a control, or a constraint shaping the piece.

  2. Which device or path could create that behavior in your studio?
    Answer

    Pick one place: Digitakt II pattern data, MicroFreak source tone, Live device chain, MOTU M4 input path, or a performance move. If the mechanism lives everywhere, you have not isolated it yet.

  3. What should stay boring so the mechanism remains audible?
    Answer

    Keep the supporting material plain. A good replication drill often has boring notes, boring drums, or boring source audio so the real mechanism can be heard clearly.

  4. What would prove you copied the system rather than the style?
    Answer

    Your study should use different material while producing the same kind of behavior. If it only works when you copy the sound, melody, or sample, the mechanism is still missing.

Steps

  • Button map for the steps below: use these controls whenever the lesson names the device.
  • Live 12: ⌘ T creates an audio track, ⇧⌘ T creates a MIDI track, ⌘ R renames the selected track, ⌘ , opens Settings, ⌘ M enters MIDI Map mode, and ⌘ J consolidates the selected clip.
  • Digitakt II: [TRK] + [TRIG 1-16] selects the active track. Press [RECORD] for grid recording, press numbered [TRIG] keys to place or remove hits, and press [PLAY] to hear the pattern.
  • Digitakt II sound and timing: press [SRC] for sample/source settings, [FLTR] for filter, [AMP] for envelope/level, [LFO] for modulation, [FX] for sends, and [TEMPO] for tempo and swing.
  • Digitakt II sync: press [SETTINGS] for MIDI Config → Sync. Use [YES] and the arrow keys to turn clock and transport receive on or off.
  • MicroFreak: use the Preset encoder to choose a plain starting sound, Type to choose the oscillator, Cutoff and Resonance for the filter, Matrix for modulation routing, and Save only after the study behaves correctly.
  • MOTU M4: MicroFreak usually enters input 1. Digitakt II main L/R enter inputs 3/4. Inputs 3/4 have no front-panel gain knobs, so set their record level from the Digitakt II Main Volume.
  • Sound target: use simple source material so the space behavior is exposed.
  • Recording target: peaks around -12 dBFS in Live before any processing.
  • Copy target: a solo flip between REF and your print makes the shared behavior obvious.
  • Variation rule: change one variable per pass and write it down.
Your rig
Phase 1 · ~20 min

Put the reference in Live and write the hypothesis

Do this
  1. Drop the reference onto a track named REF.
  2. Loop the clearest 30-60 seconds.
  3. Add Utility to REF and lower it until it is comfortable.
  4. Write: “The mechanism is ___.” Use the listening tag if you need a first guess: the loop as a haunted room.
  5. Write the one thing you will not copy: sound, melody, sample, room, mix polish, or performance style.
  6. Create tracks named SOURCE, SYSTEM, and PRINT.
  1. Work in Live 12 only for this phase: keep REF looped from The Upsetters — Blackboard Jungle Dub and use Echo, Hybrid Reverb, Auto Filter, sends, and resampling.
  2. Build the phase goal — Put the reference in Live and write the hypothesis — as the smallest Live set that proves a spatial path: delay, reverb, feedback, width, mono, or subtractive routing.
  3. Keep every source on named tracks, then use Utility to match the level against REF before judging it.
  4. Record or resample the result to a fresh audio track named PRINT; do not leave the proof only as devices and MIDI.
  5. A/B ten seconds of PRINT against REF and change only the setting that most affects the loop as a haunted room.
  1. Use Octatrack MKII for this phase: translate Put the reference in Live and write the hypothesis into Flex/Static machines, slices, recorder buffers, scenes, the crossfader, and main-out recording.
  2. Program the pattern, source, or control path so it demonstrates a spatial path: delay, reverb, feedback, width, mono, or subtractive routing, not just the surface of The Upsetters — Blackboard Jungle Dub.
  3. Keep one clear performance control for the main change in this phase; name it before recording so the move is repeatable.
  4. Record the device into Live or a recorder as a full pass, then compare it with REF at matched level.
  5. If the target box suggests an easier shortcut, use it only when the same mechanism stays audible.
Expected result

You have a looped reference, three working tracks, and one written hypothesis before any programming starts.

Phase 2 · ~35 min

Build the space path and keep it controllable

Build the smallest system that can prove the hypothesis. Keep the material plain. The point is to make the mechanism audible, not to finish a track.

Do this
  1. Create a dry two-voice pattern: one pulse and one sparse accent. Keep sends at 0 for now.
  2. [TRK] + [TRIG 1] selects track 1. Load the pulse sound.
  3. [TRK] + [TRIG 2] selects track 2. Load the accent sound.
  4. Add one long tone only if the reference needs a sustained object for space to act on.
  5. Create tracks named DRY, SPACE, and PRINT. Set Digitakt II input to Ext. In 3/4.
  6. Add one spatial device only: Echo, Hybrid Reverb, Delay, Corpus, or a feedback return. Keep the dry path audible beside it.
  7. Map one control to the wet amount or feedback amount. Do not move three space controls at once.
  8. Check mono with Utility. If the mechanism disappears in mono, fix the routing before recording.

Phase 3 · ~15 min

Route the studio so the print is clean

Make the signal path boring and dependable. If the recording path is confusing, you will misdiagnose the mechanism.

MicroFreak · input 1 + Digitakt II · inputs 3/4 → MOTU M4 → USB → Live 12 · Monitor Off
Do this
  1. Plug MicroFreak into input 1 if the study uses it. Set the input gain while watching Live.
  2. Plug Digitakt II left/right into inputs 3/4.
  3. Set hardware tracks to Monitor Off. This prevents doubled monitoring.
  4. Arm one record track at a time and check peaks. Aim for -12 dBFS, not “as loud as possible.”
  5. If inputs 3/4 are too quiet or too hot, adjust Digitakt II Main Volume and write the position down.
  6. Record 60-90 seconds of the system.
  7. Press ⌘ J to consolidate the best section into a clean clip named PRINT-A.

Phase 4 · ~20 min

Make the system map and the A/B proof

Draw the mechanism, then test the print against the reference. The map is not decoration; it is how you find what failed.

Do this
  1. Put REF and PRINT-A on adjacent tracks.
  2. Use Utility to match loudness before judging.
  3. Listen to 10 seconds of REF, then 10 seconds of PRINT-A.
  4. Write one sentence: “The first thing that gives mine away is ___.”
  5. Fix only that one thing and record PRINT-B.
  6. Keep both prints. The difference between A and B is the lesson.

Final check

Play the reference and your best print at matched loudness, then answer these in the logbook:

  • What mechanism did you copy?
    Answer

    Name the mechanism, not the genre: timing rule, processor path, space path, decision rule, performance move, or authoring constraint.

  • Where does that mechanism live in your setup?
    Answer

    Point to the device or path: Digitakt II pattern, MicroFreak patch, Live device chain, MOTU M4 routing, or your hands during the print.

  • What did you deliberately not copy?
    Answer

    The answer should be material: melody, sample, exact sound, mix polish, or artist style. The workshop copies systems, not identity.

  • What rule carries into the related lab?
    Answer

    Write it as a usable instruction. Example: “If the behavior disappears when the source changes, I copied the sound, not the system.”

  • Logbook — the miss list, three lines: Matched: ___ · Missed: ___ (best guess why) · Change next: ___
  • System diff sentence: “I thought the system worked like ___; after building it, I discovered it worked like ___.”
  • Carry into the lab: open lab-020.html and reuse the corrected rule, not the reference sound.
