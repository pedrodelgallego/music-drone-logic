The Live Set

Thirty minutes, performed and recorded. Not thirty minutes of material played back — thirty minutes operated: a system you designed, certified on lab 031’s rig, rehearsed to fail, and then ran in real time from the first sound to the last. The gate is not whether the set was good; it is whether, when the system became unstable, you understood it well enough to recover musically.

Start here
  • Confirm the prerequisites: labs 027–031 filed, programme L7 complete with its stolen principle named, R7’s miss list in hand, and lab 031’s rig certification current — an uncertified rig does not perform this set.
  • Write both specs first. The Engineering Spec for a performed work is the live routing diagram plus the panic path — drawn before any material is chosen.
  • Schedule the performance date in the logbook before you feel ready. A live set without a date is a studio project.
  • Plan the failure rehearsal as a session of its own, before the set — it is documented evidence, not a warm-up.
What a gate is

A block does not end when its labs are done; it ends when you can run the operating loop without a recipe. C6 runs the loop at S4 and in real time: the brief is constraints only, the specs are authored blank, and the fault — for the first time in three years — is not injected into a session you can pause. The audience can be one person, a room, or a camera; what cannot be substituted is the clock running forward only.

1 — Brief & constraints

Design, rehearse, and perform a 30-minute live set of fully original material, in one continuous performance, recorded. The set is a system: committed elements and performed elements declared in advance, transitions designed, every section escapable. The stolen principle you named during L7 is now due: it must appear structurally in the set — in its form, its rig, or its rules — and you must point to it in the defense.

  • Support level: C6 is S4 — both specs authored from the brief alone. The System Spec declares inputs, transforms, variation, constraints, and the human role minute by minute; the Engineering Spec is led by its live fields: signal path, gain plan, clock, monitoring, panic path, recording strategy, failure mode.
  • Material: fully original — your Year 3 pieces (027–030 output) are the natural repertoire, reworked for performance rather than played back. Steal systems, never material. Nothing from any replication session appears in the set.
  • The rig: the configuration certified in lab 031, on your own hardware — Live, Digitakt, MicroFreak, the mixer, and whatever the Block 7 gear (Octatrack-class sampler, ride controller) added. Any change to the rig after certification re-opens the certification.
  • Constraint menu: the full menu is included here: track ceilings, committed/performed ratios, no-laptop-screen clauses, and single-fallback rules. The four constraints below are the fixed core.
  1. Thirty minutes, one take. The set runs continuously from first sound to last. Stopping is a panic-path event, documented in the fault report — not a reset.
  2. Performed, not played back. At least one declared element is ridden live in every section; a set that could run identically with your hands in your pockets fails the brief by definition.
  3. Recorded, redundantly. The recording strategy names a primary print path and one fallback (board feed + room, or two independent printers). A set that exists only in memory did not happen.
  4. Every section escapable. The panic path — mute, limiter, bypass, stop, fallback loop — is wired, physically reachable, and rehearsed for every section of the set before performance day.
Decision check

For each minute of the planned set, ask: if the worst plausible fault hit right here, what would I do, and how long would the music be undefended? If any answer is “I don’t know” or “longer than a phrase”, the design isn’t finished — redesign the section or extend the fallback layer before rehearsing it.

2 — Bench delta

Gate capstones renew the bench certification — and this gate extends it to the live rig. The bench delta for C6 has three parts: the live routing diagram, the failure rehearsal, and the signal-map re-version.

  1. Draw the full live routing diagram from memory — every source, every gain stage, the clock topology, the monitoring path, the record path, and the panic path drawn as part of the signal flow, not as a footnote. Then compare against the rig and correct; the diff is the lesson.
  2. Run the failure rehearsal, documented, before the set. From your live signal-flow diagram, list the five most plausible performance faults — Block 7’s territory: a runaway level, a dead element, a clock dropout, a wrong scene, a monitoring loss. For each: predict the symptom, trigger it in rehearsal, walk the panic path, time the recovery, write the rule learned. The rehearsal log — five faults, five timed recoveries — files with the bundle. This is lab 031’s checklist run in anger, and it is required evidence, not advice.
  3. Re-version the Studio Signal Map. The map re-versions at every capstone; the live rig configuration becomes its own sheet of the map, marked with what differs from the studio configuration and why.
  4. Re-run lab 031’s pre-set checklist — cables, power, clock, gain, record-arm — on performance day, out loud, before the first sound.
Rig certification — renewed at C6
  • Draw the full live path, panic path included, from memory
  • Identify every gain stage between every source and both the room and the recording
  • State the clock master and what happens to every follower if it drops
  • State what you are monitoring at any moment of the set — and what the recording is taking, which is not the same thing
  • Demonstrate every escape route with the set running, in under two seconds each
  • If any item fails: return to lab 031 before scheduling the performance. The gate waits — there is no penalty for the detour, only for performing on a rig you cannot draw.

3 — Report & synthesis

C6 files the full report bundle with the performed-work additions below. The capstone’s fault is not injected after the fact: whatever actually became unstable during the thirty minutes — and something will — is the fault entry, diagnosed from the recording in the post-performance fault report.

The C6 bundle
  • The set recording — the full thirty minutes, 24-bit WAV, primary print path, with the fallback recording archived beside it
  • Live routing diagram — drawn before, corrected after; panic path drawn in the signal flow
  • Both specs, authored blank, dated before the build began
  • The failure-rehearsal log — five faults, predicted symptoms, timed recoveries, rules learned, dated before the performance
  • The set plan — committed vs performed, minute by minute, with escape routes per section
  • The post-performance fault report — every instability of the actual set: symptom, moment (timestamped against the recording), cause, what you did, what the panic path contributed, prevention rule. Written within a day of the set, from the recording, not from memory
  • Reflection answers, including the listening comparison — which L7 observation contributed what to the set’s design
  • One removal decision — what was cut from the set and why
  • Print map — what of this set now exists: audio, scenes, patterns, patch state; what is reproducible and what was only ever live
  • System diff sentence — “I thought the set worked like X; after performing it, I discovered it worked like Y.”
  • Minimum pass gate: the set recording · live routing diagram · post-performance fault report · print map · system diff. Missing any one of these means C6 is incomplete regardless of how the set sounded — a musically strong set cannot bypass the engineering spine.
Pass standard — rubric detail
Minimum passThe minimum gate above is complete, the rig certification is renewed, the failure rehearsal is documented and dated before the set, and the defense is answered.
Solid passThe set audibly distinguishes committed from performed material, at least one real instability was recovered musically, and the stolen principle can be located in the set’s structure.
Advanced transferYou can explain how the same set would be re-rigged for an unfamiliar venue — different monitoring, different power, no second take — without losing its structure or its escape routes.
ExcellenceThe set clears a higher bar than “recovered safely” on every front:
  • A recovery is invisible — an instability is turned into material a listener would read as intended, not survived.
  • The stolen principle is load-bearing, not decorative: remove it and the set’s form collapses, and the defense can show why.
  • The thirty minutes hold as one statement — a developed arc with an earned climax and a changed return, not a paced loop.
  • The post-performance fault report teaches something the rig certification did not predict — a fault class worth adding to lab 031’s checklist for next time.

The seven-layer worksheet

Capstones only — one page, seven answers, written about this performance. Each layer is a question you answer in a few honest lines.

The defense

Answered in the logbook — but out loud first, as if walking a promoter’s engineer through the rig before doors. Gate-scaled: short answers, checked against the recording and the diagrams.

  1. Draw the final audio path from memory.
  2. Draw the control path separately.
  3. Identify the most fragile part of the system.
  4. Explain one failure and its fix.
  5. Explain what was printed, what remains live, and what cannot be recovered.
  6. Name the stolen principle and point to where it appears structurally in the set.
  7. Name one thing you removed and why.

Block synthesis

One paragraph in the logbook: what does Block 7 claim, in your own words — and where did your performance disagree with it? The disagreement is not a failure of the block or of you; it is the most useful sentence you will write this cycle.

  • Passing the gate: bundle complete + rig certification renewed + defense answered. Then Block 8 — Synthesis & Voice opens: the final block, where the operator becomes the author.
Gate principle

The set is not the gate; the recovery is. Anyone can perform a system that behaves — the graduate performs the one that doesn’t.
