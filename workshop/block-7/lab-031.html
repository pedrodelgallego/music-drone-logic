The Performance Rig

Certify the live system before C6: routing, gain, clock, monitoring, panic path, recording plan, and failure rehearsal. A live set is not ready when it sounds good; it is ready when the rig can fail predictably and recover safely.

The point

This lab turns the studio into a reliable performance instrument. Every cable, clock, mute, limiter, recorder, and panic action is written and rehearsed before the live set exists.

Certification checklist

Pass these in order
  1. Draw the full live audio path from every source to recorder and monitors.
  2. Draw the control path: MIDI, clock, transport, mappings, mutes, macros, and manual controls.
  3. Label every hardware input: MicroFreak → M4 input 1; Digitakt II L/R → M4 inputs 3/4; any mixer return or live sampler path named separately.
  4. Set the gain plan and record one minute of maximum-density audio. Peaks must stay below -6 dBFS on the print path.
  5. Write the panic path as five gestures: mute, bypass, stop, lower monitor, save recording.
  6. Rehearse three faults: silent source, runaway feedback, and lost clock or loop.
  7. Record a 10-minute technical rehearsal and file the fault log.
Rig depth — certify what you actually run

Certification scales to the rig you bring, not a minimum one. If the set uses more than the FaderFox EC4, the extra surface is certified too: a fader-per-channel mixer means every channel level is its own gain stage and its own panic point — label them, set their unity marks, and rehearse the dub-mute on each. Expression pedals are control inputs with their own failure mode (a pedal stuck at heel or toe is a silent or runaway parameter) — declare what each pedal owns and what its safe rest position is. Outboard spring or tape is a feedback-capable effect on a real send: its return gets a gain stage on the gain plan, and “effect runs away” joins the rehearsed fault list. The rule is unchanged — every control reachable in the set is drawn, labeled, gain-staged, and failure-rehearsed before C6.

3 · Build

Build the lab in five passes. Each pass produces an artifact, not just a better-sounding session: a written rule, a printed comparison, a fault entry, and a corrected system sentence.

Build recipe · The Performance Rig

Make the performance system audible

Do this in order
  1. Map the hands. Limit the rig to six reachable controls. Name every control, its safe range, and what it changes in the form.
  2. Write the panic path. Before sound, write the one-gesture mute, bypass, stop, monitor-lower, and save procedure. Rehearse it silently.
  3. Rehearse without repair. Run the first minute twice without recording. Fix the map, not the take.
  4. Print the take. Record one full take in Live. Do not edit automation after the fact; failed takes are performed again.
  5. Run the fault. Trigger the performance fault safely: wrong mute, too-hot feedback, lost loop, or missed cue. Recover using the written panic path.

Build exit check

The lab can move to diagnosis only when the printed audio proves the written rule. If the rule sounds true only while you explain it, simplify the system and print again.

Deep dive — a future Feedback Lab Note

The self-oscillating systems this rig can host — no-input mixing, outboard feedback, resonant electroacoustic loops — are sketched here only as far as certification needs. A future Feedback Lab Note extends this material into its own discipline: no-input desks as instruments, controlled self-oscillation, stability thresholds, and the safety practice that makes a runaway loop a steerable voice rather than a hazard. Treat what you certify here as the safe entry point to that deeper study.

Bench principle

A performance rig is an instrument only after its failures have been rehearsed.
