MOTU M4 Troubleshooting Appendix

A quick symptom-to-fix reference for the most common M4 problems. Use this when something stops working. If the issue isn't here, check Lab 001 (signal path) and Lab 002 (clock and latency) first.

Before you start diagnosing

Power-cycle the M4 first. Unplug the USB cable, wait 10 seconds, replug. This clears most stuck states. If the problem persists, then work through the table below.

Symptom reference

Symptom Likely cause Fix
No input signal — the input meter in Live shows nothing, even with a source connected and gain up Wrong input selected in Live, or gain too low, or cable fault 1. In Live: Preferences → Audio → Input Config — confirm the correct channel pair is activated (e.g. Ext. In 1/2).
2. Check the M4's front-panel gain knob for the relevant channel — turn it up slowly while watching the M4's signal LED.
3. Swap the cable. Test with a known-working cable from a different source.
4. If using a microphone, confirm the XLR is seated fully and phantom power (+48V) is on if the mic needs it — press the 48V button on the M4 front panel.
Doubled/hollow monitoring sound — a comb-filter effect, as if the sound is phasing against itself Direct monitoring on the M4 and software monitoring in Live are both active at the same time. You are hearing the signal twice, slightly offset in time. Choose one monitoring path, not both.
• Direct monitoring (recommended for low latency): press the DIRECT button on the M4 so its LED is lit. In Live, set the track's Monitor switch to Off.
• Software monitoring only: press DIRECT again so its LED is off. In Live, set the track's Monitor switch to In or Auto.
Never have both active simultaneously.
Clicks, pops, or crackles during playback or recording — not occasional, but regular Buffer size too small, or sample rate mismatch between M4 and Live project 1. Check sample rates match: M4 internal rate (shown on front display or in MOTU Audio Setup) must equal the Live project rate (Preferences → Audio → Sample Rate). Both should be 48 kHz.
2. Increase the buffer size: Preferences → Audio → Buffer Size — try 256 samples first, then 512 if clicks continue. See Lab 002 for the latency trade-off.
3. Quit other audio apps (browsers with video, streaming apps) that may be competing for the audio driver.
High latency — noticeable delay between playing a key/pad and hearing the result through monitors Buffer size too large Lower the buffer size: Preferences → Audio → Buffer Size — try 128 samples for real-time playing, 64 if your CPU can handle it. If clicks appear, raise it back one step. See Lab 002 for the full latency-setting procedure.
Missing or wrong channel configuration — you can see fewer channels than expected, or channels are mapped incorrectly Input/output configuration not enabled in Live's preferences In Live: Preferences → Audio → Input Config and Output Config. Click to activate the channel pairs you need. Channels must be toggled on here before they appear in the track I/O selectors. Check that you haven't accidentally limited the channel count — MOTU Audio Setup (the standalone app) can also restrict available channels.
M4 not appearing in Live's audio device menu — the M4 is plugged in but doesn't show up under Preferences → Audio → Audio Device Driver not loaded, USB power issue, or Live needs a restart 1. Check the M4's front display — if it shows no USB indicator, the connection hasn't been established. Try a different USB port (prefer USB-A directly on the computer, not a hub).
2. Open MOTU Audio Setup (standalone app, installed with the M4 driver). If the M4 doesn't appear here either, the issue is driver-level — reinstall the MOTU driver from motu.com.
3. If MOTU Audio Setup sees the M4 but Live doesn't, quit Live completely and relaunch it.
Ground hum or electrical whine — a low 50/60 Hz hum or high-pitched tone that stays even when the input gain is at zero Ground loop between devices sharing power, or electrical interference from a nearby power supply 1. Plug the M4, computer, and all connected hardware into the same power strip or outlet. Ground loops occur when devices share different circuits or different ground paths.
2. If using balanced TRS or XLR cables, check that they are fully seated. Unbalanced cables (TS, RCA) are more susceptible to hum — replace with balanced equivalents where possible.
3. Move any switching power supplies (phone chargers, LED drivers) away from the M4 and cables. They radiate interference.
4. If the hum only appears on a specific hardware input (e.g. Digitakt II): the issue is likely a ground loop between that device and the M4. Try a DI box (a passive direct-injection box) inline — it breaks the ground loop without affecting the signal.

First five checks

If you are not sure where to start, run this checklist in order. It resolves the majority of M4 problems in under two minutes.

  • Power-cycle the M4. Unplug USB, wait 10 seconds, replug. Check the front display lights up.
  • Sample rate match. M4 rate = Live project rate = 48 kHz. Check both: MOTU Audio Setup and Live Preferences → Audio → Sample Rate.
  • One monitoring path only. Either DIRECT on + Live Monitor Off, or DIRECT off + Live Monitor In. Never both.
  • Channels activated in Live. Preferences → Audio → Input Config / Output Config — confirm the channel pairs you need are toggled on.
  • Buffer size appropriate for the task. Playing live → 128 samples. Mixing/processing → 256 or 512 samples. Clicks → raise it. Latency → lower it.
  • Still stuck after the checklist? Open Lab 001 and retrace the signal path from source to monitor, step by step. Most problems are a single broken link in the chain — the lab's signal-flow diagram makes it easy to find.
  • Clock and sync problems (out-of-time audio, drifting hardware) are covered in Lab 002. If you are running the Digitakt II or MicroFreak via MIDI clock and notice drift, start there.
