Hardware Expansion — Optional & Gated

External & Performance Mixer

A hardware mixer lets you mute, blend, and route sources with your hands while the session plays — and it makes no-input feedback a real instrument. But input count or cueing must actually be a bottleneck first. This page stays locked until the gate passes.

Optional — gated

You do not own this gear yet. The free substitute is Live's mixer and a VCV Rack feedback loop — use them until hands-on sends, physical cueing, or a no-input practice prove themselves as real needs. Buy only after this gear passes the Gear Purchase Gate. Come back to this page then.

When and why

A hardware mixer earns its place when you run out of M4 inputs, when you need to cue a source in headphones without sending it to the main out, or when you want to perform mutes and sends live without touching a mouse. It also unlocks no-input feedback as a standalone instrument — something Live's mixer cannot do safely without extra routing.

Key concepts

Channel gain vs fader

The channel gain (sometimes called "trim") is the first knob a signal hits. Set it so the meter reaches around −12 dBFS on the loudest moments. The fader then controls the level you send to the mix — think of gain as "set it once" and the fader as "ride it live." Never use the fader to correct a gain problem; fix the gain first.

Aux sends and returns

An aux send takes a copy of a channel and routes it somewhere else — a reverb unit, a delay, a monitor speaker. The send level is separate from the channel fader. The processed signal comes back on an aux return (or a spare channel). This lets you share one effects unit across many channels and control the wet amount per source.

Inserts

An insert breaks the channel's signal path: the signal goes out to a processor and comes back before the fader. Use it for compression or EQ that should affect only that channel. Unlike a send, the full signal passes through the processor — there is no dry/wet blend at the mixer level.

Subgroups

A subgroup collects several channels under one fader. Assign all percussion channels to a subgroup and you can ride them together without touching each fader. Useful when you have more sources than hands.

Cue / headphone bus

The cue bus sends signals to the headphone output independently from the main out. You can listen to a channel before it goes live — for checking levels, rehearsing a transition, or monitoring a return without the audience hearing it.

Main out vs record out

The main out drives your monitors (and venue PA). The record out (or a post-fader direct out) feeds the M4 for archiving. Keep them separate so monitor level changes do not affect the recording level.

Performance mutes

The fastest mix move is a mute button. Map sources to mutes before the session so you know exactly what each button does. Practice the order you will hit them — a missed mute during a live set is permanent.

No-input mixer (feedback as instrument)

A no-input mixer technique — developed by Toshimaru Nakamura and explored in Block 7 Lab 029 — routes the mixer's own outputs back into its inputs, creating feedback loops that become an instrument in themselves. The mixer is the instrument; there are no external sources.

The signal chain: send output → channel input, possibly through EQ and aux sends to shape the feedback character. Small fader and EQ changes cause large tonal shifts. The result is unpredictable and changes in real time.

Always prepare a panic path: a dedicated mute or VCA that can cut all output instantly. Set this up before any feedback session and know where it is without looking.

Proof gate

Before using this gear in a session

Route the MicroFreak, Digitakt II, a Live return channel, and one send effect through the mixer. Record a main mix via the record out into the M4. When the print is done, explain — in one sentence per item — what cannot be changed after that print without re-recording. Then demonstrate: perform live mutes, ride at least one send, cue a source in headphones without it appearing in the main out, and execute a panic mute. You are done when the print exists, the explanation is written, and each move was performed cleanly.
