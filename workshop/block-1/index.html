Block 1 — The Bench

Listen, calibrate the studio, then prove it with a piece

Engineering focus

By C1 you can draw the full studio path, identify every gain stage, distinguish input, track, bus, and master clipping, say exactly what you are monitoring — and record the same source cleanly at three levels and defend one.

Required path

Do these in this order
  1. Programme L1 — run the three listening sessions and name one stolen principle
  2. Lab 001 — certify the signal path and gain stages
  3. Lab 002 — certify latency, clock, and monitoring
  4. R1 — rebuild the eight bars from L1 Session 2 and correct the worksheet
  5. Required drills — grid + clock, p-locks + mutes, sends + riding — the three moves Lab 003 assumes
  6. Lab 003 — build the worked hypnotic piece
  7. Lab 004 — build the one-voice étude
  8. Lab 004b — build its beatless twin on the same patch, and amend the gain ledger for sustained material
  9. Capstone C1 — make a second piece from constraints and renew bench certification
  • Replication lessons are optional drills. Use them when a reference exposes a specific weakness in your listening or bench technique — except the three that Lab 003 names as prerequisites, and R1: those are required.

Listening programme L1 — repetition & hypnosis

Session 1 · Records

Four engines of repetition

The same idea — a loop that never bores — built four different ways. The fourth is the limit case: repetition with the rate at zero. Analyze each with the six-question frame, then name what distinguishes the engines.

Basic Channel — “Quadrant Dub I”dub-filtered loop
Plastikman — Consumed (title track)subtractive automation
Donato Dozzy & Nuel — The Aquaplano Sessionsmodulation drift
Coil — Time Machinesthe zero-rate loop
Listen for

What actually changes over eight minutes? What never changes? Which machine or hand is making the change?

Session 2 · Samples

Deconstruction — one track to its sources

Pick one hypnotic track and trace it backwards: kick, hats, chord stab — what raw material, what processing chain, in what order? The sixth question does the heavy lifting: what path had to exist for this record to exist?

A hardware-techno live set, watched for restraint

Watch the hands, not the lights: where are they? What is being ridden live and what is pre-planned? What is being monitored? What is the failure risk, and what is impossible to recover later?

Minimum pass gate

Track labs & capstones — before any quality judgment
  • Audible output exists
  • Routing diagram exists — drawn before patching, corrected after
  • One fault was diagnosed: symptom, cause, fix, prevention rule
  • Print map exists — what is audio, what is state, what is gone
  • System diff sentence: “I thought it worked like X; it worked like Y”
