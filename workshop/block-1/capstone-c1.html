A Second Hypnotic Piece

What a gate is

A block does not end when its labs are done; it ends when you can run the operating loop without scaffolding. C1 is deliberately the same genre as lab 003 — the only new variable is the absence of instructions.

1 — Brief & constraints

Build one original hypnotic techno piece, 6–8 minutes, fully original — steal systems, never material — from the constraint menu below. The stolen principle you named during L1 is now due: it must appear structurally in this piece, and you must point to it in the defense. Not a flavor, not a sound — a structural idea you can locate in the arrangement, the routing, or the rules the piece runs on.

  • Scaffolding: C1 sits at the S1→S2 boundary — the spec no longer arrives finished; you complete it. Concretely: write your own filled System Spec and Engineering Spec before patching anything, using lab 003’s filled specs as the model for granularity. That is the completion exercise. A spec written after the piece is a description, not a spec.

Pick four constraints from the menu and write them at the top of your spec before anything else. The constraints are the brief; everything not forbidden is permitted.

  1. Five tracks maximum. Buses and returns count toward nothing; voices count toward everything.
  2. No hi-hats. Whatever carries the eighth-note pulse, it cannot be a hat.
  3. One synth voice only — drums may come from the Digitakt; melody and texture may not come from anywhere but that single voice.
  4. Exactly one effect in the whole project, used as architecture — not decoration. One delay, or one reverb, or one filter: it must be load-bearing.
  5. Nothing drawn with the mouse. Every automation curve is ridden and recorded — knobs, faders, the MicroFreak’s matrix, the Launchpad. Editing a ridden take is allowed; drawing a curve is not.
  6. No 8-bar stretch repeats exactly. Something — a level, a filter, a probability, a mute — must differ between any two adjacent 8-bar windows.
  7. The kick drops out exactly once, for exactly 8 bars, and its return is the loudest event in the piece.
  8. Tempo outside the comfort zone: below 122 or above 136 BPM.
Decision check

Do your four chosen constraints contradict each other anywhere? One synth voice plus one effect plus no exact repetition is a hard corner — where will variation come from? Finding the contradiction before patching is part of the spec work; finding it during the build means the spec was incomplete.

2 — Bench delta

Gate capstones renew the bench certification. The studio you certified in labs 001–002 has been used, repatched, and partly forgotten since; the gate checks whether the map in your head still matches the room.

  1. Redraw the Studio Signal Map from memory — no peeking at v1, no opening the session. Every source, every gain stage, every monitoring path, as you believe it is.
  2. Compare against Signal Map v1 from lab 001. Mark every discrepancy — a missing gain stage, a wrong monitoring assumption, a path you forgot exists. The diff is the lesson.
  3. File the redrawn map as Signal Map v1.1. The map re-versions at every capstone and every gear arrival; from now on, the current version is whichever one survives a redraw.
  4. Re-run the certification list below. Out loud or on paper, but completely.
Bench certification — renewed at C1
  • Draw the full studio path
  • Identify every gain stage
  • Distinguish input, track, bus, and master clipping
  • State what you are monitoring at any moment — direct, software, pre/post-FX, printed
  • Record the same source at three levels and defend one
  • If any item fails: return to lab 001 or 002 before continuing. The gate waits — there is no penalty for the detour, only for skipping it.

3 — Report & synthesis

C1 files the full report bundle — the same ten items as a track lab, with the capstone additions below. Capstones run one fault like any track lab: choose a gain, monitoring, or latency fault — Block 1’s territory — and run the full ritual.

The full bundle — ten items
  • Rendered piece — 24-bit WAV, peaking around −3 dBFS
  • Routing diagram — drawn before patching, corrected after; the diff is the lesson
  • Recording plan — what gets printed, when, at what stage
  • Patch notes — the settings that mattered
  • Session notes
  • Reflection answers, including the listening comparison — which reference contributed what to the hypothesis
  • One removal decision — what was cut from the finished piece and why
  • Fault entry — predict the symptom, create the fault, diagnose, fix, write the prevention rule
  • Print map — what now exists as audio, MIDI, automation, patch state, or device memory; what is reproducible and what is gone
  • System diff sentence — “I thought the system worked like X; after building it, I discovered it worked like Y.”
  • Minimum pass gate: audible output · routing diagram · fault entry · print map · system diff. Missing any one of these means C1 is incomplete regardless of how the piece sounds — a musically strong piece cannot bypass the engineering spine.
Pass standard
Minimum passThe minimum gate above is complete, bench certification is renewed, and the defense questions are answered.
Solid passThe four constraints audibly shape the finished piece, and the stolen principle can be located in the arrangement, routing, or rules.
Advanced transferYou can explain how the same system would change on another bench without losing the core principle.

The seven-layer worksheet

Capstones only — one page, seven answers, written about this piece. Each layer is a question you answer in a few honest lines.

Seven layers — answered for this piece
What is the sound, at the level of oscillator, sample, or noise?Material
Where does audio travel, change level, clip, widen, sum?Signal
What is MIDI, automation, modulation, gesture?Control
What is clocked, what drifts, what resets?Time
Which choices were rules, and which were taste?Decision
What did your hands actually do, and when?Human
What is printed, named, versioned — and what is unrecoverable?Archive

The defense

Answered in the logbook — but out loud first, as if explaining the piece to a colleague who will check your drawings against the session. Gate-scaled: short answers, no essays.

  1. Draw the final audio path from memory.
  2. Draw the control path separately.
  3. Identify the most fragile part of the system.
  4. Explain one failure and its fix.
  5. Explain what was printed, what remains live, and what cannot be recovered.
  6. Name the stolen principle and point to where it appears structurally in this piece.
  7. Name one thing you removed and why.

Block synthesis

One paragraph in the logbook: what does Block 1 claim, in your own words — and where did your build disagree with it? The disagreement is not a failure of the block or of you; it is the most useful sentence you will write this cycle.

  • Passing the gate: bundle complete + certification renewed + defense answered. Then Block 2 — Sources opens.
