Replicate: Lorenzo Senni — “Persona”

Copy the reference's working mechanism, not its style. Build a short original study with Live 12, Digitakt II, MicroFreak, and the MOTU M4, then A/B it against the reference at matched loudness.

The point

This drill teaches one thing: a timing rule. Here the rule is that a rising trance arpeggio, stripped of its drums, can carry a whole piece as a pointillist rhythmic engine. The reference gives you the clue; the build proves whether you understood it.

Before you start
  • Live 12 set saved as REP-SENNI-PERSONA.
  • Digitakt II connected by USB for sync and by main outs to MOTU M4 inputs 3/4.
  • MicroFreak output connected to MOTU M4 input 1 — this lesson lives on the synth, so the MicroFreak is your main source.
  • MOTU M4 headphones at the Lab 001 mark; Live tracks set to Monitor Off while recording hardware.
  • Reference on a Live audio track named REF, with Utility inserted for loudness matching.
  • What you will make: a 60-90 second original study where a drumless rising arpeggio is the main musical event and the only timekeeper.
  • What you will turn off: kick, snare, hats, and any pad that would let you stop listening to the arpeggio's placement.
  • What you will prove: the sequence still grooves — still feels rhythmic — after you change the patch and the notes.
  • What counts as success: remaining differences are source choices or rig limits, not a smuggled-in drum machine.
Reference pass · 10 minutes

Listen before you build

Listen to one clear section of Lorenzo Senni — “Persona”. Senni calls his method “pointillistic trance”: he takes the supersaw arpeggio — the build-up tool from late-90s rave, the part DJs use to tease the drop — and treats it as the finished music. No kick ever arrives. The euphoria is held open indefinitely, so your ear, trained to wait for a beat, starts hearing the arpeggio's own note placement as the rhythm. The timing rule here is subtraction: remove the drums and the gate pattern of the synth becomes the groove. Write one short answer for each question before you touch the gear.

  1. What is the smallest repeated behavior you can point to?
    Answer

    A short rising note-cell that loops and re-triggers, its rhythm defined by gate length and step placement rather than by any percussion. Name the behavior, not the mood.

  2. Which device or path could create that behavior in your studio?
    Answer

    The MicroFreak running a bright saw/super-saw voice, sequenced from the Digitakt II so you control gate and step placement. The groove is authored in the pattern data, not in a drum track.

  3. What should stay boring so the mechanism remains audible?
    Answer

    The harmony. Keep the chord or scale static and the timbre plain. If you add chord changes or filter drama, you will not be able to tell whether the rhythm of the arpeggio is carrying the piece.

  4. What would prove you copied the system rather than the style?
    Answer

    Swap the supersaw for a plain triangle and change the note pitches: if it still reads as a drumless groove, you copied the placement engine. If it only works with a rave-saw sound, you copied the style.

Steps

  • Button map for the steps below: use these controls whenever the lesson names the device.
  • Live 12: ⌘ T creates an audio track, ⇧⌘ T creates a MIDI track, ⌘ R renames the selected track, ⌘ , opens Settings, ⌘ M enters MIDI Map mode, and ⌘ J consolidates the selected clip.
  • Digitakt II: [TRK] + [TRIG 1-16] selects the active track. Press [RECORD] for grid recording, press numbered [TRIG] keys to place or remove hits, and press [PLAY] to hear the pattern.
  • Digitakt II sound and timing: press [SRC] for sample/source settings, [FLTR] for filter, [AMP] for envelope/level, [LFO] for modulation, [FX] for sends, and [TEMPO] for tempo and swing.
  • Digitakt II sync: press [SETTINGS] for MIDI Config → Sync. Use [YES] and the arrow keys to turn clock and transport receive on or off.
  • MicroFreak: use the Preset encoder to choose a bright saw or super-saw voice, Type to choose the oscillator, Cutoff and Resonance for the filter, Matrix for modulation routing, and Save only after the study behaves correctly.
  • MOTU M4: MicroFreak usually enters input 1. Digitakt II main L/R enter inputs 3/4. Inputs 3/4 have no front-panel gain knobs, so set their record level from the Digitakt II Main Volume.
  • Sound target: one bright arpeggiated synth voice and nothing else — no drums in the room.
  • Recording target: peaks around -12 dBFS in Live before any processing.
  • Copy target: a solo flip between REF and your print makes the shared drumless-groove behavior obvious.
  • Variation rule: change one variable per pass — gate length, step placement, or arpeggio order — and write it down.

Phase 1 · ~20 min

Put the reference in Live and write the hypothesis

Do this
  1. Drop the reference onto a track named REF.
  2. Loop the clearest 30-60 seconds — pick a stretch where the arpeggio is the only thing playing.
  3. Add Utility to REF and lower it until it is comfortable.
  4. Write: “The mechanism is ___.” Use the listening tag if you need a first guess: a drumless arpeggio whose gate placement is the groove.
  5. Write the one thing you will not copy: the supersaw timbre, the exact notes, the chord progression, or the mix polish.
  6. Create tracks named SOURCE, SYSTEM, and PRINT.
  1. Work in Live 12 only for this phase: keep REF looped from Lorenzo Senni — “Persona” and use a software synth, MIDI clips, the arpeggiator MIDI effect, and clip automation.
  2. Build the phase goal — Put the reference in Live and write the hypothesis — as the smallest Live set that proves a drumless arpeggio carrying the groove through note placement and gate.
  3. Keep every source on named tracks, then use Utility to match the level against REF before judging it.
  4. Record or resample the result to a fresh audio track named PRINT; do not leave the proof only as devices and MIDI.
  5. A/B ten seconds of PRINT against REF and change only the setting that most affects whether the arpeggio alone still grooves.
  1. Use Digitone II for this phase: translate Put the reference in Live and write the hypothesis into an FM lead track, sound locks, trig conditions, LFOs, macros, and Overbridge or main-out recording.

  3. Keep one clear performance control for the main change in this phase — gate length on the macro encoder — and name it before recording.

Expected result

You have a looped reference, three working tracks, and one written hypothesis before any programming starts.

Phase 2 · ~35 min

Program the arpeggio so its placement is the groove

Build the smallest system that can prove the hypothesis. One synth voice, no drums, a static chord. The point is to make the placement do all the rhythmic work, not to finish a track.

Do this
  1. Press [PTN], choose an unused pattern slot, and press [PLAY]. If you hear anything, stop and choose another slot.
  2. [TRK] + [TRIG 1] selects a MIDI track routed to the MicroFreak. Set it to send notes to the synth, not to trigger a sample.
  3. Choose one static chord — say a minor triad — and place its notes as a rising 16-step cell so it climbs and re-triggers.
  4. Press [RECORD] for grid recording and lock the pitches. Do not add a kick, hat, or any percussion track.
  5. Now set the groove with timing only: vary [AMP] gate length per step and nudge microtiming so some notes clip short and some ring. This staccato/legato contrast is the beat.
  6. Change one timing variable until the cell grooves on its own — short gates make it feel percussive, long gates make it flow. Do not solve a rhythm problem by adding a drum.
  7. On the MicroFreak, dial a bright saw or super-saw and keep the filter mostly open. Keep the patch plain so the placement stays exposed.
  8. Record one pass with the gate variation obvious, then one pass with all gates equal — hear how the groove collapses when placement is flattened.
  1. Work in Live 12 only for this phase: keep REF looped from Lorenzo Senni — “Persona” and use a software synth, MIDI clips, the Arpeggiator MIDI effect, and clip automation.
  2. Build the phase goal — Program the arpeggio so its placement is the groove — as the smallest Live set that proves a drumless arpeggio carrying the groove.
  3. Use per-note velocity and clip gate (note length) automation as your rhythmic engine; leave the Drum Rack empty.
  4. Record or resample the result to a fresh audio track named PRINT; do not leave the proof only as devices and MIDI.
  5. A/B ten seconds of PRINT against REF and change only the setting that most affects the felt groove.

Phase 3 · ~15 min

Route the studio so the print is clean

Make the signal path boring and dependable. If the recording path is confusing, you will misdiagnose the mechanism.

MicroFreak · input 1 + Digitakt II · inputs 3/4 → MOTU M4 → USB → Live 12 · Monitor Off
Do this
  1. Plug MicroFreak into input 1 — it carries the whole study here. Set the input gain while watching Live.
  2. Plug Digitakt II left/right into inputs 3/4 in case you resample, but expect it near silent.
  3. Set hardware tracks to Monitor Off. This prevents doubled monitoring.
  4. Arm one record track at a time and check peaks. Aim for -12 dBFS, not “as loud as possible.”
  5. If the MicroFreak is too quiet or too hot, adjust its level or input 1 gain and write the position down.
  6. Record 60-90 seconds of the system.
  7. Press ⌘ J to consolidate the best section into a clean clip named PRINT-A.

Phase 4 · ~20 min

Make the system map and the A/B proof

Draw the mechanism, then test the print against the reference. The map is not decoration; it is how you find what failed.

Do this
  1. Put REF and PRINT-A on adjacent tracks.
  2. Use Utility to match loudness before judging.
  3. Listen to 10 seconds of REF, then 10 seconds of PRINT-A.
  4. Write one sentence: “The first thing that gives mine away is ___.”
  5. Fix only that one thing and record PRINT-B.
  6. Keep both prints. The difference between A and B is the lesson.

Final check

Play the reference and your best print at matched loudness, then answer these in the logbook:

  • What mechanism did you copy?
    Answer

    Name the mechanism, not the genre: a drumless arpeggio whose note placement and gate length carry the rhythm.

  • Where does that mechanism live in your setup?
    Answer

    Point to it: the Digitakt II MIDI pattern (pitches, gate, microtiming) driving the MicroFreak — not a drum track.

  • What did you deliberately not copy?
    Answer

    The answer should be material: the supersaw timbre, the exact notes, the chord changes, or the mix polish. The workshop copies systems, not identity.

  • What rule carries into the related lab?
    Answer

    Write it as a usable instruction. Example: “If I need a kick to make a part groove, I have not yet made its placement do the work.”

  • Logbook — the miss list, three lines: Matched: ___ · Missed: ___ (best guess why) · Change next: ___
  • System diff sentence: “I thought the system worked like ___; after building it, I discovered it worked like ___.”
  • Carry into the lab: open listening-l3.html and reuse the corrected rule, not the reference sound.
