Probability Grammar

Conditional trigs turn the Digitakt’s sequencer from a tape into a rulebook: this step fires every third pass, that one fires 30% of the time, this one only on the first cycle. You stop writing what happens and start writing how often it happens — broken techno composed as a system of odds.

The point

A conditional trig is a die the sequencer rolls for you, on a schedule you wrote. Compose the odds well and every pass through the pattern is a new performance of the same piece — the grammar is yours, the sentences are the machine’s.

How this lab works

Write the grammar on paper first

  • A guaranteed skeleton: the steps that always fire, holding the broken-techno frame.
  • A rule table: per step, its condition (%, A:B, FILL, 1ST) and the musical job that condition does.
  • Test: you can read the table aloud as sentences — “the rim fires every fourth pass, except…”

Audition the futures

  • Let the pattern run 64+ bars and listen to what the odds actually produce, against what you predicted.
  • Tune probabilities like levels: a 10% step is seasoning, a 70% step is a riff with a stutter.
  • Test: the pattern surprises you without ever sounding wrong — wrongness means a rule, not a roll, is at fault.

Prove the reset, then print

  • Declare the master clock, predict bar 64, record one drift/flam test, and write the reset strategy — what returns every counter to state zero.
  • Print long takes: the piece is curated from real passes, never from an imagined average.
  • 6–8 minutes, fully original — the odds are the composition and the odds are yours.
Pass standard
Minimum passAudible original piece, routing + timing diagrams, fault entry, print map, and system diff exist.
Solid passYour written 64-bar prediction names which events are guaranteed, which are periodic, and which are dice — and the recorded test confirms the first two categories exactly.
Advanced transferYou can hear an unfamiliar generative pattern and say which parts are composed, which are periodic conditions, and which are probability — and design a rule table that would imitate it.

New concepts — in plain words

ConceptIn plain words
Conditional trigA sequencer step with an if attached: it only fires when its condition is true. The note is written; whether you hear it this pass is decided at runtime — a doorbell that only rings on weekdays.
Probability (%)A condition that rolls a die every pass: 30% means roughly one firing in three, with no memory of last time. Like a coin, it can come up heads four times running — percent is a tendency, never a schedule.
Cycle condition (A:B)A condition with a calendar instead of a die: 1:4 fires on the first of every four passes, 3:4 on the third. Deterministic — it is how you compose long arcs without writing long patterns.
True loop lengthHow long the pattern really takes to repeat once conditions are counted: a 1-bar pattern with 1:3 and 1:4 conditions repeats every 12 bars. The pattern you see is one bar; the piece the machine plays is twelve.
State & resetThe sequencer’s hidden memory: every A:B condition keeps a counter of which pass it is on. Reset is whatever sets those counters back to zero — stop, pattern reload, program change. If you don’t know your reset, you don’t know what bar one means.

2 · Bench preflight

The path — one machine performing, one machine listening

Digitakt (rules run here) → 2 × TRS → M4 in 3/4 → USB-C → Live (record long takes) → M4 out → phones
Pre-flight — before the first take
  • Rule table written and pinned: step · condition · musical job — the table is the score; the logbook gets a copy
  • Gain set against the densest possible pass: force all conditions true once (FILL or temporary 100%) and meter that
  • Recording plan: skeleton-only reference take · 64-bar prediction take · three long generative takes — in that order
  • Timing diagram drawn, condition counters included as a clock layer
  • Reset behavior tested cold: stop/start twice, confirm whether the A:B counters resumed or restarted — write the answer down before it ambushes a take
  • Phones at the mark · a break planned every fifteen minutes

3 · Build

Build the lab in five passes. Each pass produces an artifact, not just a better-sounding session: a written rule, a printed comparison, a fault entry, and a corrected system sentence.

Build recipe · Probability Grammar

Make the timing rule audible

Do this in order
  1. Write the time card. List every voice, its step length or rule, its reset behavior, and what should happen at bar 64 before programming.
  2. Program the neutral state. On Digitakt II choose an empty pattern. Use [TRK] + [TRIG 1] for the anchor voice, [TRK] + [TRIG 2] for the contrast voice, and [RECORD] to enter grid recording.
  3. Add the rule. Apply the lab rule only: p-lock variation, probability, per-track length, euclidean distribution, or rotation. Do not add fills until the rule is audible.
  4. Record the proof. Record a neutral pass and a rule pass into Live through the MOTU M4. Label them PRINT-0 and PRINT-RULE.
  5. Run the fault. Create the block fault: drift, flam, or reset failure. Predict the symptom, create it, diagnose it, fix it, and write the prevention rule.

SymptomLikely causeFix

If a section must repeat exactly, freeze it: render the pass and treat the audio as the part
The fault ritual — reset, on purpose

Predict what a queued pattern change mid-cycle will do to each A:B event — written per step → create the fault: switch patterns at bar 3 of a 4-bar cycle, then switch back → observe 32 bars and mark every periodic event that lands somewhere other than the table says → diagnose which counters restarted and which carried over → fix it: adopt one reset discipline (state zero before every take, or no mid-take pattern changes) → write the rule learned in the logbook as one sentence.

The full report bundle — ten items, filed in the logbook
  • Rendered piece — 24-bit WAV, master peaking −3 dBFS, 6–8 minutes, edited from real passes
  • Routing diagram and timing diagram (with the counter layer) — drawn before, corrected after
  • Recording plan — skeleton take, prediction take, generative takes; what was printed, when
  • Patch notes — the full rule table: step, condition, musical job, and the final tuned values
  • Session notes — including which takes were used and why those passes won
  • Reflection answers, including the listening comparison — which reference contributed what to the hypothesis
  • One removal decision — which rule was deleted from the grammar and what its absence did
  • Fault entry — symptom, cause, fix, prevention rule (faultType: reset)
  • Print map — what exists as audio vs pattern state; note that unprinted passes are unrecoverable by design
  • System diff sentence — “I thought the system worked like X; after building it, I discovered it worked like Y.”
  • Minimum pass gate: audible output · routing diagram · fault entry · print map · system diff. Without these five, the lab is incomplete regardless of audio quality — a musically strong piece cannot bypass the engineering spine.

Listening — the hypothesis source

Ten records that break the grid by rule — some with actual probability, some composed so densely they sound like dice. Learning to tell those apart by ear is this assignment’s real job. Each gets a brief: what to listen for before you press play, and how to check yourself after.

01 Blawan — “Why They Hide Their Bodies Under My Garage?” the lurch as engine

Broken techno whose hook is a vocal chop that never lands where the bar promises — the displacement is the riff.

Listen for
  • Where the vocal stab lands against your foot tap — early, late, absent — across eight bars.
  • The kick pattern: more regular than it feels; what actually breaks is everything around it.
  • Which events you can predict by bar four, and which still ambush you at minute three.
After — verify

Is the pattern random? No — relisten and you’ll find the same lurches recur: it is a fixed broken pattern, not a rolling die, and its power is that a composed irregularity reads as chaos on first listen. Your lab can get this effect two ways — write the lurch, or write odds that produce lurches. If every bar sounded different to you, relisten to bars 1–8 twice in a row and count what actually changed.

02 Shed — The Traveller techno that swerves

An album that keeps techno’s pulse but lets breakbeat grammar interrupt it — the grid is there, and it is repeatedly, deliberately disobeyed.

Listen for
  • The moments a straight kick gives way to broken drums — are they on section boundaries or mid-phrase?
  • Hi-hat figures that skip steps you expected — absence as an event.
  • How much of the “broken” feel is drum placement versus sound choice (rough breaks against clean techno tools).
After — verify

Did the swerves follow a schedule? Mostly yes — they arrive at structural points, which is A:B thinking: periodic disobedience, composed on a calendar. The lesson for your rule table: put the big breaks on cycle conditions and keep dice for small ones. If the swerves felt random, relisten counting bars between them.

03 Jlin — “Erotic Heat” composed to sound like dice

Footwork precision from Gary, Indiana: every hit placed by hand, yet the surface reads as unpredictable — the proof that “probabilistic” is a sound, not only a method.

Listen for
  • Phrases that repeat exactly — they exist, and finding them is the assignment.
  • The empty space: how silence is deployed as aggressively as the hits.
  • Triplet figures cutting across the duple grid — two time-feels running in one bar.
After — verify

Probability or composition? Composition — relistening reveals exact repeats, and exactness is the tell: dice never repeat a cluster perfectly. So the unpredictability you felt was authored, hit by hit. Your lab sits on the other side of the same coin: rules that generate what Jlin writes. If you found no repeats, relisten to one minute and hum the loudest figure until it comes back.

04 Autechre — “Flutter” (Anti EP) the anti-quantization statement

Written so that, by the artists’ account, no two bars repeat — a protest against a UK law targeting music with “repetitive beats.” Variation itself, made total, as both technique and statement.

Listen for
  • Try to clap the drum pattern of any bar — then notice the next bar refuses your clap.
  • What does persist: tempo, palette, the pads — the constants that keep total variation listenable.
  • Whether your ear builds an “average bar” that no single bar actually contains.
After — verify

If no bar repeats, why does it hold together as a track? The invariants: one tempo, one kit, one harmonic bed — variation needs a fixed frame to register against. That is your skeleton-plus-conditions architecture stated as a manifesto. And yes, your ear did construct an average bar; relisten and try to find it once — its absence from the actual audio is the piece’s joke and its lesson.

05 Errorsmith — “Lightspeed” one synth, rules audible

Contemporary machine funk built from a single softsynth — the variation grammar is so exposed you can almost read the patch from the speakers.

Listen for
  • How few distinct sounds there are versus how much variety they produce.
  • Pitch and timbre jumps that recur on a cycle — parameter changes behaving like A:B conditions.
  • The humor: events timed like punchlines, which means timed against an expectation the track itself built.
After — verify

Where did the variety come from, with one sound source? From parameter motion under rules — the same trick as p-locks plus conditions, scaled up. And the punchlines worked because the grammar first taught you what “normal” was; surprise is a budget you must fund with regularity. If you heard many instruments, relisten and track the timbre morphing instead of switching.

06 SND — Atavism the grammar of absence

Mark Fell and Mat Steel’s duo: clipped chords and dry percussion in patterns that withhold — the rule system is mostly deciding what not to play, pass after pass.

Listen for
  • Steps you expect to fire and that don’t — count the strongest absence in any 8 bars.
  • How a pattern “develops” by subtraction: the same figure returning with pieces missing.
  • The total dryness — no reverb to smooth the gaps, so every absence is structural.
After — verify

What was the most powerful event you heard? Almost certainly a silence — the step that didn’t fire. A condition that mostly suppresses is as compositional as one that adds; consider writing at least one rule in your table whose job is removal. If the music felt static, relisten to one track and map which slots of the bar are never used — the negative space is the design.

07 T++ — “Allied” broken percussion as texture

Torsten Pröfrock smearing breakbeat shrapnel across a techno frame — the broken events arrive in flurries whose density, not whose placement, is the composed quantity.

Listen for
  • Flurries versus single hits: events arriving in weather systems rather than on grid points.
  • The steady layer underneath that lets the flurries read as weather and not as collapse.
  • Whether any two flurries are identical — and whether it matters that they aren’t.
After — verify

Could you predict when a flurry would come better than what it would contain? Yes — and that split is the design: macro-time scheduled, micro-content loose. That maps directly to your rig: cycle conditions decide when, probability decides what. If the track read as mush, relisten following only the steady layer and let the flurries pass over it.

08 RP Boo — “Bangin’ on King Dr.” footwork’s source grammar

From one of footwork’s originators: sparse samples cycled against each other at 160, made for battling dancers — the music is a rule system the body has to solve in real time.

Listen for
  • The loops’ unequal lengths — a vocal cycle and a drum cycle that don’t share a downbeat for long stretches.
  • How few elements there are; the complexity is entirely relational.
  • The claps’ placement: drilled, repeating figures that a dancer can bet a move on.
After — verify

Was the unpredictability in the elements or between them? Between — each layer is simple and loyal to its own cycle; the collisions do the composing. That is a third grammar besides dice and calendars: fixed cycles of unequal length (lab 011 makes it the whole subject). If you lost the count, good — relisten and find which layer the dancers would trust; one layer is always trustworthy.

09 Rian Treanor — ATAXIA rule systems on the dancefloor

Contemporary algorithmic dance music from Sheffield: patterns generated and permuted by software rules, then aimed squarely at a sound system — the lab’s subject taken to its current edge.

Listen for
  • Permutation as development: the same cell re-ordered, re-cut, re-spaced rather than replaced.
  • Sudden global switches — the whole pattern state changing at once, like a rule being swapped mid-run.
  • Where it stays danceable and where it deliberately tips over — the boundary is the artistic decision.
After — verify

What developed over a track: the material or the rules? The rules — the cells stay recognizable while their ordering logic mutates, which is composing one level up from notes. Your rule table is a small version of the same move. If the switches felt arbitrary, relisten to one track and mark them on a timeline; their spacing usually has its own pattern.

10 Pearson Sound — “XLB” one odd event, total economy

A near-empty track carried by one pitched tom figure that refuses to resolve — minimal broken techno where a single irregular element does all the compositional work.

Listen for
  • The tom melody’s cycle — how long until it truly repeats, and whether it ever does cleanly.
  • How much silence the track risks, and what keeps tension alive inside it.
  • What changes across six minutes — almost nothing, and exactly enough.
After — verify

How many irregular elements does the track need? One — given enough space, a single off-cycle figure sustains an entire piece, which is an argument for a small rule table over a busy one. Where did the tension live? In the unresolved cycle: your ear kept predicting a resolution that the loop length withholds. If it felt empty, relisten in headphones at low level and notice how hard you start listening — emptiness is doing the compression’s job.

Replication lesson — lesson

Before touching gear, write a one-sentence hypothesis in the logbook naming what you stole from across these ten records — a composed lurch that reads as chaos, disobedience on a calendar, dice-feel that is actually authored, total variation over a fixed frame, surprise funded by regularity, rules that subtract, scheduled weather with loose contents, simple loyal cycles colliding, permutation as development, or one odd event in maximum space. The target is something like: compose the odds, not the notes — and know which kind of irregularity you are buying. Your sentence, in your words; the 64-bar prediction take is what will test it.

Track-lab principle

A probability is a compositional material with a failure mode: it has no memory. Use dice for weather, calendars for form, and know your reset — bar one is a claim about state, not a place on the grid.
