06 — The build

Step-by-step

Only now do we touch the gear. Each step is small, each step has a decision checkpoint, and every cable and click is spelled out. Keep the studio page open in another tab — it is the wiring reference this page assumes.

Step 01

Create the Ableton project

Open a new set
  • ⌘ N for a new Live Set. Press Tab to switch to Arrangement view — this whole build lives in Arrangement, not Session.
  • Set tempo to 124, time signature 4/4.
  • Right-click the timeline ruler → Time Ruler Format → Time to see positions as m:ss. The arrangement plan on page 05 is written in minutes, so this saves constant mental math.
  • Set the metronome's count-in to 1 Bar (click the arrow next to the metronome icon).
Audio preferences
  • ⌘ , → Audio. Input Device and Output Device: MOTU M4.
  • Sample Rate 48000 Hz, Buffer Size 256 samples. (Drop to 128 only if software monitoring feels laggy; raise to 512 for the mix session.)
  • Input Config: enable mono input 1 (MicroFreak) and stereo input 3/4 (Digitakt II). Rename them MicroFreak and Digitakt L/R so every routing chooser reads in plain language.
  • Output Config: stereo 1/2 only. Nothing else is needed.
Record & warp preferences
  • ⌘ , → Record, Warp & Launch.
  • File Type WAV, Bit Depth 24.
  • Turn OFF Auto-Warp Long Samples. Your takes are already clocked to Ableton; auto-warp would quietly time-stretch them and smear transients.
  • Turn OFF Loop/Warp Short Samples defaults for the same reason. Hardware recorded against MIDI clock should stay untouched audio.
Enable the Launchpad Pro MK3
  • Plug the Launchpad in via USB. Open ⌘ , → Link, Tempo & MIDI.
  • Control Surface: Launchpad Pro MK3. Input + Output: LPProMK3 DAW.
  • Enable Track and Remote for the LPProMK3 MIDI input — note data from Note and Sequencer modes.
  • Press Session on the Launchpad — pads light up confirming the link.
Enable Digitakt II and MIDI clock
  • Connect Digitakt II by USB. In Link, Tempo & MIDI, enable Track and Sync on the Digitakt II output row — Ableton sends clock and transport out to the box.
  • Digitakt II: Settings → MIDI Config → Sync. Enable clock receive and transport receive. Disable clock send — there must be exactly one clock master.
  • Find the MicroFreak port → enable Sync on its output row.
  • MicroFreak: Shift + Tempo → source USB.
  • Verify before moving on: press play in Ableton. The Digitakt II should start instantly and its screen should read 124. Press stop; it should stop. If either fails, fix it now — every later step assumes this works.
Create the tracks
  • ⌘ T four times for audio tracks: DT DRUMS, MF BASS, MF DRONE, ATMOS.
  • Optional later: duplicate DT DRUMS into DT KICK and DT HATS/PERC if you want separate recorded passes.
  • ⌘ ⌥ T twice for return tracks: A — REVERB, B — ECHO.
  • Return A: drop in Hybrid Reverb, decay 4–6 s, Dry/Wet 100 % (returns are always 100 % wet — the send knob is the mix control).
  • Return B: drop in Echo, sync mode, 3/16 ping-pong, Feedback 40 %, Dry/Wet 100 %, and engage its high-pass around 250 Hz so echoes never thicken the low end.
  • Rename with ⌘ R. Color-code tracks: drums one color family, synths another — you will be reading these lanes at a glance for hours.
Project hygiene
  • ⌘ S → save as hypnotic_lab_001.als inside a dedicated project folder.
  • All takes record into this project's Samples/Recorded folder automatically — never drag audio in from the desktop.
  • At the end of every session: File → Collect All and Save, so the project folder is self-contained.
Step 02

Connect and gain-stage the hardware

Physical signal chain
  • MicroFreak Audio Out → MOTU M4 Input 1 with one 1/4" TS cable. This is mono and unbalanced — that is correct for the MicroFreak's output.
  • Digitakt II Main L → MOTU M4 Input 3 and Main R → MOTU M4 Input 4 with two 1/4" TRS cables. The Digitakt's outputs are impedance-balanced; TRS keeps them quiet.
  • Inputs 3/4 are line inputs on the M4's rear panel. They have no gain knobs — the Digitakt's main volume is the record level for the drums.
  • Headphone output on the M4 → DT 990 Pro. Knob at halfway to start. The 250 Ω DT 990 needs more knob than consumer headphones — that is normal, not a fault.
Pick one audio route
Routing choices
RecommendedM4 as Ableton input/output; Digitakt analog into 3/4
Digitakt USBDigitakt II as Ableton input, useful for one-cable recording
OverbridgeUse later for deeper integration or separate Digitakt tracks
MicroFreak into DigitaktUse when sampling the synth into Digitakt II first
  • Do not monitor Digitakt II through both USB audio and M4 inputs at the same time. Pick one path before recording.
  • For the first build, use the recommended M4 path. It keeps Ableton, headphones, MicroFreak, and Digitakt II on one predictable audio clock.
Configure the DT DRUMS track — recommended M4 path
  • Select DT DRUMS. ⌘ ⌥ I to show the In/Out section if it's hidden.
  • Audio From → Ext. In, channel 3/4 (the stereo pair — not mono 3, not mono 4).
  • Monitor → In while programming the Digitakt. Switch to Auto before recording.
  • On Digitakt II, keep main volume strong but clean. In Ableton, peaks should sit around -12 dBFS to -6 dBFS.
Configure the MF BASS track
  • Select MF BASS. Audio From → Ext. In, channel 1 (mono).
  • Monitor → In while dialing sounds. Switch to Auto before recording.
  • Arm the track (red circle, or F when selected).
  • Set MF DRONE identically — same input, same monitoring discipline. Only one MicroFreak track is armed at a time.
Gain staging, stage by stage
Set once, in this order, then leave alone
1 — MicroFreak master volume~ 70–80 %. Hold a loud bass note while setting the next stage.
2 — M4 Input 1 gainTurn up until the M4's on-screen meter shows healthy green with peaks into amber. Red = back off.
3 — Ableton MF BASS meterPeaks between -12 dBFS and -6 dBFS. If it's low, add gain at stage 1 or 2 — never at the fader.
4 — Digitakt II main volumeStart at 12 o'clock. Play the busiest pattern you'll use and adjust until DT DRUMS peaks just below -6 dBFS. Don't run it near full — the Digitakt's output is hot enough to clip the M4's fixed line inputs.
5 — Phones knobComfortable level. This affects nothing on disk — set it and stop touching it.
  • If you'd have to push the M4 input gain into the red, push more level out of the synth instead.
  • Headroom is the point: peaks at -6 dBFS leave room for saturation, reverb build-up, and the master bus. A take that clips is a take you re-record.
Check timing alignment once
  • Record 8 bars of the Digitakt playing a kick on every downbeat (clock-synced, monitor on Auto).
  • Zoom all the way into bar 1 of the recorded clip. The kick transient will likely sit a few milliseconds after the grid line — that is the hardware's clock-start lag plus converter latency.
  • If the offset is consistent, you have two clean fixes: drag the recorded clip left so transients sit on the grid (per-take fix), or enter the measured value in ⌘ , → Audio → Driver Error Compensation (global fix for monitored recording).
  • Do this measurement once, now. A few milliseconds doesn't matter for a drone; it matters a lot when the kick and the sidechain pump disagree.
Test record
  • ⌘ ⇧ R — record 8 bars of MicroFreak, then 8 bars of Digitakt II.
  • Play back and listen on the DT 990s for: clipping, buzz, hum, a doubled/flanged quality (two monitoring paths), or one-sided drums (mono input chosen instead of the stereo pair).
  • Confirm the waveform looks healthy — visible dynamics, not a sliver and not a brick.
Alternative — Digitakt II over USB audio
  • Digitakt II: set USB mode to audio/MIDI or Overbridge, depending on the route you are using.
  • Ableton Audio Input Device: choose Digitakt II, or create a macOS aggregate device if you also need the M4 at the same time.
  • Record a quick 8-bar test. If the drums sound hollow, you are hearing the analog and USB versions together; mute one path.
  • Use this route when the Digitakt II is the only hardware you are recording, or when the MicroFreak is being sampled into Digitakt II instead of recorded separately.
Alternative — MicroFreak through Digitakt II
  • MicroFreak Audio Out → Digitakt II Input L. Use both Digitakt inputs only if the source is stereo.
  • On Digitakt II, use the external input/sampling page to capture a bass stab, drone fragment, or noise texture.
  • After sampling, record Digitakt II to Ableton from either M4 inputs 3/4 or USB audio. The MicroFreak is now part of the Digitakt performance, not a separate Ableton track.

Symptom → check

Step 03

Build the bass sequence

Choose the oscillator
  • Shift + press the Type encoder to initialise a fresh patch — never sculpt on top of a factory preset.
  • MicroFreak Type encoder → Wavetable.
  • Turn Paraphonic off — the bass is strictly monophonic. Set the keyboard octave so the root lands at A1.
Engine choices and what they buy you
WavetableThe default here. Sweeping Wave slowly morphs the harmonic content — movement without new notes.
HarmoAdditive engine. Cleaner, glassier; Timbre redistributes harmonics. Good if Wavetable feels too gritty.
SuperWaveDetuned stack. Big and wide-sounding — use with restraint, it eats space the drone needs.
KarplusStrPlucked-string physical model. Turns the bassline percussive; pairs beautifully with delay.
Starting knob positions
Oscillator + Filter
Wave~ 30 % — enough harmonics to give the filter something to bite
Timbre~ 50 %
Shape~ 40 %
Filter — Cutoff11 o'clock — dark; the track will open it later, not you
Filter — Resonance9 o'clock — a hint of bite, no self-oscillation
Glide5 – 10 % — only audible on deliberately slid steps
Envelope + LFO
ENV — Attack0 — the note must speak instantly against the kick
ENV — Decay / Sustain / Release25 / 20 / 30 — short and controlled, leaves silence between notes
LFO — Shape / RateSine / synced 1/4
Wire the mod matrix, click by click
  • Rotate the Matrix encoder until the cursor sits on the LFO → Cutoff intersection.
  • Press the encoder to select the cell, turn to set the amount to ~ 15, press again to confirm.
  • That small amount makes the filter breathe in quarter-note time — felt more than heard. If you can clearly hear a wobble, halve it.
  • Optional second route: Cycling Env → Timbre at ~ 10, with the cycling envelope in loop mode and a slow rise/fall. Two slow modulations at different rates never repeat exactly — that is the hypnosis mechanism in miniature.
Program the sequence
  • Length 16 steps. Scale: A natural minor. Root A1.
  • 3–5 notes only. Root on step 1, leave silences — the gaps are where the kick and the listener's imagination live.
  • A proven starting shape (steal it, then mutate it):

  • Step 4 is a ghost — same note, noticeably quieter. Step 7 is the accent. The slide into step 14 gives one acid-flavored gesture per bar, no more.
  • Leave Spice & Dice at zero for the first take. Variation comes later, deliberately, not from a randomizer on day one.
  • Confirm the MicroFreak is still clocked to Ableton (Shift + Tempo → USB) — the sequence must run at exactly 124.
Record the take
  • Loop and listen for at least 3 minutes before deciding it works. Set a timer. The pattern that bores you at 30 seconds and locks you in at 3 minutes is the right one.
  • Save the patch first: press Save, write it to an empty slot as LAB BASS. The take and the patch must both survive the session.
  • Arm MF BASS, Monitor on Auto. Position the playhead at bar 1. ⌘ ⇧ R. Record 4 minutes.
  • Keep your hands off the knobs for this take. The bass take is the static raw material; all movement will come from the synced LFO inside the patch and from Ableton automation in Step 07. A static take can be reshaped forever — a take with baked-in knob moves cannot be un-moved.
Decision check
  • Can I listen to this pattern for several minutes without skipping?
  • Does it create tension instead of resolving? (If it lands back on the root feeling "finished" every bar, displace or remove a note.)
  • Is any step "decorative"? Delete it and listen for two minutes — if nothing is lost, it stays deleted.
  • Does it leave headroom for kick and drone — both in frequency (space below 100 Hz) and in rhythm (silence on the downbeats the kick owns)?
Step 04

Build the drone

Save bass, init drone
  • Confirm LAB BASS is saved, then Shift + press Type to initialise a fresh patch. Do not edit the bass patch into a drone — you will want both later.
  • Switch the armed track: disarm MF BASS, arm MF DRONE, Monitor In while designing.
Drone parameters
Drone patch
OscillatorSpeech (eerie vowels), Wavetable (smooth morph), or Harmo (organ-like, controllable)
Wave / Timbre / Shape40 / 60 / 50
ParaphonicOn — you'll hold two notes at once
Filter — Cutoff10 o'clock (dark). The drone is felt at the edges, not stared at.
Filter — ResonanceLow — resonance + long reverb = whistling artifacts
ENV — Attack / ReleaseLong on both; the drone should swell, never click
LFO — Shape / RateSine / very slow — unsynced, around one cycle per 10–20 s
Mod 1LFO → Timbre, ~ 25
Mod 2Cycling Env (loop mode, slow) → Wave, ~ 15
  • Two modulators at different, unsynced rates means the timbre never repeats exactly — a 4-minute take stays alive with zero playing.
Capture a long take — with a knob choreography
  • Press Hold. Play root A1, then add the fifth (E2). Both sustain hands-free.
  • Arm MF DRONE, Monitor Auto. ⌘ ⇧ R. Record 4 minutes, following this script:
Choreography — slower than feels natural
0:00 – 1:00Hands off entirely. Let the internal modulation do the work.
1:00 – 2:00Cutoff from 10 o'clock to 12 o'clock — spread over the full minute.
2:00 – 3:00Timbre up by a quarter turn, equally slowly. Leave Cutoff where it is.
3:00 – 4:00Both drift back toward home. Ending near where you started makes the take loopable.
  • If you can hear a knob move as an "event", you moved too fast. The listener should only notice that things are different, never that they changed.
  • Save the patch as LAB DRONE.
Process the drone
Effects chain on MF DRONE — in this order
1 — EQ Eight — HP filter120 Hz, 24 dB/oct. The drone owns the mids; the bass and kick own everything below.
2 — EQ Eight — LP shelf-3 dB above 6 kHz — tames hiss before it hits the reverb
3 — Hybrid Reverb — AlgorithmConvolution Hall or Plate
3 — Hybrid Reverb — Decay6 – 8 s
3 — Hybrid Reverb — Dry/Wet35 %
4 — Utility — Width120 – 140 %. The recorded file is mono; this plus the stereo reverb is where its width comes from — deliberate, controllable, mono-compatible.
  • EQ before reverb: you want the cleaned-up drone reverberating, not a reverb full of rumble you then can't EQ out.
Decision check
  • Does the drone create the room the track lives in?
  • Is it sitting around -16 dB peak?
  • Solo the drone against the bass take: is the 80 – 200 Hz region muddy? If yes, raise the HP filter before reaching for anything else.
  • Flip Utility to mono for ten seconds — does the drone survive? If it vanishes, pull Width back.
Step 05

Build the Digitakt II drum foundation

Create a focused Digitakt II project
  • On Digitakt II, create or load a project named HYPNOTIC LAB. Use pattern A01.
  • Tempo comes from Ableton's clock. Press play in Ableton and confirm Digitakt II starts, stops, and shows 124.
  • Assign only four tracks at first: T1 KICK, T2 HAT, T3 PERC, T4 TEXTURE.
  • Leave tracks 5–16 empty. That is intentional; the unused tracks are headroom, not a to-do list.
Choose samples
  • T1: short, dark kick with controlled sub. Avoid long tails that fight the MicroFreak bass — audition each candidate while the bass take plays, not solo.
  • T2: tight closed hat. Short decay, slightly metallic or dusty.
  • T3: one rim, conga, click, or metallic tick. It should imply space without becoming a fill.
  • T4: a short resampled MicroFreak fragment, vinyl-like noise, or low-level field texture.
  • Pick one sample per track and commit. Browsing for "a slightly better kick" mid-build is the procrastination this course is designed to kill.
Shape each sound, page by page
T1 KICK — SRC / FLTR / AMP
SRC — TuneTune the kick's fundamental to A or E so it agrees with the bass. Detuned kick + tonal bass = permanent low-end blur.
FLTRGentle high-pass just enough to remove sub-rumble below the fundamental
AMP — DecayShort enough that the tail is gone before the bass note on the next step
FX sendsNone. A dry kick is a hypnotic-techno signature.
T2 HAT + T3 PERC + T4 TEXTURE
Hat — AMP decayShort; the hat must close fully before the next kick
Hat — FLTRHigh-pass around 500 Hz — hats carry no useful low information
Perc — FXSmall delay send, and only via p-lock on chosen trigs (below)
Texture — FLTRLow-pass mostly closed, opened very slowly by a track LFO (MOD page: LFO → Filter Frequency, slow speed, small depth)

Program on Digitakt II
  • Use grid recording. On T1 KICK, place trigs on 1, 5, 9, 13.
  • On T2 HAT, place trigs on 3, 7, 11, 15. Parameter-lock alternating velocity or volume (hold a trig, turn the volume knob) so the offbeats breathe instead of ticking like a clock.

Sidechain the bass to the kick
  • Simple route: Compressor on MF BASS. Enable Sidechain, Audio From DT DRUMS, Pre FX.
  • Cleaner route: record a separate muted-kick pass from Digitakt II to DT KICK (mute T2–T4, record 4 minutes), then use DT KICK as the sidechain source. The compressor now reacts only to the kick, not to hats bleeding through.
  • Settings: Ratio 4:1, Attack 1 ms, Release 120 ms, threshold down until the gain-reduction meter shows ~ -3 dB per kick.
  • Tune the release by ear: the bass should swell back between kicks, like breathing. Too short = no pump; too long = the bass never recovers.
Record the drum pass
  • Arm DT DRUMS. Set monitor to Auto. Record 4 minutes while performing Digitakt II mutes and small filter moves.
  • Perform with intent: bars 1–32 kick only, then unmute the hat, then the perc — a rough sketch of the arrangement arc, captured as audio.
  • Immediately save the Digitakt II project and the Ableton set. The pattern and the audio take are now linked decisions.
Decision check
  • Does the groove move without thinking?
  • Is the kick burying the bass? (Solo the pair; the sidechain should resolve the overlap, not a fader war.)
  • Is the hat supporting or fronting?
  • Listen to the full loop for three minutes: does the probability trig surprise you when it lands? If you can predict it, raise the randomness or make it rarer.
Step 06

Create the atmosphere layer

Pick one method. Don't stack them — one textured layer, not a wall. If you can't choose, take Option A: it recycles material you already have, which keeps the track's DNA consistent.

Option A — Reversed drone
  • Select the drone clip, ⌘ D, drag the copy to ATMOS.
  • Double-click the clip → Reverse in Clip view.
  • Auto Filter HP @ 300 Hz, resonance 15 %. Reverb send -6 dB.
  • Why it works: the reversed envelope swells toward beats instead of decaying away from them — anticipation for free, from material already in key.
Option B — Resample reverb tail
  • Temporarily drop a Hybrid Reverb on the master: decay 15 s, Dry/Wet 100 %.
  • New audio track, Audio From → Resampling. Arm. ⌘ ⇧ R. Record 16 bars of the track playing through the huge reverb.
  • Delete the master reverb afterwards — the captured tail moves to ATMOS and the master goes back to clean.
Option C — Stretched noise
  • MicroFreak Oscillator Noise. Wave 80, Timbre 30. Record 60 s into ATMOS.
  • Clip view → Warp on, Warp Mode Texture (grain size large). Set the clip's BPM field to 62 — half speed, doubled length, clouds instead of hiss.
  • Auto Filter BP @ 1.2 kHz → Reverb send -6 dB.
Option D — FX-only
  • Short drone fragment → Grain Delay: Freq 8.0, Pitch -7 st, Spray 60 ms, Dry/Wet 60 %.
  • Hybrid Reverb large algorithm, decay 10 s. EQ to live in the mids only.
Option E — Digitakt II resample
  • On Digitakt II, resample a short pattern with only T4 TEXTURE, delay, and reverb active.
  • Place the resample on a new Digitakt II track, reverse it or move the sample start, then record one quiet pass into ATMOS.
  • High-pass in Ableton around 250 Hz. Keep it below the point where it reads as another rhythm part.
Decision check
  • Muting the atmosphere should make the track feel smaller, not missing.
  • If you can clearly name what it's doing, it's too loud. Target around -20 dB on the fader.
  • Check it against the drone: do they fight in the same band? Carve one with EQ until they interlock.
Step 07

Draw the automation

Two kinds of movement — know which is which
  • Printed movement lives inside the audio you recorded: MicroFreak knob rides, Digitakt p-locks and LFOs. It cannot be edited anymore — only kept or re-recorded.
  • Drawn movement is Ableton automation on devices sitting on top of those recordings: Auto Filter, sends, Saturator, volume. Infinitely revisable.
  • This step is entirely about drawn movement. If a moment needs more printed movement, go back and re-record the take — don't fake it with extreme plugin automation.
Setup
  • Drop an Auto Filter (low-pass, 24 dB) on MF BASS — this is the lane the whole arc hangs on.
  • A to show automation lanes. Pencil (B) for drawing. Click a device parameter first and its lane appears on the track.
  • Right-click any segment → Curve (or drag the segment middle) to bend ramps into S-curves. S-curves read as intention; straight lines read as a ruler.
  • Rule of thumb: no ramp shorter than 8 bars in this genre. Most of yours will be 16–48 bars.

Perform automation on hardware
  • ⌘ M for MIDI map mode. Click Auto Filter Cutoff → tap a Launchpad pad/control. Set the mapping's Min/Max to 600 Hz / 3500 Hz so the full pad range covers exactly the musical range — no overshooting into harshness.
  • Map four controls and no more: cutoff, resonance, echo feedback, reverb send.
  • On Digitakt II, perform track mutes, filter sweeps, and FX send changes directly on the pattern while Ableton records audio — that's printed movement, captured live.
  • Exit map mode. F9 for Arrangement Record, play the whole 8 minutes, and ride the mapped controls slower than feels natural. Two takes, keep the calmer one.
  • After: right-click each lane → Simplify Envelope to thin the breakpoint clutter while keeping the shape.
Decision check
  • If you can hear the automation move, it's too fast.
  • Can you remove one lane without weakening the arc? If yes, remove it.
  • Play 4:00 – 5:00 blind: does the peak arrive because things opened, not because things were added? That's the test of the whole exercise.
Step 08

Place clips on the timeline

  • Locators turn the abstract arrangement plan into physical targets you drag clips against.

Clip placement, section by section
  • 0:00 – 1:00 / INTRO — Drone from bar 1, atmosphere underneath at -22 dB. Nothing rhythmic. The listener should settle into the room before anything asks for attention.
  • 1:00 – 2:00 / KICK IN — Digitakt II audio enters at bar 32, kick-only section of the recorded performance. If your drum take opened with the full pattern, ⌘ E-split it and use only the kick portion here, or lower the clip gain 2 dB for the first 16 bars.
  • 2:00 – 3:00 / BASS IN — Bass audio from bar 63. ⌘ E to split the first 4 bars and draw a fade-in via the clip corner handle, so the bass surfaces rather than arrives.
  • 3:00 – 4:30 / FULL — Hat section of the drum take from bar 94, perc audible around bar 110. Bass filter automation is climbing 600 Hz → 3.5 kHz through this whole span.
  • 4:30 – 5:30 / PEAK — Nothing new enters. Energy comes entirely from the automation peaks drawn in Step 07. This is the thesis of the genre, in one section.
  • 5:30 – 6:30 / STRIP — Perc gone, hat thinned for 32 bars, drum clip gain down 2 dB via automation. The strip-back makes the earlier fullness retroactively bigger.
  • 6:30 – 8:00 / OUTRO — Cut bass at bar 202, kick out around 7:00. Drone + atmosphere carry the exit; master volume draws a slow fade over the final 30–45 seconds.
Editing craft
  • Make every cut on a kick downbeat. ⌘ E at the playhead, then drag with grid snap on.
  • Where two takes of the same track meet, overlap them by one bar and use 4-beat crossfades (drag clip fade handles over the seam) — hard butt-joins click and lurch.
  • When a section's edits are final, select its clips and ⌘ J consolidate — one clean clip per section keeps the timeline legible.
  • Loop a suspicious seam with ⌘ L and listen three times before deciding it passes.
Rehearse the arc on Digitakt II first
  • Copy A01 to A02 and A03.
  • A01: kick only. A02: kick + hat. A03: full pattern with perc and texture.
  • Use Digitakt II pattern mutes to rehearse the 8-minute arc. Prepare mutes before releasing them so changes land cleanly on a bar line.
  • Record one full drum performance into DT DRUMS, then edit the best moments in Arrangement. Performance first, surgery second.
Polyrhythmic perc on Digitakt II
  • Set T3 PERC to a 14-step track length while the pattern remains 16 steps (per-track length is an Elektron superpower — find it in the page/length settings for the track).
  • Place 2–3 hits. 14 against 16 phase-shifts every bar and realigns every 7 bars — drift without adding new instruments.
  • Record a long pass into Ableton and keep only the sections where the drift supports the trance. The drift is seasoning, not the meal.

Step 09

Basic mix pass

Group and reset
  • Select DT DRUMS plus any optional separate Digitakt passes → ⌘ G → DRUMS.
  • MF BASS, MF DRONE, ATMOS → ⌘ G → SYNTHS.
  • Raise the buffer to 512 — latency is irrelevant now and the CPU headroom is free.
  • Pull all faders to -inf. Bring up one at a time, 30 s between each: kick first, then bass against it, then drone, then atmosphere, then returns. Each element earns its place against what's already there.

Frequency separation
  • DT DRUMS: EQ HP 25 – 30 Hz, then small cuts only. Fix drum balance on Digitakt II first — re-record the pass rather than fighting it with EQ.
  • MF DRONE: EQ HP 120 Hz, 24 dB/oct.
  • ATMOS: HP 220 Hz, LP shelf 8 kHz -3 dB.
  • MF BASS: narrow bell cut 200 – 300 Hz by 1.5 dB if woolly.
  • The pecking order below 200 Hz: kick owns 40–80 Hz, bass owns 80–180 Hz, nothing else is allowed in. Every low-end problem traces back to a violation of this.
Verification passes
  • Sidechain check: watch the bass compressor's gain-reduction meter through the FULL section — steady ~3 dB dips locked to the kick. If the dips vary wildly, the sidechain source has bleed; switch to the muted-kick pass.
  • Mono check: flip Utility Width to 0 % every few minutes — anything disappearing in mono = widening overdone (usual suspects: drone Utility, atmosphere reverb).
  • Solo-pair check: kick + bass alone for 16 bars; drone + atmosphere alone for 16 bars. Each pair should sound resolved by itself.
  • Volume check: listen once quietly. At low volume the mix's real balance shows — if the bass vanishes or the hat dominates, fix that before anything else.
Master bus check
  • Spectrum + Utility on master. Nothing else. No limiter for this exercise — loudness is a distribution problem, and this track isn't being distributed.
  • Peaks ≤ -3 dBFS across the whole 8 minutes, including the PEAK section's reverb bloom.
  • Take headphones off every 15 minutes. Ear fatigue is the main cause of bad mixes here — the DT 990's 7–10 kHz lift will have you carving highs that were never wrong.
