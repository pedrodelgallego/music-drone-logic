Lab Note B3-002
Probability · 04 — Setup

The rig, the signal path, and every condition the gear can make

Probability needs a stable clock and a clean capture path before it needs anything clever. This page wires the Digitakt II as the conditional brain, sets Ableton as clock master and recorder, and catalogues every trig condition the box offers plus the Live and Max for Live tools that mirror them.

Signal path

Ableton (clock master) → USB MIDI clock + transport → Digitakt II (conditions) → Main L/R → MOTU M4 in 3/4 → DT DRUMS audio track
  • Why this direction: conditions are evaluated by the Digitakt at the start of each pattern cycle. If the clock drifts or restarts unexpectedly, A:B counters reset and PRE/NEI chains break. One master clock (Ableton) feeding one slave (Digitakt) keeps the count honest across an 8-minute take.
Setup 01

Lock the clock

Ableton side
  • ⌘ , → Link, Tempo & MIDI. On the Digitakt II output row enable Track and Sync — Ableton sends clock and transport out.
  • Tempo 124, 4/4. The whole build assumes this.
Digitakt II side
  • Settings → MIDI Config → Sync: enable clock receive and transport receive. Disable clock send — exactly one master.
  • Verify: press play in Ableton. The Digitakt should start instantly, read 124, and stop on stop. Fix this before going further; conditional counters depend on it.
Setup 02

The Digitakt II trig conditions, in full

Where they live
  • Hold a trig and read the TRG page. The condition parameter scrolls through the full list below. A trig with a condition shows a small flag on the step.
  • Conditions evaluate per pattern cycle. They never alter the sound — only whether the step fires.
Percentage — %
  • Independent probability, decided fresh each cycle. 50 % plays roughly half the passes, with no memory.
  • The default tool for a single ghost hit. One 35 % perc makes a 16-step loop feel improvised while everything else stays locked.
A:B — the deterministic cycle counter
  • A:B means "play on the A-th of every B passes". 1:4 fires on cycle 1 of every 4; 3:4 on the 3rd.
  • Not random — a perfectly periodic clock. Stack several (1:4, 2:4, 4:4) and a four-bar phrase emerges from one pattern.
PRE / PRE (inverse) — depend on the previous conditional trig
  • PRE plays only if the most recent conditional trig on the same track did fire. The inverted form plays only if it did not.
  • Put a 25 % trig, then a PRE trig after it: the second hit only ever appears as a tail of the first — a two-note figure that arrives as a unit or not at all.
NEI / NEI (inverse) — depend on the neighbour track
  • NEI looks at the same step on the track directly above and plays only if that track's conditional trig fired (inverse: only if it didn't).
  • Use it to bind two tracks by chance: a clap at 40 %, and below it a reverb-tail sample on NEI, so the tail only ever follows a clap that actually happened.
FILL / FILL (inverse) — the performable state
  • FILL trigs play only while FILL mode is active (held momentarily, or latched). The inverse plays only while fill mode is off.
  • This gives two states of one pattern with no second pattern: the normal loop and a fill variant, switched on demand and captured live.
FIRST / 1ST — first cycle only
  • Plays only on the first pass after the pattern (re)starts. Useful for a one-shot intro accent that never returns within the loop.
Setup 03

Live MIDI tools & Max for Live that mirror conditions

Stock Live devices
Reproducing conditions in the box
Velocity (Random)Adds bounded random velocity to incoming notes — the velocity-ghost axis from page 03, in MIDI.
Note Length / Random MIDI effectRandom device sets a chance and range; chain after a clip to add independent-probability hits to a fixed MIDI pattern.
Clip "Chance" per note (Live 11+)Each note in a MIDI clip carries a probability slider — the direct equivalent of the Digitakt % condition.
Follow ActionsPeriodic, A:B-like behaviour at the clip level: play clip A three times, then clip B once.
Max for Live
  • Probability / dice devices on the M4L pack give per-step chance with a seed, so a "random" pattern can be made reproducible take to take.
  • Bouncy / euclidean M4L sequencers generate periodic structures you can layer under genuine randomness — the page-03 mix of form and texture, built in software.
  • Keep the same discipline as the hardware: bound every random parameter, and let one fixed element anchor the bar.
Decision check
  • Does the Digitakt start, stop, and read 124 from Ableton's clock, every time?
  • Can I name where each of the six conditions lives on the TRG page without looking it up?
  • Have I chosen one environment (hardware or Live) as the conditional brain for this track, rather than splitting it?
