Lock · 05 — Palette

The moves that build lock

Lock has a small vocabulary. There are four moves — swap the clock master, set the buffer, choose the monitor source, blend the direct monitor — and a handful of reference numbers. This page lays each one out with concrete values and the exact menu it lives in, so the build is a matter of choosing, not hunting.

The frame for this page

Every move here serves one of the two promises: a tighter clock or a cleaner monitor path. Before you change anything, say which promise the change serves. A move you can't justify against one of the two promises is fiddling, not lock.

Move 1 — clock master swap

Choosing who leads. Exactly one device may send clock; everything else receives. The default for this course is Live as master.

Clock master — the two configurations
Live as master (default)Live: Link, Tempo & MIDI → Sync ON for the Digitakt output port. Digitakt: clock + transport receive ON, send OFF. MicroFreak: source USB. Live's tempo rules.
Digitakt as masterDigitakt: clock + transport send ON. Live: enable Sync on the Digitakt input port; Live's tempo now follows the box. Use only when composing hands-on at the hardware.
ForbiddenBoth devices sending clock. This is two free-running tempos — guaranteed drift, no fix.
  • One sentence to confirm the swap: "Live is master; the Digitakt's screen reads Live's tempo and it starts on Live's bar one." If you can say that, the clock is locked. If the screens disagree, sync is broken.

Move 2 — buffer size

The single knob that trades delay against CPU. Lives in ⌘ , → Audio → Buffer Size. Change it per task, not per song.

Buffer size — pick per task
128Designing or playing a synth live through Live's FX, when 256 feels laggy. Watch for crackles — if they appear, your CPU has hit its limit.
256The default. Programming, recording, jamming. Stable and low enough to play against.
512 or higherMixing and exporting. The takes are printed; latency no longer matters and the CPU wants the headroom.
  • Re-measure after every change. The driver-error compensation value is only valid at the buffer it was measured on. Change the buffer, re-run the loopback, enter the new number.
  • Compensation ≠ feel. Raising the buffer makes the recording compensation different and makes the input feel later under your hands. Two effects, one knob — remember which you're chasing.

Move 3 — monitor source

Choosing which single path carries the input to your ears. Exactly one open at a time, always.

The three monitor sources — never two at once
M4 direct monitoringKnob toward input, Live track Monitor Off. Near-zero delay, dry. For designing and jamming.
Live software monitoringLive track Monitor Auto or In, M4 knob toward playback. Hears Live's FX, late by the round trip. For tracking when FX are part of the sound.
Digitakt's own outsAnalog L/R into M4 in 3/4, Live Monitor Off. You hear the box directly while a USB copy records silently — verify the printed copy after.
The one forbidden combination

M4 direct and Live software, both open on the same input, is the doubled-monitoring trap — comb filtering, the hollow phasey colour. There is no task it helps. One path, every time.

Move 4 — direct-monitor blend

The M4's monitor-mix knob sets how much live input you hear against the computer playback. It is the cue/main balance for hardware monitoring.

Where to set the monitor-mix knob
Fully toward inputYou hear only the live hardware, zero delay, no track. For sound design where the synth is the whole world.
CentredCue: live input balanced against the track. For jamming a part over an existing loop.
Fully toward playbackYou hear only the computer — required when software-monitoring, so the only input you hear is Live's processed copy.
  • The knob is listening level, not record level. It changes what you hear, never what is printed. Riding it while setting gain wrecks your level judgement — set it once, comfortable, and leave it.
  • When software-monitoring, go full playback. If any direct input bleeds through while Live also monitors, you've reopened the doubling trap.

The palette rule

Change one move at a time and re-listen. Lock problems compound — swap the clock master and the buffer and the monitor source all at once and you will never know which fixed it. One change, one check, then the next.

Before the build
  • I have chosen my clock master and can name the port it sends on.
  • I know my buffer size and the round-trip number it gives.
  • I have a default monitor source and know when I switch it.
  • I know the monitor-mix knob is listening level, not record level.
  • I can match each of the four faults to the move that fixes it.
