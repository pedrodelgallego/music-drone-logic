Lock · 04 — Setup

The rig, wired for lock

Two flows run in parallel through this rig: audio, and control + clock. Lock lives in both. This page wires the Digitakt, Live, and the M4 so that one clock drives everything and one monitor path carries every input — and explains every cable, every sync setting, and the one trap that doubles your sound if you skip it.

Ableton Live 12 Suite

  • The only clock master — sends MIDI clock + transport
  • Holds the buffer setting and driver-error compensation
  • Records the Digitakt's audio and any synth input

Elektron Digitakt II

  • Clock + transport receiver — runs at Live's tempo
  • Stereo main outs into the M4's line inputs 3/4
  • Its own tempo display must read Live's number

MOTU M4

  • 4-in / 4-out USB-C interface — the single audio device
  • Monitor-mix knob blends direct input against playback
  • Low, stable round-trip latency at 256 samples

Arturia MicroFreak

  • The live input you'll monitor and prove a path for
  • Mono out into M4 input 1; synced LFO follows Live
  • Used here to test direct vs software monitoring

Beyerdynamic DT 990 Pro

  • Open-back, 250 Ω — wide stage makes flam legible
  • Honest about doubling and comb-filter hollowness
  • Your entire monitoring world during this work

One ¼″ TS patch cable

  • The loopback cable for measuring round-trip latency
  • Out 3 → In 3 on the M4 rear, for the latency ritual
  • Keep it on the desk — hunting for it breaks the measurement

Why clock discipline is the whole game

If the clock jitters or two devices both try to lead, the rig drifts and no compensation fixes it. If two monitor paths are open at once, every input doubles. Lock both first — one clock, one path — and every later decision is one you actually made.

One clock to rule them all

Exactly one device generates tempo; everything else listens. If the Digitakt's screen shows a tempo different from Live's, sync is broken — stop and fix it before recording, because a take recorded against a drifting clock cannot be repaired.

  • No unpowered hubs for the clock. USB clock jitter is inaudible on a direct port and can become audible through a cheap hub. Plug the Digitakt and MicroFreak straight into the computer.
  • One audio device. Set the M4 as both the input and output device in Live. One device, one clock, no aggregate-device complexity.
  • Avoid the doubled-audio trap. If the Digitakt is connected by both analog cables and USB audio, decide which path Live records — recording both at once causes comb filtering. For this work, record the Digitakt from M4 In 3/4 and use USB for MIDI clock only.

The monitor path — exactly one open at a time

Avoid doubled monitoring
  • If a channel's M4 monitor mix is toward input and the Live track Monitor is In/Auto with the track armed, you hear that input twice — a few ms apart. That is comb filtering.
  • Pick one: M4 direct (knob toward input, Live Monitor Off) for instant, dry feel — or Live software (Live Monitor Auto, M4 knob toward playback) to hear effects, late by the round trip.
  • The fastest check: play a sustained note and sweep the filter. If the sound is hollow and phasey, two paths are open — kill one.

The latency measurement you cannot skip

Measure the round trip once, before any recording
  • Set Live to buffer 256 samples at 48 kHz, M4 as input and output device. Keep the phones knob at its mark.
  • Patch the ¼″ TS cable: M4 rear Out 3 → M4 rear In 3. The signal leaves the computer, passes both converters, and returns.
  • Send a sharp click out Ext. Out 3 only (not Master), record it on a track from Ext. In 3 with Monitor Off. Zoom to maximum, measure the gap from grid line to the click's first edge in ms.
  • Compare against Live's reported input + output latency. The shortfall is your driver error — enter it in ⌘ , → Audio → Driver Error Compensation. Re-record until the click lands on the grid.
Buffer size at 48 kHz — pick per task
128 samples~ 2.7 ms each way. Use if software monitoring feels laggy under your hands.
256 samples~ 5.3 ms each way, ~ 12–15 ms round trip. The default for recording and playing.
512+ samplesMix and export only. Latency no longer matters; CPU headroom does.
  • The loopback never touches Master. Route the click to Ext. Out 3 only and keep the recording track's Monitor Off, or the returned click feeds back on itself and howls.
  • One configuration, one number. The compensation value is only valid at the buffer and sample rate it was measured on. Change either and re-measure from scratch.

Tempo & settings

Recommended tempo: 122–126 BPM. Start at 124 BPM, buffer 256, sample rate 48 kHz. Reduced Latency When Monitoring stays off so every measurement is taken under the same conditions.
