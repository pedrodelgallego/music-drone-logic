Lab Note B1-003
Lock · 06 — The build

Bring a two-device rig into lock, step by step

Now we touch the gear. Every step is small, has a decision checkpoint, and produces something you can hear or measure. We make Live the clock master, lock the Digitakt to it with no drift, measure the round trip, and prove a single clean monitor path — then deliberately break it once, so you own the fix. Keep the setup page and the palette open in other tabs.

Step 01

Create the project and set the audio device

New set
  • ⌘ N, then Tab to Arrangement view. Tempo 124, signature 4/4.
  • ⌘ , → Audio: device MOTU M4 as both input and output, sample rate 48000 Hz, buffer 256 samples.
  • Input Config: enable mono input 1 (MicroFreak), stereo 3/4 (Digitakt), and mono 3 (for the loopback). Output Config: enable mono output 3.
  • Check the Options menu: Reduced Latency When Monitoring stays OFF for this entire build, so every number is taken under the same conditions.
  • Save as lock_b1_003.als in a dedicated project folder.
Decision check
  • Is the M4 selected as both input and output, at 48 kHz / 256?
  • Are inputs 1, 3, 3/4 and output 3 all enabled?
  • Is Reduced Latency When Monitoring off?
Step 02

Name the clock master and lock the Digitakt to it

Make Live the only clock master
  • ⌘ , → Link, Tempo & MIDI → MIDI Ports. On the Digitakt II output port row, enable Sync (this sends clock + transport). Leave the Digitakt input port's Sync off — you are not letting the box drive Live.
  • On the Digitakt: Settings → MIDI Config → Sync — clock receive ON, transport receive ON, clock send OFF.
  • MicroFreak: Shift + Tempo → source USB.
  • Say it out loud: "Live is master. The Digitakt and MicroFreak follow." Exactly one device is sending clock.
Confirm the lock
  • Program a Digitakt kick on quarter notes, one bar, nothing else. Press play in Live.
  • The Digitakt must start instantly and its screen must read 124. If the tempo display disagrees, or it doesn't start, sync is broken — fix it before doing anything else.
  • Stop and start Live a few times. The Digitakt should start cleanly on bar one every time. A device that follows tempo but starts on the wrong bar has transport receive off.
Live (master) → clock + transport → USB → Digitakt (follower)
Decision check
  • Does the Digitakt's screen read exactly 124 when Live plays?
  • Does it start on bar one every time you restart the transport?
  • Is exactly one device generating clock?
Step 03

Feel the buffer before you measure it

The buffer experiment
  • Create one audio track, input Ext. In 1 (MicroFreak), Monitor In — software monitoring, so the signal makes the full round trip before you hear it. Turn the M4 monitor-mix knob fully toward playback so you hear only Live's path.
  • Set the buffer to 1024. Play a fast line of short, separated notes. Then repeat at 256, 128, 64. At each size, write what the delay feels like in your own words.
  • Note the buffer at which the synth stops feeling like an instrument and starts feeling like a remote control. Set the buffer back to 256 — the working size.
What you should find
  • At 1024 the lag is obvious — you hear the note clearly after your finger. Somewhere between 256 and 64 your hands stop noticing. That point is personal: it is your playable threshold.
  • If 64 causes crackles, your computer has hit its limit there — write that down. The number you live at is the largest buffer you can comfortably play through.
Decision check
  • Can I name, in my own words, what each buffer size feels like?
  • Do I know my playable threshold in samples?
  • Am I back at 256 for the rest of the build?
Step 04

Measure the round trip with a loopback

Patch and send
  • Mute or lower the monitors; keep the phones knob at its mark. Patch the ¼″ TS cable: M4 rear Out 3 → M4 rear In 3. The signal leaves the computer, passes both converters, and returns.
  • Make a click track: a one-bar MIDI clip, quarter notes, driving a sharp click sample (attack at zero). Open its routing with ⌘ ⌥ I and set Audio To to Ext. Out 3 only — not Master. The loopback chain must never touch Master, or it can feed back.
  • Create an audio track: Audio From Ext. In 3, Monitor Off, armed. Record eight bars of the click.
Measure and compensate
  • Right-click the ruler → switch to minutes : seconds : milliseconds. Zoom to maximum on the first recorded click. Drag-select from the grid line to the click's first edge and read the length in ms. Convert if you like: samples = ms × 48.
  • Compare against Live's reported input + output latency in ⌘ , → Audio → Latency. The shortfall is your driver error — the part the interface's report missed.
  • Enter that difference in Driver Error Compensation (greyed out unless the external device is selected). Re-record and re-measure until the click sits on the grid line. Unpatch the cable.

Decision check
  • Is my measured round trip in the low-to-mid teens of ms (the minimum at 256 is ≈ 10.7 ms)?
  • Did entering compensation pull the click onto the grid?
  • Did I record the configuration alongside the number, so it's reusable?
Step 05

Try the three monitor paths and prove what prints

Walk each path
  • Path (a) — M4 direct: monitor-mix knob toward input, the MicroFreak track's Monitor Off. Play. Immediate, dry, no effects. This is the now under your hands.
  • Path (b) — Live software: knob toward playback, Monitor In, drop one obvious effect (a delay or saturator) on the track. Play. Processed, but late by your Step 04 number.
  • Path (c) — Digitakt's own outs: knob toward input, run a Digitakt pattern, arm a stereo track on Ext. In 3/4 with Monitor Off. You hear the box directly; a USB copy records silently behind your back.
  • After each, write one line: what was instant, what was late, what was processed.
Prove heard vs printed
  • For each path, record four bars and listen back. Compare the playback to what you remember hearing while you played.
  • Path (a): the recording matches your hands, but you never heard the FX. Path (b): you performed against a late copy — any drag is baked in. Path (c): what you heard is not the signal that recorded; the USB copy must be auditioned separately.

Decision check
  • Can I name, from memory, the heard-vs-printed difference for each path?
  • Is exactly one path open in each test — never two?
  • Have I chosen a default path for designing and one for tracking with FX?
Step 06

The 64-bar drift test — is it latency or drift?

Set up the measurement
  • Build a reference click in Live: a one-bar MIDI clip on quarter notes, frozen or rendered to an audio track named REF CLICK. This copy never leaves the computer, so its timing is exact.
  • With the Digitakt locked as a follower (Step 02), program a kick on quarters and route its mains to Ext. In 3/4.
  • Record the Digitakt's audio for 64 bars alongside the reference click, Monitor Off.
Measure at both ends
  • Using the Step 04 technique — ms ruler, maximum zoom — measure the gap between click and kick at bar 1 and again at bar 64.
  • A fixed small offset is latency: you can measure and compensate it. If the offset at bar 64 is larger than at bar 1, that is drift — and no setting fixes it. Drift means the clock setup is wrong: two masters, a bad hub, or an old driver.

  • Reset rule: stopping and restarting Live's transport realigns the follower to bar one. Write it as a rule — restart the transport before any take that matters.
Decision check
  • Is the offset fixed (latency) or growing (drift)?
  • If it drifts, did I confirm only one master and a direct USB port?
  • Have I written the reset rule into my logbook?
Step 07

The fault ritual — make doubled monitoring on purpose

Cause the fault
  • Arm the MicroFreak track. Set Live's Monitor to In and turn the M4 monitor-mix knob toward input at the same time. Now the synth reaches your ears twice — once direct, once through Live, a few ms apart.
  • Play a sustained note and slowly sweep the filter. Listen to the hollow, phasey, tin-can colour move. That is comb filtering — the same cancellation you heard in dub doubles, but here it is unwanted.
Fix it and hear the snap
  • Close one path: either turn the M4 knob fully to playback, or set Live's Monitor to Off. The sound snaps solid the instant the second copy disappears.
  • Do it a few times — open, hollow; close, solid. The contrast is the lesson. You will now catch this in a second when it happens by accident.
Decision check
  • Can I produce the hollow comb-filter sound at will?
  • Can I name which two paths were open?
  • Can I make the sound snap solid by closing exactly one of them?
Step 08

Record one locked, cleanly-monitored take

Put it all together
  • Clock: Live master, Digitakt follower, confirmed reading 124 and starting on bar one. Restart the transport once for a clean start.
  • Latency: compensation entered, recorded transients land on the grid.
  • Monitor: exactly one path. For a dry hardware take, M4 direct with Live Monitor Off. For a take with Live FX, Live Auto with the M4 knob to playback. Confirm only one is open.
  • Record a short pass — the Digitakt drums plus a MicroFreak line over them. Set the Digitakt main volume so the input peaks just below −6 dBFS.
Verify the take is in lock
  • Zoom into the recorded drums — the kick transients sit on the grid, not after it. No flam, one clean hit each.
  • Solo the MicroFreak take and listen: solid, not hollow. If it's phasey, a second monitor path was open while recording — though the print itself is dry, the lesson is to check before the take, not after.
  • Play 0:00 against the end of the pass — the Digitakt and Live are in the same relationship throughout. No drift.
Decision check
  • Do the recorded transients land on the grid?
  • Is the take free of flam and comb-filter hollowness?
  • Did the rig hold lock from the first bar to the last?
Step 09

Write the lock report and save the configuration

The three numbers, written down
  • In your logbook, record the three decisions as a single line: clock master = Live; buffer = 256 @ 48 kHz; monitor path = M4 direct (design) / Live Auto (FX takes).
  • Add the loopback table and the drift verdict (latency or drift). A compensation value without its configuration is a future problem — write the configuration next to the number.
  • File → Collect All and Save. Save the set as a template so the next track starts already in lock instead of from scratch.
Final decision check
  • Can I name my clock master, buffer size, and monitor path without looking?
  • Is the loopback number stored alongside the configuration it belongs to?
  • Could I rebuild this lock on a fresh set in under five minutes?
